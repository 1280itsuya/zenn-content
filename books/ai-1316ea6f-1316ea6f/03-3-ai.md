## 第3章　収益を自動化する：AIツールで「寝ている間に稼ぐ」仕組みの作り方

自動化の本質は「自分が寝ていても、システムが働き続ける状態を作ること」だ。本章では、AIを使って収益パイプラインを構築する具体的な手順を解説する。

---

## 自動化の基本設計：3層構造で考える

収益自動化は次の3層で設計する。

1. **生成層**：AIがコンテンツや商品を作る
2. **投稿層**：自動でプラットフォームへ公開する
3. **収益層**：アフィリエイトや販売で収益化する

この3層をつなぐだけで、朝起きたら記事が公開されアフィリ報酬が入る、という仕組みが成立する。

---

## ステップ1：Claude CLIでコンテンツ生成を自動化する

まずAnthropicのClaude APIを使い、記事を自動生成するスクリプトを作る。

```python
import anthropic

client = anthropic.Anthropic()

def generate_article(topic: str) -> str:
    message = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=2000,
        messages=[{
            "role": "user",
            "content": f"以下のテーマで1500字のブログ記事を書いてください：{topic}"
        }]
    )
    return message.content[0].text
```

トピックリストをCSVで管理し、`for topic in topics:` で毎朝ループ実行するだけで、複数記事を無人生成できる。

---

## ステップ2：Zennへ自動投稿する

生成した記事をZennのGit連携で公開する。Zennはリポジトリにpushするだけで記事が公開される仕組みだ。

```bash
# 記事ファイルを生成してcommit・push
python generate.py --topic "Pythonで業務自動化"
git add articles/
git commit -m "auto: 記事追加 $(date +%Y-%m-%d)"
GIT_TERMINAL_PROMPT=0 git push origin main
```

このスクリプトをWindowsのタスクスケジューラに登録し、毎朝7時に実行するよう設定する。

```
タスクスケジューラ → 新しいタスク作成
トリガー：毎日 07:00
操作：python C:\auto-money\run_pipeline.py
```

## ステップ3：収益化の仕組みを埋め込む

記事生成時、本文にAmazonアソシエイトリンクを自動挿入する。キーワードと商品URLの対応表をJSONで管理し、生成後に置換処理を走らせる。

```python
AFFILIATE_MAP = {
    "Python入門書": "https://www.amazon.co.jp/dp/XXXXXXXX?tag=YOUR_TAG",
    "自動化ツール": "https://www.amazon.co.jp/dp/YYYYYYYY?tag=YOUR_TAG",
}

def inject_affiliates(text: str) -> str:
    for keyword, url in AFFILIATE_MAP.items():
        text = text.replace(keyword, f"[{keyword}]({url})")
    return text
```

## 稼働後のチェック：週1回だけ数字を見る

完全放置ではなく、週1回だけダッシュボードを確認する習慣をつける。確認すべき数字は3つだけだ。

- Zennの記事ビュー数（上位記事のテーマを次週に増産）
- Amazonアソシエイトのクリック率
- パイプラインのエラーログ（`logs/pipeline.log`）

上位記事のテーマをフィードバックし、次の生成プロンプトに反映する。これが「稼ぐシステムを育てる」ループになる。

## まとめ

生成→投稿→収益化の3層を自動化し、タスクスケジューラで毎朝動かす。最初の構築に数日かかるが、一度動き出せば睡眠中もシステムが働き続ける。次章では、この仕組みをさらに拡張し「複数チャネルへの同時展開」を解説する。
