## 6ヶ月コスト内訳：合計¥47,200の実数値

6ヶ月（2025年12月〜2026年5月）の月次コストを集計した結果。

| 月 | Wan 電気代 | Flux API | SDXL 電気代 | 月計 |
|---|---:|---:|---:|---:|
| 12月 | ¥980 | ¥4,200 | ¥180 | ¥5,360 |
| 1月 | ¥1,100 | ¥5,800 | ¥200 | ¥7,100 |
| 2月 | ¥1,250 | ¥6,200 | ¥250 | ¥7,700 |
| 3月 | ¥1,300 | ¥7,400 | ¥230 | ¥8,930 |
| 4月 | ¥1,280 | ¥8,100 | ¥210 | ¥9,590 |
| 5月 | ¥1,320 | ¥7,000 | ¥200 | ¥8,520 |
| **合計** | **¥7,230** | **¥38,700** | **¥1,270** | **¥47,200** |

Flux APIがコスト全体の82%（¥38,700）を占める。Wan/SDXL のローカル電気代は RTX 4090（350 W）× 稼働時間 × 電力単価 ¥27/kWh で算出した。月平均4時間/日の生成で ¥1,200 前後に収まる計算になる。

```python
# scripts/local_elec_cost.py  GPU電気代の算出
GPU_WATT = 350
YEN_PER_KWH = 27

def local_cost_yen(hours: float) -> int:
    kwh = GPU_WATT / 1000 * hours
    return int(kwh * YEN_PER_KWH)

# 例: 月120時間稼働 → local_cost_yen(120) == 1134
```

## プロバイダ別エラー率：Wan 0.8%・SDXL 1.1%・Flux 3.2%

実測値（6ヶ月・総生成 14,230 枚）：

| プロバイダ | 生成枚数 | エラー数 | エラー率 |
|---|---:|---:|---:|
| Wan 2.2 ローカル | 8,410 | 67 | 0.8% |
| SDXL ローカル | 3,290 | 36 | 1.1% |
| Flux API | 2,530 | 81 | 3.2% |

Flux のエラー内訳は NSFW フィルター誤検知（41%）・タイムアウト（35%）・レート制限（24%）。ローカルモデルは OOM（SDXL）と CUDA カーネルクラッシュ（Wan）が大半を占める。

## ProviderDispatch v1→v7：変更7本の根拠

差分コミット7本の要点を抜粋する。

| Ver | 変更内容 | きっかけとなった障害 |
|---|---|---|
| v1 | 単純 if/else 分岐 | 初期実装 |
| v2 | タイムアウト追加（30s） | Flux が無応答のままプロセスが詰まる |
| v3 | リトライ3回 + 指数バックオフ | 429 レート制限で朝バッチが全滅 |
| v4 | Wan→SDXL フォールバック | Wan OOM で夜間バッチ停止 |
| v5 | 月次コスト上限ガード | Flux で月¥12,000 超過 |
| v6 | Prometheus メトリクス埋め込み | エラー原因が Slack ログだけでは不明 |
| v7 | Flux 専用非同期プール（並列4） | 同期呼び出しでスループットが上限に達する |

v5 の「コスト上限ガード」が最も効いた変更で、月次超過がゼロになった。

```python
# dispatcher.py  v5 で追加したコスト上限チェック
MONTHLY_FLUX_LIMIT_YEN = 8_000
YEN_PER_FLUX_IMAGE = 208  # $0.055/step × 25steps × ¥151/USD

class ProviderDispatch:
    def __init__(self):
        self._flux_spent_yen = 0  # 月次リセットは外部スケジューラに委ねる

    def _flux_within_budget(self) -> bool:
        return self._flux_spent_yen + YEN_PER_FLUX_IMAGE <= MONTHLY_FLUX_LIMIT_YEN
```

## Flux フォールバック実装：実効エラー率 3.2% → 0.4%

Flux エラーを放置すると月間81枚が無音で失敗する。フォールバックを3段階にした結果、失敗数は57枚→8枚（6ヶ月合計）まで減少した。

```python
# dispatcher.py  v7 のフォールバックチェーン
import asyncio
from typing import Optional

async def generate(self, prompt: str) -> Optional[bytes]:
    # 1st: Flux API（コスト上限内なら優先）
    if self._flux_within_budget():
        try:
            result = await asyncio.wait_for(
                self._flux_client.generate(prompt), timeout=45
            )
            self._flux_spent_yen += YEN_PER_FLUX_IMAGE
            return result
        except (TimeoutError, NSFWError, RateLimitError):
            pass  # 次段へ

    # 2nd: SDXL ローカル（VRAM 10 GB 以上空きがあるとき）
    if self._vram_free_gb() >= 10:
        try:
            return self._sdxl.generate(prompt)
        except RuntimeError:  # OOM
            pass

    # 3rd: Wan 2.2 ローカル（最終手段）
    return self._wan.generate(prompt)
```

Flux タイムアウト時に SDXL が吸収し、SDXL OOM は Wan が吸収する構造。各段の失敗はすべてメトリクスに記録する。

## Prometheus アラートクエリ：生成失敗率 > 2% で Slack 通知

v6 で埋め込んだメトリクスをベースにアラートルールを設定した。

```python
# metrics.py  アプリ側の計装
from prometheus_client import Counter

gen_requests = Counter(
    "image_gen_requests_total", "生成リクエスト数", ["provider"]
)
gen_errors = Counter(
    "image_gen_errors_total", "生成エラー数", ["provider", "reason"]
)

# 呼び出し側
gen_requests.labels(provider="flux").inc()
gen_errors.labels(provider="flux", reason="timeout").inc()
```

```yaml
# prometheus/rules/image_gen.yml
groups:
  - name: image_generation
    rules:
      - alert: HighGenerationErrorRate
        expr: |
          rate(image_gen_errors_total[10m])
          / rate(image_gen_requests_total[10m]) > 0.02
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "生成エラー率 {{ $value | humanizePercentage }} が閾値超過"
          description: "プロバイダ: {{ $labels.provider }} / 理由: {{ $labels.reason }}"
```

Slack への転送は Alertmanager の `slack_api_url` に Incoming Webhook URL を設定するだけで動く。

## 月次コスト自動集計スクリプト

毎月1日 09:00 に実行してコストレポートを Slack へ送る。

```python
#!/usr/bin/env python3
# scripts/monthly_cost_report.py
import os, requests
from datetime import date
from prometheus_api_client import PrometheusConnect

PROM = PrometheusConnect(url="http://localhost:9090")
SLACK_WEBHOOK = os.environ["SLACK_WEBHOOK_URL"]
GPU_WATT = 350
YEN_PER_KWH = 27
YEN_PER_FLUX_IMAGE = 208

def fetch_image_count(provider: str, days: int = 30) -> float:
    q = f'sum(increase(image_gen_requests_total{{provider="{provider}"}}[{days}d]))'
    result = PROM.custom_query(q)
    return float(result[0]["value"][1]) if result else 0

def local_elec_cost(provider: str, sec_per_image: int, days: int = 30) -> int:
    count = fetch_image_count(provider, days)
    hours = count * sec_per_image / 3600
    return int(GPU_WATT / 1000 * hours * YEN_PER_KWH)

def main():
    wan_cost  = local_elec_cost("wan",  sec_per_image=25)
    sdxl_cost = local_elec_cost("sdxl", sec_per_image=8)
    flux_cost = int(fetch_image_count("flux") * YEN_PER_FLUX_IMAGE)
    total     = wan_cost + sdxl_cost + flux_cost

    msg = (
        f"*月次コストレポート {date.today().strftime('%Y-%m')}*\n"
        f"Wan 2.2 電気代: ¥{wan_cost:,}\n"
        f"SDXL 電気代:   ¥{sdxl_cost:,}\n"
        f"Flux API:      ¥{flux_cost:,}\n"
        f"合計:          ¥{total:,}"
    )
    requests.post(SLACK_WEBHOOK, json={"text": msg})

if __name__ == "__main__":
    main()
```

crontab への登録例：

```bash
# crontab -e
0 9 1 * * /usr/bin/python3 /opt/imgpipe/scripts/monthly_cost_report.py
```

このスクリプトが返す数値が、本章の冒頭に載せたコスト表の生成元になっている。Prometheus を立てていない場合は `gen_errors` を SQLite に書き込む形に差し替えても同じロジックが使える。
