## 「自動販売機型」とは何か

自動販売機は、一度設置すれば24時間365日、人手なしで商品を売り続ける。AIを使った収益設計も、この構造を目指す。コンテンツを生成し、プラットフォームに掲載し、決済・納品まで自動化すれば、あなたが寝ている間にも売上が積み上がる。

重要なのは「販売フロー全体を自動化する」という発想だ。記事を書くだけ、商品を作るだけでは足りない。**集客→信頼構築→購入→納品**の4段階をつなぐパイプラインとして設計する必要がある。

## ステップ1：デジタル商品を自動生成する

まず「売れる形」にコンテンツを変換する。たとえばZennやDev.toに蓄積した技術記事をPDF化してBooth（同人誌即売サービス）で販売する構成が実践的だ。

```bash
# 記事をPDF化してBoothアップロード用ディレクトリへ
python src/generate_pdf_product.py --channel zenn --output data/booth/
```

AIに「読者が実際に使えるチェックリスト形式」で再構成させることで、無料記事とは明確に差別化した有料商品になる。

## ステップ2：集客を自動でまわす

商品を作っても、人が来なければ売れない。そこでZennやQiitaへの自動投稿をロングテールSEOの入口として機能させる。記事の末尾に商品へのCTA（行動喚起）を毎回自動挿入する。

```python
CTA_TEMPLATE = """
---
📦 この記事の実装をすぐ使えるキット → [Booth販売ページ](URL)
"""
```

これにより記事を書くたびに販売ページへの送客経路が1本増える。記事100本で100本の入口ができる「複利型」の集客構造だ。

## ステップ3：決済と納品を無人化する

BooothやGumroadはデジタルデータなら決済後に自動ダウンロードURLを発行する。つまり**人間が納品作業をする必要はゼロ**だ。月額サービスであればStripe＋スプレッドシート連携で請求書も自動化できる。

```bash
# Gumroadへの商品アップロードを自動化（presignフロー）
python src/gumroad_uploader.py --file data/booth/kit_v1.zip --price 1980
```

## ステップ4：実績データで信頼をループさせる

自動販売機が強くなるのは「売れた実績」が積み上がるときだ。AI予想サイトなら的中履歴、ブログなら閲覧数を自動で記録・公開する。

```python
# 日次実績をサイトのJSONに自動追記
python src/record_results.py --sport keirin --date today
```

実績が増えるほど信頼が上がり、購入率が向上する。これが「自動販売機が育つ」状態だ。

## まとめ：設計の核心は「自分が動かないこと」

自動販売機型マネタイズの本質は、**あなたの時間を使わずに価値が届く仕組み**を持つことだ。生成・掲載・決済・納品・実績蓄積のすべてをコードとプラットフォームに委ねる。初期設定に数日かかっても、その後は毎日無人で動き続ける。これが、AIを「道具」ではなく「従業員」として使う発想の核心である。
