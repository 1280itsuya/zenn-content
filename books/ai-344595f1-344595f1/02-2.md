## 収益の"歯車"を設計する――ツール選定からマネタイズ導線まで

自動化で稼ぐ仕組みは、複数の歯車が噛み合って初めて回る機械に似ている。歯車が一枚欠けても、あるいはサイズが合わなくても、全体は止まる。本章では「生成」「投稿」「収益化」という三つの歯車を選び、正しく繋ぐ方法を解説する。

## 歯車①　生成エンジンの選定

コスト構造を最初に固めることが最重要だ。APIを従量課金で使うと、月の記事量が増えるほど赤字になりやすい。筆者が行き着いたのは **Claude Code（Maxプラン月額定額）** をCLIから叩く構成だ。

```bash
claude -p "以下のテーマで800字のZenn記事を書いてください：$(cat topic.txt)" \
  > output/article.md
```

定額内に収まる限り生成コストはゼロ。記事を100本書いても200本書いても月額は変わらない。この"変動費ゼロ"が自動化投資の前提条件になる。

## 歯車②　投稿チャネルの選定

生成した記事をどこに流すかで、測定できる成果が変わる。筆者が重視したのは「プラットフォームが集客を担ってくれるか」という点だ。

| チャネル | 集客の主体 | 計測できる指標 |
|---------|-----------|--------------|
| Zenn    | プラットフォーム検索 | view数（即時） |
| Qiita   | プラットフォーム検索 | view数（即時） |
| 自前ブログ | Google SEO（3〜6ヶ月後） | クリック数（遅延大） |

立ち上げ期は自前サイトにこだわらず、ZennやQiitaへ自動投稿してview数を早期に回収する方が合理的だ。

```python
# posters/zenn.py の核心部分
def publish(article_path):
    subprocess.run(["npx", "zenn", "preview"], check=True)
    subprocess.run(["git", "add", article_path], check=True)
    subprocess.run(["git", "commit", "-m", f"add: {Path(article_path).stem}"], check=True)
    subprocess.run(["git", "push"], check=True)
```

## 歯車③　マネタイズ導線の設計

view数を収益に変えるルートは複数用意する。記事の末尾に「関連ツール → Booth/Gumroad商品」へのリンクを挿入し、読者をそのまま購入ページへ送る。

```
記事（Zenn view）
  └─ 記事末尾CTA
       └─ Booth商品ページ（¥980〜¥2,480）
            └─ 即時ダウンロード → 収益確定
```

重要なのは **導線を自動で埋め込む** ことだ。記事生成スクリプトのテンプレートにCTAブロックを含めておけば、100本の記事すべてに漏れなくリンクが入る。

## 三つの歯車を「スケジューラ」で繋ぐ

生成・投稿・導線が揃ったら、あとはタスクスケジューラで毎朝7時に起動するだけだ。

```
[タスクスケジューラ 07:00]
  → orchestrator.py
       → 生成エンジン（Claude CLI）
       → Zenn/Qiita投稿
       → レポート通知（Discord）
```

歯車の設計で肝なのは、**一か所を止めずに済む構造を作ること**だ。たとえばQiitaがレート制限でブロックされても、Zennへの投稿は続く。チャネルを複数持ち、各歯車が独立して動くよう設計しておくと、システム全体の稼働率が大幅に上がる。
