## Notion APIで週10件の「価値情報」を5分で一括収集する

Notionの「学んだこと」「試したツール」「解決したエラー」を週10件蓄積するだけでよい。これをAPIで取得する関数が全体の起点になる。

```python
import os
import httpx
from datetime import datetime, timedelta

NOTION_TOKEN = os.environ["NOTION_TOKEN"]
DATABASE_ID  = os.environ["NOTION_DATABASE_ID"]

def fetch_weekly_notes() -> list[dict]:
    since = (datetime.utcnow() - timedelta(days=7)).isoformat() + "Z"
    headers = {
        "Authorization": f"Bearer {NOTION_TOKEN}",
        "Notion-Version": "2022-06-28",
        "Content-Type": "application/json",
    }
    payload = {
        "filter": {"property": "作成日時", "date": {"on_or_after": since}},
        "page_size": 20,
    }
    res = httpx.post(
        f"https://api.notion.com/v1/databases/{DATABASE_ID}/query",
        headers=headers, json=payload,
    )
    res.raise_for_status()
    pages = res.json()["results"]
    return [
        {
            "title": p["properties"]["名前"]["title"][0]["plain_text"],
            "body": (p["properties"].get("メモ", {})
                     .get("rich_text", [{}])[0].get("plain_text", "")),
        }
        for p in pages
    ]
```

プロパティ名（`名前` / `メモ`）は自分のDBに合わせて変更する。

## claude-sonnet-4-6 で構成案+本文初稿を50行で生成する

週次メモをまとめてClaudeに渡し、Zenn記事初稿を生成する。1呼び出しのコストは入力約2,000トークン+出力約1,500トークンで約$0.009（≒¥1.4）。月20本でも¥28。

```python
import anthropic
from datetime import datetime

client = anthropic.Anthropic()

SYSTEM_PROMPT = """\
あなたはZennでバズ実績のあるエンジニア向けライター。
与えられたメモ群から1本のZenn記事を生成する。
要件:
- タイトルに固有名詞と数値を必ず含める
- ## 小見出し4〜5個、各見出しにコードブロック1つ以上
- 「思います」「ぜひ」「まとめ」は使わない
- 出力はMarkdownのみ
"""

def generate_article(notes: list[dict]) -> str:
    notes_text = "\n\n".join(f"### {n['title']}\n{n['body']}" for n in notes)
    msg = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=4096,
        system=SYSTEM_PROMPT,
        messages=[{"role": "user", "content":
            f"以下のメモ群からZenn記事を1本生成してください。\n\n{notes_text}"}],
    )
    return msg.content[0].text

if __name__ == "__main__":
    notes   = fetch_weekly_notes()
    article = generate_article(notes)
    slug    = datetime.now().strftime("%Y%m%d")
    path    = f"articles/{slug}.md"
    with open(path, "w") as f:
        f.write(f"---\ntitle: ''\npublished: false\n---\n\n{article}")
    print(f"Generated: {path}")
```

`published: false`で保存し、タイトルだけ手動確認してから`true`に変える。ここが唯一の人間ゲートで、所要30秒。

## zenn-cli × GitHub Actions で自動デプロイする.yml全文

```yaml
# .github/workflows/weekly-draft.yml
name: weekly-draft

on:
  schedule:
    - cron: "0 22 * * 0"   # 日曜22時UTC = 月曜7時JST
  workflow_dispatch:

jobs:
  generate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"

      - run: pip install anthropic httpx

      - name: Generate draft
        env:
          NOTION_TOKEN:        ${{ secrets.NOTION_TOKEN }}
          NOTION_DATABASE_ID:  ${{ secrets.NOTION_DATABASE_ID }}
          ANTHROPIC_API_KEY:   ${{ secrets.ANTHROPIC_API_KEY }}
        run: python scripts/generate_draft.py

      - name: Commit and push
        run: |
          git config user.name  "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add articles/
          git diff --cached --quiet || git commit -m "draft: weekly auto-generated"
          git push
```

ZennはGitHubリポジトリ連携でpushと同時に反映される。`published: false`の記事はZennの下書き一覧に表示されるだけで公開されない。

## view≒0だった記事パターン3つと次の改稿アクション

**パターン1: タイトルに固有名詞ゼロ**  
「副業で月5万稼ぐ方法」→ view 12。「Claude API×GitHub Actionsで副業月収¥15,000の自動化全手順」→ view 2,300。タイトルに固有名詞+数値を入れるだけで190倍の差が出た。

**パターン2: 本文にコードブロックなし**  
ZennはLGTM数がランキングに直結する。概念説明のみの記事はLGTMされず埋もれる。コードなし記事を洗い出すワンライナー:

```bash
# コードブロックが存在しない記事を列挙する
grep -rL '```' articles/ | sort
```

このリストを改稿対象キューにして消化順に並べる。

**パターン3: 問題設定が抽象すぎる**  
「生産性を上げたい人向け」では検索されない。「Notionのメモ50件がZenn記事になるまで0秒」のように変換量・時間・ツール名の組み合わせが検索意図と一致する。

次のアクション: 既存記事のタイトルを上記3基準で採点し、スコア2点以下の記事から順番に改稿する。改稿後のviewを1週間で測定し、2倍増えなければタイトルを再変更する。改稿スクリプトも同じ`generate_article()`を使い回せる。
