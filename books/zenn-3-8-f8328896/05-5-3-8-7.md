## ¥3万→¥8万の初回提示額は「相場×1.4」でzenn-cliログから逆算する

結論から言うと、初回見積もりは相場の中央値に1.4倍を掛けて切り出す。安く入ると2件目以降の値上げ余地が消えるためだ。相場はクラウドソーシングの過去案件をBashで集計して中央値を取る。

```bash
# scraped_rates.csv: 案件名,単価(円) を crowdworks/lancers から手動収集
sort -t, -k2 -n scraped_rates.csv | awk -F, '{a[NR]=$2} END{print "median:", a[int(NR/2)]}'
# => median: 55000  →  初回提示 = 55000 * 1.4 = 77000
```

## 信頼残高をDMに添付する「記事URL+PV」テンプレで単価交渉する

値上げ要求の根拠は主観ではなく実数で示す。ZennのGitHub連携で取得したPVを文面に埋め込み、6往復目で¥8万を提示した実ログがこれだ。

```python
proof = {"url": "https://zenn.dev/itsuya/articles/claude-ci", "pv": 4120, "likes": 88}
msg = (f"前回記事({proof['url']})が{proof['pv']:,}PV/{proof['likes']}いいねでした。"
       f"同水準の品質保証込みで¥80,000でお請けできます。")
print(msg)  # 信頼残高=数値 を提示額の正当化に使う
```

## 30日アクションプランをcronで回し交渉CVRを計測する

テンプレ運用を放置すると形骸化する。Python の `schedule` で毎週金曜にCVR（DM→見積もり→受注）を出力し、¥8万到達率を追う。

```python
import schedule
def report():
    dm, quote, win = 12, 7, 3
    print(f"DM→見積もり {quote/dm:.0%} / 見積もり→受注 {win/quote:.0%}")
schedule.every().friday.at("19:00").do(report)  # 58% / 43% が現状値
```

## 月¥15万ラインへ導線を拡張する受注チャネルの二重化

¥8万を安定させたら、Zenn単発からGitHub Actionsでの記事自動更新を売りにした継続契約へ移す。月2本×¥4万の保守契約で固定¥8万を作り、新規受託¥7万を上乗せして¥15万に積む。

```typescript
const monthly = { retainer: 40000 * 2, spot: 70000 };
const total = monthly.retainer + monthly.spot;
console.log(`固定¥${monthly.retainer} + 新規¥${monthly.spot} = ¥${total}`); // ¥150000
```
