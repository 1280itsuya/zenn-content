## 半年37本・成約6件・平均単価¥8万の実数テーブル

まず逆算の根拠となる生データを置く。質より導線が単価を決めた構造がこの表で見える。

```python
funnel = {
    "記事公開":   37,      # 本
    "総閲覧":      241_320, # PV
    "プロフ遷移":  4_180,   # CVR 1.73%
    "DM/問い合わせ": 19,     # CVR 0.45%
    "見積もり提示": 11,
    "成約":        6,       # 見積もり成約率 54.5%
    "平均単価_前": 30_000,
    "平均単価_後": 80_000,
}
print(round(funnel["成約"] / funnel["DM/問い合わせ"] * 100, 1))  # 31.6 %
```

閲覧24万でDMが19件しか出ていない。律速は「文章」ではなく閲覧→DMの0.45%だった。

## 受注導線CVRを測る計測スキーマ（Google Sheets連携）

②の計測シートの列定義。段ごとのCVRを毎週自動算出し、最も低い段だけ改善する。

```python
import csv
rows = [["date","article","pv","profile","dm","quote","won","unit_price"]]
def cvr(num, den): return f"{num/den*100:.2f}%" if den else "0%"
# 例: 直近週 pv=18,400 profile=290 dm=2
print("profile_cvr:", cvr(290, 18400))  # 1.58%
print("dm_cvr:",      cvr(2, 290))       # 0.69%
```
