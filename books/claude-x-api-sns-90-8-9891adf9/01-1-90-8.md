## 90分の内訳を行動ログで計測する

副業アカウント運用の1日90分は、感覚ではなく `time` コマンド付きのログで分解した。3週間の中央値がこれだ。

```bash
# daily_log.sh : 作業の開始/終了をタイムスタンプ記録
echo "$(date +%H:%M:%S) $1" >> ~/x_work.log
# 集計結果(21日間の中央値)
# TL巡回         42分
# ミュート判断   18分
# 有益投稿の保存 30分  → 合計 90分
```

42分のTL巡回が最大の浪費で、ここを丸ごとClaudeに渡す。

## ゴールは毎朝7時のDiscordダイジェスト

完成形は、cronが叩いた8分の処理結果がDiscordに届く1通のメッセージだ。実際の出力を先に見せる。

```text

```

90分が、目視ゼロの8分12秒に置き換わる。

## X API + Claude + Python の構成図

部品は4つに分かれ、第2章以降で1つずつ組む。レート制限と429は第3章で実測する。

```python
# pipeline.py : 全体の骨格(各関数は2章以降で実装)
def main():
    tweets = fetch_timeline(max_results=300)   # 第2章: X API取得
    judged = classify_with_claude(tweets)      # 第3章: 適合率0.91の判定
    mute_low_value(judged)                     # 第4章: 自動ミュート
    post_to_discord(summarize(judged))         # 第5章: 配信

if __name__ == "__main__":
    main()
```

## 取得・判定・ミュート・配信の責務分担

各部品の入出力と、章で晒す実測値を表に固定する。

```yaml
parts:
  fetch:    {章: 2, 入力: bearer_token, 出力: 300件, 計測: 429リトライ回数}
  classify: {章: 3, 入力: tweet本文,   出力: keep/mute, 計測: 適合率0.91/再現率0.88}
  mute:     {章: 4, 入力: mute判定,    出力: muting/users API, 計測: 誤ミュート率2%}
  deliver:  {章: 5, 入力: keep投稿,    出力: Webhook, 計測: 整形後9件}
```

## 章末リポジトリと実行コマンド

最終的に手に入るのは、`git clone` 後にこの3コマンドで動く構成だ。

```bash
git clone https://github.com/example/x-auto-mute && cd x-auto-mute
cp .env.example .env          # X bearer / ANTHROPIC_API_KEY / DISCORD_WEBHOOK
pip install -r requirements.txt
python pipeline.py            # 手動実行で8分版を即確認
# 自動化はこの1行をcronに登録(毎朝7時)
# 0 7 * * * cd ~/x-auto-mute && /usr/bin/python pipeline.py
```

何が削れて何が動くかは、この無料章の出力例とコマンドで判断できる。月額コストの内訳(Claude API ¥31/日)とX APIの429挙動の実測は、次章から数値で開く。
