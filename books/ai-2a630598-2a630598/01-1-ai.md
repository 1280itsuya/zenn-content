## AIは「疲れない社員」である

午前3時。あなたが眠っているとき、サーバー上のスクリプトはすでに動き出している。

```bash
# Task Schedulerが毎朝7時に自動起動する例
python orchestrator/main.py --mode morning
```

このコマンド一行が、記事の生成・投稿・アフィリエイトリンクの埋め込みまでを一気に実行する。人間が関わるのは、最初にシステムを組んだ数日間だけだ。

## 「自動収益」とは何か——誤解を解く

よくある誤解は「AIに任せれば何もしなくていい」という幻想だ。正確には違う。**自動収益とは、あなたの労働を「仕組みへの初期投資」に変換すること**である。

たとえば本書で構築するシステムは、次の流れで動く。

1. **コンテンツ生成** — Claude APIがテーマを受け取り、Zenn・Qiita向け技術記事を自動執筆
2. **品質ゲート** — LLM-as-judgeが「タイトルと本文の一致」「具体性スコア」を検査し、不合格は差し戻し
3. **投稿** — Git pushでZennへ公開、Qiita APIで同時配信
4. **収益化** — 記事末尾にGumroadやBooth商品へのCTAを自動挿入

人間の作業は「システムの設計」と「月1回の精度チェック」のみ。それ以外の繰り返し作業はすべてコードが担う。

## 全体アーキテクチャ——5分で掴む鳥瞰図

本書のシステムは**3層構造**で成り立つ。

```
[Layer 1] スケジューラー  ← Task Scheduler / cron
    ↓
[Layer 2] オーケストレーター ← orchestrator/main.py
    ↓
[Layer 3] 各エージェント
    ├── content_agent.py   # 記事生成
    ├── poster_zenn.py     # Zenn投稿
    ├── poster_qiita.py    # Qiita投稿
    └── affiliate_agent.py # リンク挿入
```

Layer 1がトリガーを引き、Layer 2が全体を指揮し、Layer 3の各エージェントが専門タスクを実行する。役割が分離しているため、「Qiitaだけ止める」「新チャネルを追加する」といった変更が1ファイルの修正で済む。

## なぜ今、このシステムが成立するのか

3つの条件が2024年以降に揃った。

- **LLMのAPI化** — GPTやClaudeが月数千円で呼び出せるようになった
- **プラットフォームのオープン化** — Zenn・Qiita・Booth・GumroadがAPIを公開
- **Claude Maxの定額制** — 従量課金ゼロで重いプロンプトも実行可能

これらが組み合わさることで、**個人が「編集部」を持てる時代**になった。本書はその構築手順を、コピー&ペーストできるレベルで解説する。

次章から、環境構築・API取得・最初のエージェント起動まで、一歩ずつ進めていこう。
