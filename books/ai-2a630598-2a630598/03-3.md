## 収益自動化の全体像

「寝ている間に稼ぐ」とは比喩ではない。AIエージェントがコンテンツを生成し、プラットフォームへ投稿し、アフィリエイトリンクを経由して購買が発生する——この連鎖が24時間止まらずに回り続ける状態のことだ。本章ではその設計図を具体的に示す。

## 収益が発生する3つの経路

自動収益は大きく3経路に分類できる。

**① コンテンツ×アフィリエイト**：AIが記事を書き、Amazon・楽天のアフィリエイトリンクを埋め込んで投稿する。読者が購入するたびに報酬が積み上がる。

**② デジタル教材の自動販売**：Gumroad・Boothに商品を登録しておけば、決済から納品まで完全無人で完結する。

**③ SaaS型サブスクリプション**：月額制サービスは一度契約を取れば翌月以降も自動課金される。コンテンツは自動生成で定期更新する。

## パイプラインの設計図

```
[スケジューラ(cron)] 
  → [テーマ選定エージェント]
  → [記事生成エージェント(Claude API)]
  → [品質チェックエージェント]
  → [投稿エージェント(Zenn/はてなブログ/Dev.to)]
  → [収益レポート集計]
```

毎朝7時に起動するよう `cron` を設定するだけでよい。

```bash
# crontab -e
0 7 * * * cd /home/user/auto-money && python orchestrator.py >> logs/daily.log 2>&1
```

`orchestrator.py` の中では各エージェントを順番に呼び出す。

```python
from agents import theme_agent, writer_agent, poster_agent

theme  = theme_agent.run()          # 今日のテーマを決定
draft  = writer_agent.run(theme)    # Claude APIで本文生成
result = poster_agent.run(draft)    # Zennへ自動投稿
```

## 「品質ゲート」が収益を守る

エージェントが量産した記事をそのまま公開するのは危険だ。タイトルと本文が食い違う「ドリフト」や、プレースホルダのままのリンクが残るケースが頻発する。必ず品質チェックを挟む。

```python
def quality_gate(title: str, body: str) -> bool:
    # タイトルの主要キーワードが本文に含まれるか検証
    keywords = extract_keywords(title)
    return all(kw in body for kw in keywords)
```

これを通過しなければ投稿をスキップし、翌サイクルで再生成する。

## 収益が積み上がる仕組み

重要なのは「複利」の視点だ。今日投稿した記事は検索にインデックスされ、1か月後・1年後も読まれ続ける。1日10本を30日続ければ300本の資産になる。アフィリエイト報酬は記事が増えるほど複利的に増加する。

さらにBluesky・はてなブックマークへの自動シェアを加えると、投稿直後の初動流入も確保できる。

```bash
python poster_agent.py --target zenn --share bluesky
```

## 今夜から動かす最小セット

1. `.env` に `ANTHROPIC_API_KEY` と各プラットフォームのトークンを設定する
2. `cron` でオーケストレーターを毎朝7時に起動する
3. 翌朝、ログとZennのダッシュボードで投稿とviewを確認する

最初の1週間はログを毎日確認し、ハングや認証エラーを潰す。安定稼働を確認したら、あとは放置するだけだ。設計図は以上で完成している。次章では各エージェントの内部実装を詳しく解説する。
