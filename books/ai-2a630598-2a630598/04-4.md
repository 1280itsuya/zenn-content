## 収益の三本柱を設計する

自動化システムが稼働しはじめると、次の問いが生まれる。「どこから収益を得るか」。答えはシンプルだ。**アフィリエイト・デジタル商品・サービス販売**の三本柱を同時に走らせること。単一の収益源は脆い。三本が互いを補完し合うとき、はじめて「寝ている間に稼ぐ」が現実になる。

## 柱1　アフィリエイト──記事が自動で紹介料を生む

AIエージェントが毎朝生成するブログ記事の末尾に、A8.netやAmazonアソシエイトの実リンクを自動挿入する。コードは一行で済む。

```python
from src.affiliate_injector import inject_links
body = inject_links(body, niche="節約")
```

ポイントは**ニッチ特化**だ。「節約」「AI副業」「競馬予想」など日替わりで12ニッチをローテーションし、各ニッチに対応するアフィリリンクを`config/affiliate_map.json`で管理する。スパム判定を避けるため、1記事あたりのリンク数は3本以内に制限する。

## 柱2　デジタル商品──一度作れば永遠に売れる

PDFやZIPで配布できるツールキットをGumroadに出品し、Booth・Zennから送客する。たとえば「Claude×MCP導入キット（¥2,480）」は、ZennのAI記事に読了後CTAを1行追加するだけで流入が生まれる。

```markdown
👉 [すぐ使えるMCPスターターキットを入手する](https://gumroad.com/l/xxx)
```

出品自体も自動化できる。Gumroad APIのpresignフローを使えば、新バージョンのZIPをアップロードするだけで商品ページが更新される。

```bash
python src/gumroad_publisher.py --product mcp-kit --version 1.2
```

## 柱3　サービス販売──代行で即金を得る

コンテンツ収益は積み上げに時間がかかる。即金が必要なときは**サービス代行**が最速だ。ココナラ・ランサーズに口コミ返信代行（¥3,000/件）やGEO対策記事作成を出品し、受注後の納品はエージェントが全自動で行う。

```bash
python src/meo_reply_gen.py --client "山田歯科" --reviews reviews.json
```

ポイントは**分母を1にすること**。月5万円の目標なら、¥50,000のサービスを1件取るだけで達成できる。三本柱のなかでサービスだけが「集客ゼロでも成立する」唯一の収益源だ。

## 三本柱を一つのパイプラインに統合する

毎朝7時に走るオーケストレーターが、記事生成→アフィリリンク挿入→Zenn/Qiita投稿→デジタル商品CTAの埋め込みを一括で実行する。サービス受注はプラットフォームが通知してくれるので、エージェントが返信文を生成して下書き保存するだけでよい。三本が並走することで、コンテンツが育つまでのゼロ収益期をサービス収益が下支えし、デジタル商品が複利で積み上がる構造が完成する。
