## AIエージェントは「指示待ち社員」ではない

従来のスクリプト自動化は「手順書通りにしか動けない派遣社員」だ。手順が変わった瞬間に止まる。AIエージェントは違う。目標を与えれば、状況を判断しながら手順を自分で組み立てる。この違いが、24時間稼働する「動く社員」を実現する核心にある。

## 採用基準：エージェントに何をやらせるか決める

まず「業務定義書」を書く。人間を雇うときと同じだ。

```
役職：コンテンツ生成エージェント
業務：毎朝7時にトレンドKWを取得し、1200字のZenn記事を生成・投稿する
成果基準：タイトルと本文のテーマが一致していること
権限：Zenn API・OpenAI APIの呼び出し
```

この定義を `config/agents.json` に落とし込む。曖昧な指示のままエージェントを動かすと、タイトルと無関係な内容を生成する「テーマドリフト」が多発する。

## 入社手続き：環境変数で権限を付与する

エージェントへの「IDカード発行」が `.env` ファイルだ。

```bash
OPENAI_API_KEY=sk-xxxx
ZENN_GITHUB_REPO=your-account/zenn-articles
GIT_TERMINAL_PROMPT=0
```

`GIT_TERMINAL_PROMPT=0` を忘れると、git pushが認証ダイアログを待ち続けて数時間ハングする。自動化あるあるの落とし穴なので必ず設定する。

## 業務マニュアル：プロンプトが「社内規程」になる

エージェントの行動品質はプロンプト設計で決まる。「記事を書いて」ではなく、以下のように構造化する。

```
あなたはZenn向け技術記事ライターです。
テーマ：{topic}
要件：
- タイトルと結論が同じ主張であること
- コード例を1つ以上含めること
- 1200字以内
禁止：テーマと無関係な内容、プレースホルダーリンク
```

「禁止事項」を明示するのがポイントだ。制約がないエージェントは自由に暴走する。

## 出勤管理：タスクスケジューラで定刻出勤させる

WindowsならタスクスケジューラでPythonスクリプトを定刻起動する。

```powershell
schtasks /create /tn "AutoBlog_7AM" /tr "python C:\auto-money\orchestrator\main.py" /sc daily /st 07:00
```

これで毎朝7時にエージェントが「出勤」し、記事生成・投稿・レポート送信まで一気に終わらせる。自分はまだ寝ている。

## 勤怠チェック：ログで稼働を監視する

エージェントは黙って止まることがある。ログを必ず残す。

```python
logging.basicConfig(
    filename="logs/agent.log",
    level=logging.INFO,
    format="%(asctime)s %(levelname)s %(message)s"
)
```

`agent.log` を毎朝確認するだけでいい。エラーが3行以上続いていたら即調査する、というルールを決めておくと管理コストが最小化される。

## まとめ：雇用関係を設計するという発想

AIエージェントは「ツール」ではなく「役割を持った働き手」として設計する。業務定義・権限付与・マニュアル・出勤管理・稼働チェック——この5ステップを踏むだけで、眠っている間も収益を積み上げる自律的な仕組みが完成する。次章では、複数エージェントをチームとして連携させる「マルチエージェント設計」に進む。
