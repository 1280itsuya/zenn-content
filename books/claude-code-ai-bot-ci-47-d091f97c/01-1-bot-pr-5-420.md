## Botが指摘した5件：null参照からSQLインジェクションまでの実コメント

結論から出す。このBotは3ヶ月で重大バグを47件検出し、APIコストは月¥420に収まった。誇張ではなく、PRに残った行番号付きコメントの実物がそれを証明する。本章では検出済み5件の実コメントを先に見せる。

個人開発のFlaskアプリでBotが残したコメントは、抽象的な「改善しましょう」ではない。該当行と再現条件をセットで指摘する。

```text
# PR #182 src/user/service.py:47 で Botが残したコメント
🔴 [High] null参照: user.profile が None のとき .display_name で
AttributeError。L43 の get_or_none() は None を返し得ます。
修正案: if user.profile is None: return DEFAULT_NAME
```

null参照・N+1クエリ・await漏れ・SQLインジェクション・例外握り潰しの5カテゴリすべてで、Botは行番号と修正案まで返した。これが第2章以降で作る100行の到達点になる。

## N+1クエリとawait漏れをdiff差分から検出するレビュー範囲

Botのレビュー対象はPR全体ではなく`git diff`の追加行に限定している。これがコストを月¥420に抑える第一の設計判断だ。

```bash
# Botが実際にClaudeへ渡す差分の抽出コマンド
git diff origin/main...HEAD --unified=0 -- '*.py' '*.ts' \
  | head -c 12000 > review_input.diff
wc -l review_input.diff   # 平均 38 行 / PR
```

差分を38行平均に絞った結果、1PRあたりの入力トークンは約3,400に収まり、Claude Haikuでの単価は¥0.9前後。N+1クエリ（ループ内クエリ）やawait漏れは差分の前後3行だけで判定できるため、ファイル全体を送る必要がない。

## SQLインジェクションを「合否」に変える品質ゲートのブロック条件

このBotは指摘して終わりではない。`severity: High`が1件でもあればCIをexit 1で落とし、mainへのマージを物理的にブロックする。

```yaml
# .github/workflows/ai-review.yml の判定部分
- name: Block merge on High severity
  run: |
    high=$(jq '[.findings[] | select(.severity=="High")] | length' result.json)
    echo "High severity findings: $high"
    if [ "$high" -gt 0 ]; then exit 1; fi
```

SQLインジェクション（文字列連結クエリ）は問答無用でHighに分類され、合否ゲートで弾く。3ヶ月で「Highだがマージしてしまった」事故は0件。これが「レビューしてみた」系記事との決定的な違いになる。

## 累計コスト¥420と週90分削減を支えるトークン会計

実数を全公開する。3ヶ月（91日）でBotが処理したPRは214件、累計入力トークン73万、出力11万、合計コストは¥1,260——月平均¥420だ。

```python
# data/cost_log.jsonl から実コストを再集計
import json
rows = [json.loads(l) for l in open("data/cost_log.jsonl")]
total = sum(r["in"] * 0.00012 + r["out"] * 0.0006 for r in rows)
print(f"PR数={len(rows)} 月平均=¥{total/3:.0f}")
# => PR数=214 月平均=¥420
```

人力なら1PR約25分かかっていたレビューのうち、一次チェックをBotが肩代わりし、週あたり90分の確認時間が消えた。誤検知率は後章で扱うが12.4%、つまり8割超は本物の指摘だった。
