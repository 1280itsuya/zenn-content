## 重大バグ47件の内訳：NullPointer 14件・境界条件 9件・認可漏れ 7件

90日でBotがmainマージをブロックした重大バグは47件。最多はNullPointer系14件、次いで配列/日付の境界条件9件、認可チェック漏れ7件。残り17件はレース条件・例外握り潰し・SQLインジェクション候補に分散した。集計はBotのJSON出力を`jq`で畳むだけで済む。

```bash
# 90日分のレビュー結果(severity=critical)をカテゴリ別に集計
cat reviews/2026-0*/*.json \
  | jq -r 'select(.severity=="critical") | .category' \
  | sort | uniq -c | sort -rn
#  14 null_dereference
#   9 boundary_condition
#   7 missing_authorization
#   5 race_condition
#   ...
```

認可漏れ7件は全てPRの差分行数が200行超のときに混入していた。差分が大きいPRほどBotの貢献度が上がる、という運用上の実感を数値が裏付けた。

## 見逃した3件のパターン：型は通るが意味が壊れる欠陥

Botがマージを通過させ、後に本番で顕在化した見逃しは3件。すべて「型・構文は正しいが業務ロジックの意味が壊れている」欠陥で、Claudeが差分単体からは仕様を推測できなかった。

```python
# 見逃し例: 単位の取り違え(秒とミリ秒)。型はintで通る
def is_expired(issued_at: int, ttl: int) -> bool:
    now = int(time.time())          # 秒
    return now - issued_at > ttl    # ttlはミリ秒で渡されていた → 常にFalse
```

対策はBot側ではなくプロンプトに「単位・通貨・タイムゾーンの一貫性を疑え」を追加し、関連定数を差分外から3ファイルまで添付する方式に変えた。これで類似パターンの再現テストは検出側に転じた。

## 誤検知率11%の正体：42件中の真因はテストコード除外漏れ

レビュー指摘382件のうち誤検知は42件、誤検知率11.0%。内訳を見ると26件がテストコードへの過剰指摘で、`pytest`のフィクスチャをBotが本番コードと誤認していた。除外globを足すだけで誤検知は11%→4%に落ちる。

```yaml
# .github/ai-review.yml : レビュー対象から除外
exclude:
  - "**/tests/**"
  - "**/conftest.py"
  - "**/*.snap"
  - "migrations/**"   # 自動生成は指摘しても直さない
```

残る誤検知の主因は「意図的なTODO実装」への指摘で、これはPR本文の`<!-- ai-skip -->`マーカーで個別黙認できるようにした。

## 月¥420の費用対効果：Haikuで1次・Sonnetで2次の2段構え

90日のClaude API費用は合計¥1,260、月平均¥420。鍵は全PRをいきなりSonnetに投げない2段構成。1次フィルタをHaiku 4.5で回し、`critical`候補が出たPRだけSonnet 4.6で精査する。これで全件Sonnet運用比でトークン費が68%減った。

```python
draft = review(model="claude-haiku-4-5", diff=diff)      # 1次: 全PR
if any(f["severity"] == "critical" for f in draft):
    final = review(model="claude-sonnet-4-6", diff=diff)  # 2次: 候補のみ
    post_blocking_comment(final)
```

レビュー粒度を「関数単位」まで上げると検出は1.2倍に増えたが、トークンは2.9倍に膨らみ月¥420が¥1,210へ。費用対効果のピークは「変更ファイル単位」だった。

## 個人開発で品質ゲートを回し続ける運用チェックリスト

3ヶ月でBotが定着した条件は、検出力より「止まらないこと」。APIエラー時はマージを止めず警告に降格し、月次でカテゴリ集計を見てプロンプトを1行ずつ更新する。次に任せる拡張はテスト自動生成と、見逃し3件の再発監視だ。

```bash
# 月初に回す運用ルーチン(1コマンド)
make review-stats   # カテゴリ集計+誤検知率を再計算

make canary-replay  # 過去の見逃し3件を最新Botで再検査 → 全件検出を確認
```

47件・11%・¥420という実数が示すのは、個人開発でも品質ゲートは「高価な完璧」より「安価で止まらない常駐」が回るという一点に尽きる。
