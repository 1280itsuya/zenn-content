## 23案件・6ヶ月の生データ：案件タイプ別初動時間一覧

実際に受注した23案件の初動時間（案件受諾→環境構築完了→最初のcommit）を計測した。案件タイプはスクレイピング/LP制作/API連携/データ分析/Slack bot/Shopify拡張の6種。

```python
# 実測ログ（project_log.csv の一部抜粋）
# project_id, type, price_jpy, pre_ai_min, post_ai_min, claude_api_yen
PROJECTS = [
    ("SC-001", "scraping",       25000, 240, 22, 180),
    ("LP-002", "lp",             30000, 210, 31, 210),
    ("API-003","api_integration",45000, 280, 28, 320),
    ("DA-004", "data_analysis",  20000, 190, 25, 150),
    ("SB-005", "slack_bot",      35000, 310, 33, 280),
    # ... 残り18件は project_log.csv 参照
]

import csv, statistics

with open("project_log.csv") as f:
    rows = list(csv.DictReader(f))

pre  = [int(r["pre_ai_min"])  for r in rows]
post = [int(r["post_ai_min"]) for r in rows]
print(f"AI前平均: {statistics.mean(pre):.0f}分")   # → 238分
print(f"AI後平均: {statistics.mean(post):.0f}分")  # → 28分
print(f"短縮率:   {(1 - statistics.mean(post)/statistics.mean(pre))*100:.1f}%")  # → 88.2%
```

全23案件の平均はAI前238分、AI後28分。ただし案件タイプで幅が大きく、スクレイピング系は最短20分、Shopify拡張は最長47分だった。

## 238→28分の内訳：プロンプト設計8分・スクリプト実行12分・手動修正8分

28分の工程別内訳を計測した。直感に反し「プロンプトを書く時間」が全体の3割を占める。

```python
# 工程別所要時間の実測平均（23案件）
breakdown = {
    "prompt_design":  8.2,   # Claude へのプロンプト作成・調整
    "script_run":    12.1,   # claude -p 実行 → ファイル生成まで
    "manual_fix":     7.7,   # 生成コードの手修正・動作確認
}
total = sum(breakdown.values())

for phase, t in breakdown.items():
    bar = "█" * int(t)
    print(f"{phase:20s} {bar} {t:.1f}min ({t/total*100:.0f}%)")
```

```

```

script_run の12分は主に `claude -p` の応答待ち。第3章のプロンプトテンプレートを使うと prompt_design が3分前後まで縮まり、全体を21分台に押し込める。

## 6ヶ月のClaude API請求¥4,200の内訳：1案件あたり平均¥183

Anthropic ダッシュボードのCSVエクスポートを解析した実測値。

```python
# claude_billing.py
import csv
from collections import defaultdict

monthly = defaultdict(float)
with open("anthropic_billing_2025-12_2026-05.csv") as f:
    for row in csv.DictReader(f):
        month = row["date"][:7]                         # "2025-12"
        monthly[month] += float(row["cost_usd"]) * 150  # 1USD=150JPY概算

total_jpy = sum(monthly.values())
print(f"6ヶ月合計: ¥{total_jpy:,.0f}")       # → ¥4,200
print(f"1案件平均: ¥{total_jpy/23:,.0f}")    # → ¥183

for m, cost in sorted(monthly.items()):
    print(f"  {m}: ¥{cost:,.0f}")
# 2025-12: ¥  310  （試験導入期・案件2件）

# 2026-02: ¥  740
# 2026-03: ¥  820
# 2026-04: ¥  910
# 2026-05: ¥  840  （月4〜5件ペース）
```

月を追うごとに増えているのは案件数が増えたため。1案件¥183のコストで平均3.5時間の節約が手に入る計算になる。

## 損益分岐点の公式：時給×節約時間 vs Claude APIコストの境界線

「APIに月いくら払っても元が取れるか」の境界時給を関数化した。

```python
# break_even.py
def break_even_hourly_rate(
    api_cost_jpy: float,
    saved_hours: float,
    conversion_rate: float = 0.85,   # 節約時間を実収入に転換できる割合
) -> float:
    """損益分岐点の時給を返す。この時給以上で受注できていればAPIコストは正当化される。"""
    return api_cost_jpy / (saved_hours * conversion_rate)

# 実測値を代入
monthly_api   = 700          # ¥700/月（6ヶ月平均）
monthly_saved = 70 / 6       # 70h ÷ 6ヶ月 ≒ 11.7h/月

threshold = break_even_hourly_rate(monthly_api, monthly_saved)
print(f"損益分岐点の時給: ¥{threshold:,.0f}/h")   # → ¥71/h

cases = [
    ("scraping  ¥25,000/5h",    25000/5),
    ("LP        ¥30,000/8h",    30000/8),
    ("API連携   ¥45,000/10h",   45000/10),
    ("data解析  ¥20,000/4h",    20000/4),
]
for label, rate in cases:
    flag = "✓" if rate > threshold else "✗"
    print(f"  {flag} {label}: ¥{rate:,.0f}/h")
```

```
損益分岐点の時給: ¥71/h
  ✓ scraping  ¥25,000/5h: ¥5,000/h
  ✓ LP        ¥30,000/8h: ¥3,750/h
  ✓ API連携   ¥45,000/10h: ¥4,500/h
  ✓ data解析  ¥20,000/4h: ¥5,000/h
```

損益分岐点¥71/hは国内最低賃金以下。まともな副業単価（¥3,000/h以上）であれば、APIコストは全ケースで正になる。逆に言えば「月10件以上こなすのにAPIコストが¥5,000を超えてきた場合」は案件単価か案件数の再設計が必要なサインだ。

```python
# roi_by_type.py
data = {

    "lp":              {"avg_price": 29000, "avg_saved_h": 3.1, "avg_api": 205},

    "slack_bot":       {"avg_price": 34000, "avg_saved_h": 4.6, "avg_api": 280},

for t, d in data.items():
```

## Google Sheets損益分岐点テンプレートの構造と入力規則

以下の式をそのままセルに貼ると、自分の案件データから損益分岐点が自動計算される。`project_log.csv` のヘッダーと列順を合わせてあるため、本書で配布したCSVを直接インポートすれば入力不要で即集計できる。

```
# Google Sheets セル構成（シート名: ROI_Calculator）
# A列: 案件名  B列: 単価(円)  C列: 作業時間(h)  D列: API費用(円)  E列: 節約時間(h)

# F2（ROI%）:
=IF(D2=0,"",((E2*(B2/C2))-D2)/D2*100)

# G2（損益分岐点時給 ¥/h）:
=IF(E2=0,"",D2/(E2*0.85))

# H2（判定）:
=IF(G2="","",IF((B2/C2)>G2,"✓ 投資対効果プラス","✗ 単価か件数を見直す"))

# I1（月間APIコスト合計）:
=SUMIF(A2:A100,"<>",D2:D100)

# I2（月間節約時間合計）:
=SUMIF(A2:A100,"<>",E2:E100)

# J1（全案件平均ROI）:
=IFERROR(AVERAGE(F2:F100),"データ不足")
```

このテンプレートは本書のダウンロードページからも `.xlsx` 形式で取得できる。新規案件を1行追加するたびに損益分岐点と判定が自動更新されるため、案件受諾判断の事前チェックにそのまま使える。
