## 必須PRのみ起動：opened/synchronizeとdraft除外でトリガー回数を削減

毎PR・毎pushで起動した初期構成は月$60に膨らんだ。`on.pull_request`の`types`を絞り、`draft == true`を弾くだけで起動回数が約6割減り、月$60→$22まで落ちる。

```yaml
on:
  pull_request:
    types: [opened, synchronize, ready_for_review]

jobs:
  review:
    if: github.event.pull_request.draft == false
    runs-on: ubuntu-latest
```

`reopened`を外すのが要点で、レビュー済みPRの再オープンで二重課金していた分が消える。

## labelでopt-inとprompt cachingでキャッシュヒット率71%

全PR自動ではなく`ai-review`ラベル付与時のみ起動させ、対象を実装PRに限定する。さらにsystemプロンプトを`cache_control`で固定すると、3週間の実測でキャッシュヒット率71%・入力トークン課金が1/3になり、月$22→$8まで圧縮できた。

```yaml
    if: >
      github.event.pull_request.draft == false &&
      contains(github.event.pull_request.labels.*.name, 'ai-review')
```

```python
import anthropic
client = anthropic.Anthropic()
resp = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=2000,
    system=[{
        "type": "text",
        "text": REVIEW_RULES,          # 固定の長文レビュー規約
        "cache_control": {"type": "ephemeral"},
    }],
    messages=[{"role": "user", "content": diff_text}],
)
u = resp.usage
print(u.cache_read_input_tokens, u.input_tokens)
```

## Actions実行ログからトークン課金を日次集計しSummaryへ表示

`usage`を1行JSONで吐き、`$GITHUB_STEP_SUMMARY`へ表へ整形する。Sonnet 4.6の$3/$15(1Mトークン)単価でPRごとの実コストが可視化され、$8/月の内訳をその場で追える。

```python
import json, os
cr, ic, oc = u.cache_read_input_tokens, u.input_tokens, u.output_tokens
cost = (ic*3 + cr*0.3 + oc*15) / 1_000_000   # cache readは1/10単価
with open(os.environ["GITHUB_STEP_SUMMARY"], "a") as f:
    f.write(f"| PR | in | cache | out | $ |\n|--|--|--|--|--|\n")
    f.write(f"| #{os.environ['PR']} | {ic} | {cr} | {oc} | {cost:.4f} |\n")
print(json.dumps({"in": ic, "cache": cr, "out": oc, "usd": cost}))
```

## OIDC寄りSecrets運用でAPIキー漏洩を防ぐ最小権限設定

`ANTHROPIC_API_KEY`をfork PRのworkflowへ渡さないため`pull_request_target`を使わず、`permissions`を最小化する。Secretsはjob単位でのみ展開し、`pull_request`イベントのforkにはSecretsが渡らないGitHub仕様を防御線にする。

```yaml
permissions:
  contents: read
  pull-requests: write   # コメント投稿のみ許可
env:
  ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

ログ流出対策に`echo "::add-mask::$ANTHROPIC_API_KEY"`を初手で実行し、誤printでもマスクされるようにする。

## continue-on-errorで障害時もCIを落とさずスキップする設計

Anthropic API側の429やタイムアウトでマージ全体を止めないため、レビューstepを`continue-on-error: true`にし、失敗時はSkipコメントだけGitHub上へ残す。3週間でAPI起因の失敗は4件あったが、いずれもマージはブロックせず処理できた。

```yaml
      - name: AI review
        id: ai
        continue-on-error: true
        run: python review.py

      - name: Notify skip
        if: steps.ai.outcome == 'failure'
        run: gh pr comment "$PR" --body "⚠️ AIレビューはAPI障害でスキップ（CIは継続）"
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          PR: ${{ github.event.pull_request.number }}
```

`outcome`で分岐するとjob自体はgreenを保つため、必須チェックに組み込んでもデプロイ列を止めない。この5点で月$8・レビュー漏れゼロの本番運用が成立する。
