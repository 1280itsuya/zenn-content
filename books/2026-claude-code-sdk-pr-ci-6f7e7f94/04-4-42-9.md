## 誤検知42%の内訳：style指摘・テスト過敏・生成ファイル誤爆

3週間で記録した指摘100件のうち、ノイズ判定は42件。内訳をログから集計すると次の3カテゴリで全体の88%を占めた。

```python
noise = {"style/format": 19, "test_file_overreact": 12, "generated_file": 6, "other": 5}
total = 100
print(f"誤検知率: {sum(noise.values())/total:.0%}")  # => 42%
print(f"上位3カテゴリ占有: {(19+12+6)/sum(noise.values()):.0%}")  # => 88%
```

この3カテゴリを潰せば42%は理論上5%まで落ちる。実測では9%まで下がった。残り4%は仕様起因の判断揺れで、プロンプトでは消せなかった。

## systemプロンプトに追加した除外3行の差分

`claude.md` のレビュー指示に、以下のdiffをそのまま追加した。styleとテスト過敏の31件がこれで消えた。

```diff
 You are a PR reviewer. Report only actionable defects.
+- Do NOT comment on style, formatting, naming, or import order.
+  Those are handled by Biome/Ruff in a separate job.
+- Test files (*.test.ts, test_*.py): comment ONLY when the
+  corresponding production logic changed in the same diff.
+- Treat any code as correct unless you can name a concrete
+  failing input or runtime path.
```

「concrete failing input を挙げられないなら指摘するな」の1行が効く。根拠なき"念のため指摘"が17件→2件に減った。

## severity自己評価でlow以下を捨てるフィルタ

Claude に severity を high/medium/low で自己採点させ、low を機械的に破棄する。生指摘の3割がlowで、ここがノイズの温床だった。

```python
import json

def filter_findings(raw: str, keep={"high", "medium"}) -> list:
    items = json.loads(raw)["findings"]
    return [f for f in items if f.get("severity", "low") in keep]

# 出力スキーマをプロンプトで強制
SCHEMA = '各指摘を {"severity":"high|medium|low","file":..,"reason":..} の配列で返す'
```

low破棄で誤検知は9件減り、同時にPRコメント数が平均14→6件になりレビュアー疲労も下がった。

## paths-ignoreで生成ファイルをCI段階で除外

プロンプト前に、そもそも生成物をdiffに渡さない。`pnpm-lock.yaml` や `dist/` を読ませていたのが6件の誤爆原因だった。

```yaml
on:
  pull_request:
    paths-ignore:
      - '**/*.lock'
      - 'pnpm-lock.yaml'
      - 'dist/**'
      - '**/__generated__/**'
      - '**/*.snap'
```

トークンも節約できる。`pnpm-lock.yaml` 1ファイルで約8,000トークン課金していたのが、除外後はゼロになった。

## レビュー漏れ7件の真因：max_turns不足

誤検知を下げる一方、見逃しも7件発生した。全PRのログを追うと、5件が同一原因——`max_turns` が4で、大きいdiffの途中でClaudeが打ち切られていた。

```python
# 漏れたPRはすべて変更ファイル数 >= 9 だった
leaked = [p for p in prs if p["status"] == "missed"]
print([p["changed_files"] for p in leaked])  # [11, 14, 9, 12, 9, 3, 2]
# max_turns を 4 -> 8 に。9ファイル以上のPRで漏れ5->0
```

`max_turns` を8へ引き上げ、9ファイル以上のPRはレビュー前に論理単位で分割する運用に変更。漏れは月7件→1件まで減った。誤検知9%・漏れ1件が、3週間運用で到達した実数値である。
