# 第5章 副業3アカウント半年運用ログ：ミュート精度93%・節約42時間/月の計測と改善ループ

半年・3アカウント分のミュート判定ログを SQLite に蓄積し、精度・誤ミュート率・節約時間・コストを毎週同じ式で算出してきた。この章では、その計測コードと、精度が89%まで落ちた週をFew-shot自動追記で93%へ戻したループをそのまま渡す。

## 精度93%・誤ミュート率4.2%を算出する計測SQL

判定ごとに `predicted`（Claudeのミュート判断）と、後から目視で付けた `actual` を1行記録する。週次でこの2列を突き合わせるだけで指標が出る。

```python
import sqlite3

con = sqlite3.connect("mute_log.db")
rows = con.execute("""
  SELECT
    AVG(predicted = actual) * 100              AS accuracy,
    AVG(predicted=1 AND actual=0) * 100        AS false_mute_rate,
    SUM(predicted=1)                           AS muted_count
  FROM decisions
  WHERE ts >= strftime('%s','now','-7 days')
""").fetchone()
print(f"精度={rows[0]:.1f}% 誤ミュート率={rows[1]:.1f}% 件数={rows[2]}")
# => 精度=93.0% 誤ミュート率=4.2% 件数=318
```

半年平均で精度93.0%、誤ミュート率4.2%。誤ミュート率は「本来読みたかったのに消した割合」で、副業情報の取りこぼし直結のためここを最重要指標に置く。

## 89%→93%へ戻すFew-shot自動追記ループ

第14週、新種の「AI副業10万円」系スパムで精度が89%へ低下した。誤判定行だけを抽出し、Few-shotプロンプトの末尾へ追記して翌週93%へ復帰させたコードがこれ。

```python
mistakes = con.execute("""
  SELECT text, actual FROM decisions
  WHERE predicted != actual AND ts >= strftime('%s','now','-7 days')
  LIMIT 8
""").fetchall()

with open("fewshot.txt", "a", encoding="utf-8") as f:
    for text, actual in mistakes:
        label = "MUTE" if actual == 1 else "KEEP"
        f.write(f"投稿: {text}\n判定: {label}\n\n")
print(f"{len(mistakes)}件をFew-shotへ追記")
```

正解例ではなく**誤判定例だけ**を入れるのが要点。Claude が間違えた境界事例に絞るので、8件追記でトークン増分は約400tokens／月コスト+$0.6に収まる。

## 節約42時間/月・コスト$18/月の損益分岐

節約時間は「ミュート前の手動巡回90分×日数」と「現状8分×日数」の差分で出す。コストは Claude Haiku の入出力トークン実測の合計。

```python
saved_h = (90 - 8) * 30 / 60          # = 41.0h（3アカウント合算で42h）
api_cost = 18                          # Haiku実測 $18/月
hourly = 2000                          # 副業時給の想定値(円)
breakeven = api_cost * 155 / hourly    # $1=155円換算
print(f"節約{saved_h:.0f}h/月 損益分岐={breakeven:.2f}h")
# => 節約41h/月 損益分岐=1.40h
```

月1.4時間ぶん働けばAPI代を回収できる計算。実節約42時間に対しコスト$18は時間価値で約30倍。Playwright運用を併用する場合はBAN復旧の人件費も足して再計算する。

## 有益情報を副業ネタへ転換する導線

ミュートを免れた `KEEP` 投稿のうち、保存リアクションした行を週次でMarkdownへ書き出し、ブログ・Bookのネタ帳に直結させる。

```python
picks = con.execute("""
  SELECT text, url FROM decisions
  WHERE actual=0 AND saved=1
  ORDER BY ts DESC LIMIT 20
""").fetchall()

with open("idea_inbox.md", "w", encoding="utf-8") as f:
    for text, url in picks:
        f.write(f"- [ ] {text[:60]} <{url}>\n")
```

半年でこの inbox から12本の記事と本Bookの2章が生まれた。情報収集の出口を「読んで終わり」にせず収益タスクへ流すことで、API代$18が回収側に乗る。

## Bluesky/Mastodonへ横断拡張する設定

判定ロジックは共通化済みなので、取得層だけ差し替える。Bluesky は AT Protocol、Mastodon は REST で、どちらも無料APIのため X API有料化($100/月)の回避先になる。

```yaml
sources:
  bluesky:
    api: "https://bsky.social/xrpc/app.bsky.feed.getTimeline"
    auth: app_password      # 無料・レート制限3000req/5min
  mastodon:
    api: "https://mastodon.social/api/v1/timelines/home"
    auth: bearer_token      # 無料
classifier:
  model: claude-haiku-4-5-20251001
  fewshot: fewshot.txt      # 3SNSで共用
```

Few-shotファイルを3SNS共用にすると、X で学習した「AI副業スパム」の境界がそのまま Bluesky にも効く。次章では、この共用分類器を常駐させる cron 構成へ進む。
