## 半年で積み上がった手動ミュート340件の限界

X(旧Twitter)を半年運用した結果、手動で登録したミュートワードは340件、ブロックは120件に達した。設定画面からエクスポートすると、運用の限界が数値で見える。

```bash
# muted_keywords.json は X の設定 > プライバシー からエクスポート
jq '.mutedKeywords | length' muted_keywords.json   # => 340
jq '.blockedAccounts | length' blocked.json         # => 120
```

週あたり12件のペースで増え続け、登録作業だけで月20時間を消費していた。手動運用はスケールしない。

## ノイズ率を目視ラベルで「62%」と定義する

主観を排すため、タイムライン直近500件にプロモ/バズ便乗/スパムRTの3ラベルを目視付与し、ノイズ率を算出した。

```python
import json
labeled = json.load(open("timeline_500.json"))  # {"text":..., "label":"noise"|"ok"}
noise = sum(1 for t in labeled if t["label"] == "noise")
print(f"noise rate = {noise / len(labeled):.0%}")   # => 62%
```

500件中310件がノイズ、つまり62%。フォロー中タイムラインの3分の2が情報収集の妨げになっていた。

## Claude Haikuで62%→8%へ落とす最小分類器

置き換えの中核がこの20行。`claude-haiku-4-5` は1万件あたり¥48で、340件のルールを書くより安く正確に判定する。

```python
import anthropic
client = anthropic.Anthropic()

def classify(text: str) -> str:
    r = client.messages.create(
        model="claude-haiku-4-5-20251001",
        max_tokens=5,
        system="投稿を noise か ok の一語で分類。"
               "noise=プロモ/バズ便乗/スパムRT。",
        messages=[{"role": "user", "content": text}],
    )
    return r.content[0].text.strip()

print(classify("【拡散希望】フォロー&RTでiPhone15が当たる🎁"))  # => noise
```

同じ500件に通した結果、ノイズ率は8%まで低下した。

## 取得→分類→ミュート→収集サマリの全体像

本書で動かすパイプラインは4部品。各章がどの部品を担うかをそのまま地図にした。

```yaml
pipeline:
  - fetch:    # 第2章: X API 無料枠でタイムライン取得 (上限実測は2章で開示)
      out: timeline.jsonl
  - classify: # 第3章: 上の classify() を並列化、RPS/失敗ログ付き
      model: claude-haiku-4-5-20251001
  - mute:     # 第4章: noise 判定の発信元を API で自動ミュート反映
  - report:   # 第5章: 収集サマリを毎朝 cron で生成
      cron: "0 7 * * *"
```

## 実分類サンプルと第2章への接続

無料章の手土産として、上の `classify()` が返した実結果を置く。これがそのまま第2章以降のcron常駐コードの入口になる。

```text
"新作スニーカー、駅前で先行販売中です"          -> ok
"いいねした人全員フォローします!相互100%"      -> noise
"地震速報 震度4 各自確認を"                      -> ok
"このツール月収100万の再現性ヤバい→ プロフへ"   -> noise
```

精度の検証ログ、X API無料枠の実測上限、失敗リトライ設計は第2章から。20行の分類器を「月20時間を返すcron」に育てる全コードを次章で渡す。
