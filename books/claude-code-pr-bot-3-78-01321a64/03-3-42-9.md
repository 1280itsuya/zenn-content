## 誤検知42%の正体：v1プロンプトが量産した「好みの指摘」

同一20PRで計測した初期Botの誤検知率は42%だった。原因は重大度の概念がなく、変数名やインデントの好みまで同列で吐き出していた点にある。v1のシステムプロンプトは次の1文しか持っていなかった。

```text
あなたはコードレビュアーです。このdiffの問題点をすべて列挙してください。
```

「すべて列挙」が命令されると、Claudeは20PRで合計211件を出力し、そのうち89件が好みベースの主観指摘だった（211件中42.2%）。レビュイーはBotコメントを既読スルーするようになり、解決率は11%まで落ちた。

## blocker/warning/nitの3段階分類で23%へ：v2の差分

v2では重大度を3段階に強制した。nitを別枠に隔離するだけで、無視される指摘がコメント本体から消える。

```diff
- このdiffの問題点をすべて列挙してください。
+ 各指摘に severity を必ず付与せよ。
+   blocker: バグ・セキュリティ・データ破壊のいずれか
+   warning: 動くが障害につながる設計上の懸念
+   nit:     好み・スタイル（出力末尾にまとめ、本体に混ぜない）
+ severity を判定できない指摘は出力しない。
```

この改修で誤検知率は42%→23%に低下した。内訳はblocker/warningのみを分母にすると、20PRで誤検知14件/総61件＝23.0%。nitを本体から外したことでレビュイーの解決率は11%→38%へ回復した。

## プロジェクト固有ルーブリックの注入：23%→9%の決定打

残る23%の大半は「このリポでは許容している慣習」をBotが知らずに叩く誤検知だった。規約をルーブリック化してプロンプトへ注入する。

```yaml
# rubric.yaml — リポ固有の許容/禁止を明文化
allow:
  - "テストコードでの any 型使用"
  - "scripts/ 配下の console.log"
deny:
  - "本番コードでの非null断定演算子 (!)"
  - "await なしの Promise 呼び出し"
```

```python
rubric = open("rubric.yaml", encoding="utf-8").read()
system = f"{BASE_PROMPT}\n\n# このリポの判定基準\n{rubric}\n" \
         "allow に該当する事象は指摘してはならない。"
```

allowリストを与えた後の再計測で誤検知は61件中9件＝9.0%まで低下した。許容慣習への誤指摘8件がそのまま消えたのが効いている。

## 根拠行の強制引用：人間が3秒で検証できる出力フォーマット

誤検知を9%まで下げても、根拠が無ければ検証コストが残る。該当行の引用を必須にし、人間が即座に真偽判定できる形へ固定した。

```python
OUTPUT_FORMAT = """
出力は各指摘ごとに以下のJSONのみ:
{ "severity": "blocker|warning|nit",
  "file": "path", "line": 123,
  "quote": "該当行をdiffから逐語コピー",
  "reason": "1文で根拠", "fix": "修正案コード" }
quote が diff 内に存在しない指摘は出力禁止。
"""
```

quote必須化は幻覚抑制も兼ねる。存在しない行を引用できないため、diffに無い箇所への指摘がv2比で6件減った。

## 抑制プロンプトとコスト：1PR¥4.7を維持したまま9%へ

断定できない指摘を黙らせる1行が、誤検知率と同時にトークン消費も削った。

```python
CONFIDENCE_GUARD = "確信度70%未満の指摘は出力するな。曖昧な懸念の列挙は禁止。"

# claude-sonnet 単価で1PR平均コストを実測
in_tok, out_tok = 18400, 1250          # 20PR平均
cost = (in_tok/1e6)*0.45 + (out_tok/1e6)*2.25  # JPY換算済単価
print(round(cost, 2))   # -> 4.7
```

出力量が減ったぶん `out_tok` が平均1,980→1,250へ下がり、1PRあたり¥4.7を維持した。誤検知42%→23%→9%の3段改修は、プロンプト追記のみでモデル変更ゼロ・追加コストゼロで完了している。次章ではこのBotをGitHub Actions常駐させる `.github/workflows` 全文を渡す。
