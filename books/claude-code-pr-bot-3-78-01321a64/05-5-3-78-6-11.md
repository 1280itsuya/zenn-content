結論から言うと、3週間でClaude Code製Botは指摘78件を出し、うち54件が採用、実バグ6件を検出し、レビュー手戻りを約11時間削減した。コストは1PRあたり¥4.7。以下、全数値と見逃し3パターンを公開する。

## 3週間41PRの自動指摘78件・採用54件を集計するスクリプト

個人開発2リポジトリ（TypeScript API / Python バッチ）で3週間運用した生データをJSONで保持し、採用率を算出した。

```python
runs = [
    {"pr": 41, "raised": 78, "accepted": 54, "real_bugs": 6},
]
r = runs[0]
print(f"採用率 {r['accepted']/r['raised']:.0%}")   # 69%
print(f"実バグ率 {r['real_bugs']/r['raised']:.0%}")  # 8%
```

採用率69%。第3章で誤検知42%だったプロンプトを改修し、最終週は誤検知9%まで下げた。残る31%は様式上の好みのため非採用とした。

## 手戻り11時間削減をClaude Sonnet単価で回収するコスト計算

Botコストは入力平均8.2k・出力1.4kトークン、Claude Sonnet単価で1PR ¥4.7。41PRで合計¥193。

```python
cost_per_pr = 4.7
prs = 41
saved_hours = 11.0
hourly = 4000  # 自分の時給想定
bot_cost = cost_per_pr * prs
value = saved_hours * hourly
print(f"投資 ¥{bot_cost:.0f} / 回収 ¥{value:.0f} / 倍率 {value/bot_cost:.0f}倍")
# 投資 ¥193 / 回収 ¥44000 / 倍率 228倍
```

¥193の投資で約11時間（¥44,000相当）を回収。228倍の費用対効果になる。

## Botが見逃した型エラーなど失敗3パターンの再現diff

包み隠さず報告する。見逃しは暗黙any・跨ぎファイルの契約破壊・テスト未更新の3種だった。

```diff
- function parse(x: string): Item {
+ function parse(x): Item {        // 暗黙any、Botは指摘せず本番でundefined

```

単一diffしか見ないため、別ファイルの呼び出し側が壊れる契約破壊は文脈不足で素通りした。テスト未更新も差分外のため検出できなかった。

## blocker指摘でマージを止める品質ゲートのYAML

severityが`blocker`の指摘が1件でもあればCIを失敗させ、マージをブロックする運用へ拡張した。

```yaml
- name: review-gate
  run: |
    blockers=$(jq '[.[]|select(.severity=="blocker")]|length' review.json)
    if [ "$blockers" -gt 0 ]; then
      echo "::error::blocker $blockers 件 — マージ不可"; exit 1
    fi
```

## 自リポ常駐前のルーブリック調整チェックリスト3項目

クロスレビュー（Sonnetで一次、見逃し補完にもう1モデル）を回す前に、下記で誤検知の出やすい規約を除外する。

```bash
# 自リポ用にチェックを取捨選択
cat > rubric.yml <<'EOF'
enabled:  [null_safety, contract_break, missing_test]
disabled: [style_preference]   # 誤検知42%の主因だったため停止
EOF
```

このルーブリックを第4章のプロンプトに差し込めば、明日から指摘の約8割が採用される常駐Botになる。
