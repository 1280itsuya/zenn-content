## コスト計算とトークン最適化：月1万円以内で量産する設計

### APIコストの実態把握

アフィリ記事を量産する前に、コスト試算が欠かせない。Claude APIの料金は以下のとおりだ（$1≈¥150換算）。

| モデル | 入力 | 出力 |
|--------|------|------|
| Haiku 4.5 | ¥0.15/千トークン | ¥0.75/千トークン |
| Sonnet 4.6 | ¥0.45/千トークン | ¥2.25/千トークン |

プロンプト500・出力1,500トークンの1記事で試算すると：

- **Haiku：** (0.5×0.15) + (1.5×0.75) = **¥1.20/記事**
- **Sonnet：** (0.5×0.45) + (1.5×2.25) = **¥3.60/記事**

月1万円の予算では Haiku なら約**8,300記事**、Sonnet なら約**2,800記事**生成できる。

### プロンプトキャッシュで入力コスト90%削減

量産パイプラインの最大の無駄は「同じシステムプロンプトを毎回送信すること」だ。共通指示が2,000トークンあり、100記事生成するだけで20万トークンを捨てることになる。

`cache_control` を設定すると、2回目以降の読み取りコストが**通常の10%**に圧縮される。

```python
import anthropic

client = anthropic.Anthropic()

SYSTEM_PROMPT = """あなたは優秀なアフィリエイト記事ライターです。
[詳細な指示2,000字が続く...]
"""

def generate_article(keyword: str) -> str:
    response = client.messages.create(
        model="claude-haiku-4-5",
        max_tokens=2048,
        system=[{
            "type": "text",
            "text": SYSTEM_PROMPT,
            "cache_control": {"type": "ephemeral"}  # ← キャッシュ有効化
        }],
        messages=[{"role": "user", "content": f"キーワード：{keyword}"}]
    )
    usage = response.usage
    print(f"キャッシュ読込: {usage.cache_read_input_tokens} / 通常入力: {usage.input_tokens}")
    return response.content[0].text
```

**落とし穴：** システムプロンプト内にタイムスタンプや記事連番を含めると毎回キャッシュが無効化される。可変部分は必ずユーザーメッセージ側に移動すること。`cache_read_input_tokens` がゼロのままなら無効化が起きている。

### モデル使い分けで品質とコストを両立

全記事をSonnetで生成する必要はない。キーワードの重要度でモデルをルーティングするだけで大幅に節約できる。

```python
def generate_with_routing(keyword: str, priority: str) -> str:
    model = "claude-sonnet-4-6" if priority == "high" else "claude-haiku-4-5"
    response = client.messages.create(
        model=model,
        max_tokens=2048,
        system=[{"type": "text", "text": SYSTEM_PROMPT,
                 "cache_control": {"type": "ephemeral"}}],
        messages=[{"role": "user", "content": keyword}]
    )
    return response.content[0].text

# 主力キーワード20本 → Sonnet、ロングテール200本 → Haiku
for kw in high_priority_keywords:
    generate_with_routing(kw, "high")   # ¥3.60/記事
for kw in longtail_keywords:
    generate_with_routing(kw, "low")    # ¥1.20/記事
```

**実測コスト：** Sonnet 20記事(¥72) + Haiku 200記事(¥240) = **月¥312**。予算の3%で220記事を量産できる。

### Batch APIでさらに半額

締め切りのない記事はBatch APIに回すと通常料金の**50%引き**になる。Haikuと組み合わせると1記事あたり¥0.60まで下がる。

```python
from anthropic.types.message_create_params import MessageCreateParamsNonStreaming
from anthropic.types.messages.batch_create_params import Request

requests = [
    Request(
        custom_id=f"article-{i}",
        params=MessageCreateParamsNonStreaming(
            model="claude-haiku-4-5",
            max_tokens=2048,
            messages=[{"role": "user", "content": kw}]
        )
    )
    for i, kw in enumerate(longtail_keywords)
]

batch = client.messages.batches.create(requests=requests)
# 数時間後に client.messages.batches.results(batch.id) で結果取得
```

### 月コスト比較まとめ

| 構成 | 月200記事のコスト |
|------|-----------------|
| Sonnet のみ | ¥720 |
| Haiku のみ | ¥240 |
| Haiku + Batch | ¥120 |
| Haiku + Batch + キャッシュ | ¥60〜100 |

キャッシュ・Batch・モデルルーティングを組み合わせれば、月1万円の予算で**数千記事**の量産は十分現実的だ。「どのキーワードにどの品質が必要か」を先に定義し、それに合ったモデルを割り当てることが設計の肝になる。全記事をSonnetで生成するのはコストの無駄遣いにほかならない。
