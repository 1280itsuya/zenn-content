## 月5万円へのロードマップ――アフィリ・Gumroad・記事代行への拡張

自動生成パイプラインが動き始めると、次の問いが生まれる。「記事は量産できた。では、どこから収益が入るのか」。生成基盤だけでは収益ゼロのままで終わる。本章では、同じパイプラインに**アフィリリンク挿入・Gumroad教材販売・記事代行**の3チャネルを接続し、月5万円に到達するまでの構成と注意点を整理する。

---

## チャネル1：アフィリリンク自動挿入

A8.net やAmazon アソシエイトのリンクを、生成プロンプトの末尾に渡すだけで記事内に埋め込める。

```python
AFFILIATE_RULES = {
    "Python 開発": "https://amzn.to/xxxxx",
    "クラウド入門": "https://a8.net/xxxxx",
}

def inject_affiliate(body: str, rules: dict) -> str:
    for keyword, url in rules.items():
        body = body.replace(keyword, f"[{keyword}]({url})", 1)
    return body
```

**落とし穴：単語分断バグ**。`replace` は最初のヒットだけを置換する設計にしないと、同じキーワードが記事中に複数登場した際にリンクが重複し、プラットフォームのスパム判定を受ける。`count=1` を明示するか、正規表現で先頭1件のみ置換する実装にすること。

月次試算：Zenn/Qiita 経由で月間3,000 PV、クリック率1%、商品単価3,000円、報酬率5% → **¥4,500**。単体では小さいが、他チャネルとの合算で積み上げる。

---

## チャネル2：Gumroad 教材の自動出品

「記事 → PDF 教材化 → Gumroad 出品」を一本のスクリプトに収めると、新しい記事が生まれるたびに商品が増える。

```python
import requests

def publish_to_gumroad(name: str, pdf_path: str, price_cents: int):
    with open(pdf_path, "rb") as f:
        r = requests.post(
            "https://api.gumroad.com/v2/products",
            data={"name": name, "price": price_cents},
            headers={"Authorization": f"Bearer {GUMROAD_TOKEN}"},
        )
    product_id = r.json()["product"]["id"]
    # presign フローでファイルを添付（別途実装）
    return product_id
```

**落とし穴：支払い方法未接続**。Gumroad はアカウントに銀行口座またはPayPal を登録しないと、APIで商品を作っても購入完了時に入金が詰まる。スクリプトを動かす前に必ず Gumroad ダッシュボードで支払い先を設定する。

PDF 1冊¥980、月5冊販売で**¥4,900**。Zenn の技術記事からGumroad へ誘導するCTA を末尾に固定すると転換率が上がる。

## チャネル3：記事代行（クライアントワーク）

最もレバレッジが高いのが記事代行だ。クライアントが持つ既存流入に乗れるため、集客ゼロの状態でも収益が発生する。

1. **ポートフォリオ記事を3本用意**する（Zenn に公開済みの実績で代替可）
2. ランサーズ・ランサーズに「AI × SEO 記事代行」として出品
3. 受注後、クライアントのキーワードをプロンプトに差し込み、生成 → 校正 → 納品

```bash
# 納品フォルダ構成例
client_work/
  client_a/
    brief.md       # クライアントから受け取るKW・ターゲット読者
    draft.md       # Claude 生成の下書き
    final.md       # 人間校正済み
    invoice.md     # 請求書テンプレ
```

**落とし穴：生成そのままは納品不可**。タイトルと本文のテーマが食い違う「ドリフト」が発生しやすい。最低限、本文の冒頭200字を読み返しタイトルとの一致を確認するレビューステップを挟む。

単価¥8,000 × 5本で**¥40,000**。記事代行1本で、アフィリとGumroad の合計を超える。

## コスト試算まとめ

| チャネル | 月間目安 | 主な変数 |
|----------|---------|---------|
| アフィリリンク | ¥3,000〜¥8,000 | PV・クリック率 |
| Gumroad 教材 | ¥3,000〜¥10,000 | 商品数・誘導導線 |
| 記事代行 | ¥20,000〜¥50,000 | 受注本数・単価 |

**API コスト**：Claude API で記事1本あたり約¥3〜¥8（claude-sonnet-4-6、800〜1,500 tokens/記事）。月100本生成しても¥800 未満に収まる。

3チャネルを同時に走らせると、低PV 期でも合計¥26,000〜¥68,000 のレンジに入る。月5万円は記事代行が主力になる前提の目標値であり、最初の受注が取れた時点で現実圏に入る。
