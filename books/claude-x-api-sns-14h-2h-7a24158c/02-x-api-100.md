## Free 月100件 vs Basic $200 の取得上限を表で直視する

X API v2 の Read 上限は 2025 年の改定で初心者を最初に止める壁になった。Free は月 100 投稿の取得、Basic は月 $200 で 15,000 投稿。週 14h かけていた手動収集を埋めるには Free では 3 日で枯れる。

| プラン | 月額 | Tweet 取得上限/月 | 実質コスト/1万件 |
|---|---|---|---|
| Free | $0 | 100 | 取得不能 |
| Basic | $200 | 15,000 | $133 |
| Pro | $5,000 | 1,000,000 | $50 |

```python
NEEDED_PER_MONTH = 12000  # 監視リスト40アカウント × 日10投稿 × 30日
FREE_CAP = 100
print(f"Free枠で足りる割合: {FREE_CAP / NEEDED_PER_MONTH:.1%}")  # 0.8%
```

0.8% しか賄えない。ここで全文 API 取得を捨てる判断をした。

## 全文 API を捨て Lists タイムラインを Playwright で読む

Basic に課金せず、ログイン済み Cookie で Lists タイムラインの HTML を読む方式へ切り替えた。リスト 1 本に監視対象 40 アカウントを束ね、API コール 0 で同等の情報量を得る。

```python
from playwright.sync_api import sync_playwright

LIST_URL = "https://x.com/i/lists/1234567890123456789"

def fetch_list_html():
    with sync_playwright() as p:
        ctx = p.chromium.launch_persistent_context(
            user_data_dir="./.x_profile", headless=True)
        page = ctx.new_page()
        page.goto(LIST_URL, wait_until="networkidle")
        page.wait_for_selector("article[data-testid='tweet']")
        articles = page.query_selector_all("article[data-testid='tweet']")
        return [a.inner_text() for a in articles]
```

## 429 を 3 日で 17 回踏んだ後の指数バックオフ

初日にループ間隔を詰めすぎ、`429 Too Many Requests` を 3 日で 17 回踏んだ。アカウント凍結の前兆だったため、1 リクエスト/8 秒の固定スロットルへ落とし、失敗時のみ指数バックオフを重ねた。

```python
import time, random

def throttled_get(page, url, base=8.0):
    for attempt in range(5):
        page.goto(url, wait_until="networkidle")
        if "Rate limit" not in page.content():
            time.sleep(base + random.uniform(0, 2))  # 固定8s+ゆらぎ
            return page.content()
        wait = base * (2 ** attempt)  # 8,16,32,64,128s
        time.sleep(wait)
    raise RuntimeError("429 が5回連続。当日の収集を中止")
```

このスロットル投入後、429 はゼロになった。

## Cookie 認証を storage_state で永続化する

毎回ログインすると 2FA とパスワード再入力で BAN フラグが立つ。`storage_state` を 1 度だけ保存し、以降は Cookie を読み込むだけにする。再ログイン頻度を 30 日 1 回まで下げた。

```python
# 初回のみ手動ログインして保存
def save_state():
    with sync_playwright() as p:
        b = p.chromium.launch(headless=False)
        page = b.new_page()
        page.goto("https://x.com/login")
        input("ログイン完了後 Enter: ")
        page.context.storage_state(path="x_state.json")

# 以降の収集はこれを再利用
ctx = browser.new_context(storage_state="x_state.json")
```

## 取得結果を SQLite へ正規化保存する

収集した投稿を Claude Haiku に渡す前に、SQLite で重複排除する。`tweet_id` を UNIQUE にすれば再実行しても二重保存が起きず、Haiku への投入トークン（$0.25/1M）を無駄に消費しない。

```python
import sqlite3

conn = sqlite3.connect("tweets.db")
conn.execute("""
CREATE TABLE IF NOT EXISTS tweets (
    tweet_id   TEXT PRIMARY KEY,
    author     TEXT NOT NULL,
    body       TEXT NOT NULL,
    fetched_at TEXT DEFAULT (datetime('now')),
    muted      INTEGER DEFAULT 0  -- Claude判定後に更新
)""")

def upsert(tweet_id, author, body):
    conn.execute(
        "INSERT OR IGNORE INTO tweets(tweet_id,author,body) VALUES(?,?,?)",
        (tweet_id, author, body))
    conn.commit()
```

`INSERT OR IGNORE` により、Playwright が同じリストを再読込しても既存行は触らない。この DB が次章の Claude ミュート判定の入力テーブルになる。
