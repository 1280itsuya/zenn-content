M4でコスト最大¥241になった理由は、few-shot例を12件→20件に増やしたプロンプト改修。M5以降はバッチサイズを50→100件に拡大してAPIコールを半減させ、精度を維持しながらコストを削減した。削減率の計算式は `ミュート件数 / (ミュート件数 + 手動で読み飛ばしたノイズ件数)` で、後者はランダム50件サンプリングから推定。

```python
# src/cost_tracker.py
import json
from pathlib import Path
from datetime import datetime

COST_PER_1K_INPUT = 0.003   # claude-3-5-haiku-20241022 $/1K tokens
COST_PER_1K_OUTPUT = 0.015
JPY_RATE = 155.0

def log_monthly_cost(month: str, input_tokens: int, output_tokens: int):
    cost_usd = (input_tokens / 1000 * COST_PER_1K_INPUT
                + output_tokens / 1000 * COST_PER_1K_OUTPUT)
    cost_jpy = cost_usd * JPY_RATE

    record = {
        "month": month,
        "input_tokens": input_tokens,
        "output_tokens": output_tokens,
        "cost_usd": round(cost_usd, 4),
        "cost_jpy": round(cost_jpy, 1),
        "recorded_at": datetime.utcnow().isoformat()
    }
    log_path = Path("data/cost_log.jsonl")
    with log_path.open("a", encoding="utf-8") as f:
        f.write(json.dumps(record, ensure_ascii=False) + "\n")
    return cost_jpy
```

5例全員を1週間以内に解除し、ホワイトリスト機構を実装した。

```python
# src/whitelist.py
import json
from pathlib import Path

WHITELIST_PATH = Path("config/whitelist.json")

def load_whitelist() -> set[str]:
    if not WHITELIST_PATH.exists():
        return set()
    return set(json.loads(WHITELIST_PATH.read_text(encoding="utf-8")))

def is_whitelisted(user_id: str) -> bool:
    return user_id in load_whitelist()

def add_to_whitelist(user_id: str, reason: str = ""):
    wl = load_whitelist()
    wl.add(user_id)
    WHITELIST_PATH.write_text(
        json.dumps(sorted(wl), ensure_ascii=False, indent=2),
        encoding="utf-8"
    )
    print(f"[whitelist] added {user_id}: {reason}")

# muter.py 側での利用
def should_mute(user_id: str, tweet_text: str, claude_verdict: dict) -> bool:
    if is_whitelisted(user_id):
        return False
    return claude_verdict.get("mute", False)
```

## 2025年8月X APIポリシー変更への緊急対応記録

2025年8月14日、X API Basic プランのユーザータイムライン取得エンドポイント（`GET /2/timelines/home_timeline`）のレートリミットが1時間あたり180→75リクエストに変更された。深夜2時にGitHub ActionsのCronが連続失敗し、Discordアラートで気づいた。

対応に要した時間は2時間。変更点は以下の3か所。

```python
# src/fetcher.py  変更前後の差分

# Before: 15分ごとに実行、1回あたり最大100件
FETCH_INTERVAL_MINUTES = 15
MAX_RESULTS_PER_CALL = 100

# After: 48分ごとに実行、1回あたり最大100件（1時間あたり75リクエスト以内）
FETCH_INTERVAL_MINUTES = 48
MAX_RESULTS_PER_CALL = 100

# レートリミット超過時のエクスポネンシャルバックオフ
import time

def fetch_with_backoff(client, max_retries: int = 5):
    for attempt in range(max_retries):
        try:
            return client.get_home_timeline(max_results=100)
        except Exception as e:
            if "429" in str(e):
                wait = 2 ** attempt * 60  # 1min, 2min, 4min, 8min, 16min
                print(f"Rate limited. Waiting {wait}s (attempt {attempt+1})")
                time.sleep(wait)
            else:
                raise
    raise RuntimeError("Max retries exceeded on rate limit")
```

## フォロー推薦システムへの拡張設計と次のステップ

ミュートデータの逆用：「ノイズ率0%のアカウント」上位を抽出し、フォロー推薦リストを自動生成する拡張を設計した。

```python
# src/follow_recommender.py（設計中・未本番）
from collections import defaultdict
import json
from pathlib import Path

def compute_noise_score(user_id: str, log_path: Path) -> float:
    """
    ミュートログからユーザーごとのノイズスコアを算出。
    0.0 = 一度もノイズ判定なし（推薦候補）
    1.0 = 全ツイートがノイズ判定
    """
    records = [
        json.loads(l) for l in log_path.read_text().splitlines()
        if json.loads(l)["author_id"] == user_id
    ]
    if not records:
        return 0.5  # データなし
    mute_count = sum(1 for r in records if r["verdict"]["mute"])
    return mute_count / len(records)

def recommend_follows(top_n: int = 20) -> list[str]:
    log_path = Path("data/classification_log.jsonl")
    user_scores: dict[str, float] = {}
    all_users = set(
        json.loads(l)["author_id"]
        for l in log_path.read_text().splitlines()
    )
    for uid in all_users:
        user_scores[uid] = compute_noise_score(uid, log_path)

    return sorted(user_scores, key=lambda u: user_scores[u])[:top_n]
```

次の実装ステップは2点。①`follow_recommender.py` をGitHub Actionsの週次Cronに組み込み、Discordに候補リストを通知する。②ミュート解除フロー（ホワイトリスト追加→X API `DELETE /2/users/:id/muting/:target_user_id` 自動実行）をUIなしで完結させる。
