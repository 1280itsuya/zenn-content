## 結論: 週6.0h→2.5hは「再学習ループ」を回した月だけ達成できた

半年(26週)のログを集計すると、情報収集は週6.0h→2.5h、削減3.5h。ただし削減幅は一定ではない。再学習を回した月の平均は2.3h、サボった2月だけ4.1hに戻った。Claude Haikuのスコアは固定ではなく腐る。

```python
import pandas as pd
df = pd.read_csv("logs/weekly_hours.csv")  # week, hours, retrained(bool)
print(df.groupby("retrained")["hours"].mean())
# False    3.98
# True     2.31  ← 差1.67h/週がメンテの価値
```

## 精度ドリフト: 興味変化でClaude判定が誤ミュート月17件に悪化

3月に副業の関心がKDPからX運用へ移った結果、「X grow」系を宣伝アカと誤判定し誤ミュートが月4→17件へ急増。判定理由を毎回保存しておくと原因追跡が一発で済む。

```python
def log_decision(tweet_id, score, reason, action):
    with open("logs/decisions.jsonl", "a", encoding="utf-8") as f:
        f.write(json.dumps({"id": tweet_id, "score": score,
            "reason": reason, "action": action}, ensure_ascii=False) + "\n")
```

## ミュート閾値の月次再学習: 0.70→0.62へ自動降下

復活(unmute)ログを正解ラベルとして、誤ミュート率が5%を超えたら閾値を0.02ずつ下げる半自動ループ。X API free枠の月1,500件内で完結し追加コストは¥0。

```python
def retune(threshold, false_mute_rate):
    if false_mute_rate > 0.05:
        return round(max(0.50, threshold - 0.02), 2)
    if false_mute_rate < 0.01:
        return round(min(0.80, threshold + 0.02), 2)
    return threshold
# 0.70 → 0.62 まで4ヶ月かけて降下、誤ミュート17→3件
```

## allowlistで救済: 復活3回でClaude判定を恒久バイパス

同一アカウントを3回unmuteしたらClaudeのスコアリング対象から外す。再学習でも揺れない確定ホワイトリストを別管理する。

```python
from collections import Counter
unmutes = Counter(l["id"] for l in load_jsonl("logs/unmute.jsonl"))
allowlist = {uid for uid, n in unmutes.items() if n >= 3}
json.dump(list(allowlist), open("config/allowlist.json", "w"))
# 半年で12アカウントが恒久救済され、再誤ミュート0件
```
