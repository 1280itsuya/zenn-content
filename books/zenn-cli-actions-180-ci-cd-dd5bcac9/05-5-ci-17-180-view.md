## steps.outcome == failure で「公開0本」を検知する Discord 通知

`zenn-cli` の deploy は GitHub Token 失効や push 競合で exit 0 のまま記事ゼロになる。`if: failure()` だけだと前段の生成ジョブ成功に引きずられ通知が飛ばないので、deploy ステップに `id` を付けて `steps.<id>.outcome` を直接判定する。

```yaml
      - name: Deploy to Zenn
        id: deploy
        run: git push origin main
      - name: Notify failure
        if: steps.deploy.outcome == 'failure'
        run: |
          curl -H "Content-Type: application/json" \
            -d "{\"content\":\"🚨 deploy failed: ${{ github.run_id }} / published=0\"}" \
            "${{ secrets.DISCORD_WEBHOOK }}"
```

## 180本の累計view と流入偏在を jq で実数集計する

Zenn API から取得した180本の `page_views` を集計すると、上位8本で全viewの71%を占めた。残り172本は1本あたり中央値9view、つまりロングテールは事実上死んでいる。

```bash
curl -s "https://zenn.dev/api/me/articles?count=180" \
  | jq '[.articles[].page_view_count] | {total: add, median: (sort | .[length/2 | floor]), top8: (sort | reverse | .[0:8] | add)}'
# => {"total": 14210, "median": 9, "top8": 10089}
```

## アフィリ計測リンク有無で生じた収益差(¥0 vs A8計測)

180本中、A8計測リンクを正しく埋めたのは34本。計測リンクなしの146本はクリックが追跡不能で、実質収益¥0として扱うしかなかった。計測ありの34本でも累計クリック41・確定報酬¥0という結果で、view総数14,210に対しCTRは0.3%未満。

```python
measured, unmeasured = 34, 146
clicks, reward = 41, 0
cvr = clicks / 14210
print(f"計測あり={measured}本 clicks={clicks} CTR={cvr:.3%} reward=¥{reward}")
# 計測あり=34本 clicks=41 CTR=0.289% reward=¥0
```

## 「自動化しても集客が律速」を view 分布が裏付ける次の打ち手

deploy は半年で安定し公開漏れはゼロになったが、収益の律速は供給ではなく流入だった。上位集中の分布が示すのは「本数を180→360に倍増しても中央値9viewのテールが増えるだけ」という構造。次章では生成量を据え置き、流入のある外部媒体（Dev.to / はてな）へのクロス投稿と計測リンク埋込率を34/180→全本へ引き上げる配線に切り替える。

```python
# 投下リソースの再配分判断：本数増 vs 計測率改善
gain_more_posts = 9 * 180        # テール増の期待view ≈ 1620
gain_fix_tracking = 14210 * 0.7  # 既存viewへ計測を全付与 ≈ 9947
print("計測率改善が本数増の", round(gain_fix_tracking / gain_more_posts, 1), "倍効く")
# => 6.1 倍
```
