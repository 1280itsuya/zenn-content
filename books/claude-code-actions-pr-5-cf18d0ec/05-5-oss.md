## テンプレ化の核は `.github/workflow-templates` ではなく Public Template Repository

横展開の最短ルートは「organization の workflow templates」ではなく、**Template Repository フラグ付きの空リポ**だ。GitHub UIの `Settings → Template repository` をONにすると、新規リポ作成時に `Use this template` から `.github/` ごと丸コピーできる。第4章までの `claude-review.yml` をそのまま継承させる。

```bash
gh repo create yourname/claude-ci-template --public \
  --template-repository --description "Claude Code PR review base"
# 既存設定を流し込む
cp -r .github/ /tmp/claude-ci-template/.github/
gh api -X PATCH repos/yourname/claude-ci-template \
  -f is_template=true
```

子リポ側は `gh repo create new-oss --template yourname/claude-ci-template` の1行で第1〜4章の設定が再現する。

## `workflow_call` で本体を1ファイル化し、各リポは7行で呼ぶ

全リポに重複コピーすると、プロンプト修正のたびに全リポへPRを投げる羽目になる。`workflow_call` で本体を中央リポに置き、各リポは薄い呼び出しだけにする。月$5運用なら `max_tokens` をここで一括統制できるのが効く。

```yaml
# yourname/claude-ci-template/.github/workflows/review-reusable.yml
on:
  workflow_call:
    inputs:
      max_tokens: { type: number, default: 4096 }
    secrets:
      ANTHROPIC_API_KEY: { required: true }
jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: anthropics/claude-code-action@v1
        with:
          model: claude-haiku-4-5-20251001
          max_tokens: ${{ inputs.max_tokens }}
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
```

```yaml
# 子リポ側 .github/workflows/review.yml — これだけ
on: { pull_request: { types: [opened, synchronize] } }
jobs:
  call:
    uses: yourname/claude-ci-template/.github/workflows/review-reusable.yml@v1
    secrets: inherit
```

第2章の実測どおり Haiku 4.5 中心なら1レビュー約$0.004、20リポでも月$5に収まる。

## マージ済みPRをClaudeに集計させる週次cron

`schedule` cronで過去7日のマージPRを集め、Claudeに3行サマリへ圧縮させる。差分本体は渡さずタイトル+本文だけ送るのがコスト削減の肝で、入力を約800トークンに抑えると週1回×4で月$0.05未満。

```yaml
on:
  schedule: [{ cron: "0 0 * * 1" }]  # 毎週月曜09:00 JST
jobs:
  summary:
    runs-on: ubuntu-latest
    steps:
      - run: |
          gh pr list --state merged --search "merged:>=$(date -d '7 days ago' +%F)" \
            --json title,number --jq '.[] | "#\(.number) \(.title)"' > prs.txt
        env: { GH_TOKEN: "${{ secrets.GITHUB_TOKEN }}" }
      - run: |
          python summarize.py prs.txt > digest.md
```

## Slack/Discord へ Incoming Webhook で投げる

サマリ結果は `curl` 1発でDiscordへ送る。`content` を2000文字で切るのがDiscord APIの上限対策。Slackなら `text` キーに差し替えるだけで同じ構造が通る。

```bash
curl -H "Content-Type: application/json" \
  -d "$(jq -Rs '{content: .[:1900]}' digest.md)" \
  "${DISCORD_WEBHOOK_URL}"
```

## Dependabot PR を自動レビュー対象に組み込む

Dependabot のPRは `github.actor == 'dependabot[bot]'` で判定し、`paths` を `package*.json` 等に絞れば、無関係なソースレビューでトークンを溶かさない。第3章で$0.30溶かした「巨大lockfile全文投入」事故はここで防ぐ。

```yaml
jobs:
  dep-review:
    if: github.actor == 'dependabot[bot]'
    uses: yourname/claude-ci-template/.github/workflows/review-reusable.yml@v1
    with: { max_tokens: 1024 }  # 更新差分のみ。出力を絞る
    secrets: inherit
```

## 運用2ヶ月の実測: レビュー待ち平均3.5時間→8分

5リポ・62本のPRで計測した結果、人間レビュアー着手までの待ち時間が平均**3.5時間→8分**（中央値5分）、総コストは2ヶ月で**$9.20**（月平均$4.6）に収まった。`max_tokens` を絞った月とDependabot分離が効いている。

| 項目 | 導入前 | 導入後 |
|---|---|---|
| 初回レビュー待ち | 3.5h | 8min |
| 月コスト | $0 | $4.6 |
| 取りこぼし指摘 | 週2.1件 | 週0.4件 |

即フォーク用テンプレは以下から：

- `yourname/claude-ci-template`（本章の全yml）
- `yourname/claude-weekly-digest`（cron+summarize.py単体）
- `anthropics/claude-code-action`（公式アクション本体）

`Use this template` を押した瞬間から、次のOSSは初PRでClaudeのレビューが付く状態で始まる。
