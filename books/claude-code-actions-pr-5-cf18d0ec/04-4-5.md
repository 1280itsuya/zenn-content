## 30日・PR40本のClaude Code実測コストを表で公開する

結論から書く。Claude Sonnet 4.6でPR約40本をレビューし、30日の請求は$4.82に収まった。内訳の8割はinputトークンで、ここを削れば赤字は回避できる。

| 項目 | 数値 |
|---|---|
| レビュー対象PR | 41本 |
| 総input | 3.9M tok |
| 総output | 0.42M tok |
| cache read | 2.1M tok |
| 月額請求 | $4.82 |

請求の生データはAnthropic Consoleの`Usage`からCSVで落とせる。

```bash
# 期間指定で日次トークンを集計
cat usage_2026-06.csv | \
  awk -F, 'NR>1{i+=$3; o+=$4} END{printf "input=%d output=%d\n", i, o}'
```

## プロンプトキャッシュ有効化でinputコストを約7割削った前後比較

Sonnet 4.6のinputは$3/Mtok、cache readは$0.30/Mtokで10分の1になる。レビュー前提となるdiff規約・コーディング基準を`cache_control`で固定すると、2回目以降のinputがcache read単価に落ちる。

```yaml
# .github/workflows/review.yml の一部
- uses: anthropics/claude-code-action@v1
  with:
    model: claude-sonnet-4-6
    system_prompt_cache: true   # 規約ブロックをキャッシュ
```

```python
# SDK直叩きの場合：規約を ephemeral cache に載せる
system=[{
    "type": "text",
    "text": REVIEW_RULES,          # 約1.8k tok の固定規約
    "cache_control": {"type": "ephemeral"}
}]
```

有効化前はinput課金$0.39/PR、有効化後は$0.12/PR。41本換算で$16→$4.9相当、約69%減だった。

## concurrencyで同一PRの多重起動を止め二重課金を防ぐ

`synchronize`イベントは連続pushで何度も発火する。これを放置した最初の3日で、同一PRが4重起動し$1.3を溶かした。`concurrency`で古いrunをキャンセルする。

```yaml
concurrency:
  group: review-${{ github.event.pull_request.number }}
  cancel-in-progress: true   # 前のレビューrunを破棄
```

これでpush連打中のレビューは最新diffの1回だけ課金される。

## pathsフィルタでドキュメント変更をスキップし無駄実行を削る

`.md`やlockファイルだけのPRにレビューを走らせるのは課金の無駄。約40本中9本がドキュメントのみで、フィルタで丸ごとスキップした。

```yaml
on:
  pull_request:
    paths:
      - '**.py'
      - '**.ts'
      - '**.go'
    paths-ignore:
      - '**.md'
      - '**/package-lock.json'
```

9本スキップで月$1.0前後を削減した計算になる。

## cron夜間バッチ化と$5に収めるしきい値・Secrets一覧

push毎レビューを止め、cronで深夜2時にopen中PRをまとめてレビューする運用に変えると、リアルタイム性は落ちるが実行回数が読めてコストが固定化する。

```yaml
on:
  schedule:
    - cron: '0 17 * * *'   # UTC17時=JST2時、1日1回
```

$5以内に収めるしきい値は「PR40本/月・1PRあたりinput25k tok以下」。これを超えたら`max_tokens`を絞る。設定するSecrets/Variablesは以下。

```bash
gh secret set ANTHROPIC_API_KEY --body "sk-ant-..."
gh variable set CLAUDE_MODEL --body "claude-sonnet-4-6"
gh variable set REVIEW_MAX_TOKENS --body "1500"   # output上限で暴走課金を遮断
```

`REVIEW_MAX_TOKENS`を1500に固定するとoutput課金が$15/Mtok×上限で頭打ちになり、1PRの最大コストが$0.15に確定する。これで月$5の天井を割らない。
