## 12%巻き込みを生んだ素朴プロンプトの失敗ログ

最初は「ノイズなら mute と返して」だけ。有益ツイート312件中38件(12.2%)を誤ミュートした。原因は出力が自由文で、技術系の強い語調を「煽り」と誤認していた点。

```python
BAD = "次のツイートがノイズなら mute、違えば keep と返答:\n{tweet}"
# → "これは一見煽りに見えますが..." と解説まで返り、parseで keep/mute を取り違える
```

## ノイズ7分類とFew-shotを固定するシステムプロンプト

`宣伝/煽り/スパム/連投/政治炎上/低情報量/有益` の7値に限定し、各2件のFew-shotで境界を教える。

```python
SYSTEM = """ツイートを7分類する。値は以下のみ:
promo, bait, spam, flood, political_flame, low_info, useful
例:
- "詳細はDMで！稼げます" -> promo
- "EXPLAIN ANALYZEでN+1を特定した手順" -> useful
JSONのみ返す。"""
```

## JSON強制・temperature 0・信頼度で1.8%まで詰める

`temperature=0` と `tools` によるJSON強制で出力を固定し、`confidence` を付けて閾値ミュートに切り替えた。これで誤ミュートは1.8%へ低下した。

```python
import anthropic
client = anthropic.Anthropic()

def judge(tweet: str) -> dict:
    msg = client.messages.create(
        model="claude-haiku-4-5-20251001",
        max_tokens=128, temperature=0,
        system=SYSTEM,
        messages=[{"role": "user", "content": tweet}],
        tools=[{
            "name": "classify",
            "input_schema": {"type": "object", "properties": {
                "label": {"type": "string"},
                "confidence": {"type": "number"}},
              "required": ["label", "confidence"]}}],
        tool_choice={"type": "tool", "name": "classify"})
    return next(b.input for b in msg.content if b.type == "tool_use")

# confidence>=0.8 かつ useful以外のみ Playwright へ渡す
```

## リトライ・レート制御・コスト計測を1関数に同梱

`529 Overloaded` を指数バックオフで吸収し、Haikuの実トークン単価でコストを積算する。1,000件で約$0.21だった。

```python
import time
COST_IN, COST_OUT = 1.0/1e6, 5.0/1e6  # Haiku 4.5 概算$/token

def judge_safe(tweet, total={"usd": 0.0}):
    for i in range(5):
        try:
            r = client.messages.create(...)  # 上のcreateと同一
            u = r.usage
            total["usd"] += u.input_tokens*COST_IN + u.output_tokens*COST_OUT
            return r
        except anthropic.APIStatusError:
            time.sleep(2 ** i)
    raise RuntimeError("retry exhausted")
```

## SQLite蓄積で閾値を後から再調整する

判定をすべて残し、`confidence` の分布から最適カットを後日SQLで探れるようにする。

```python
import sqlite3
db = sqlite3.connect("mute_log.db")
db.execute("""CREATE TABLE IF NOT EXISTS judge(
  id INTEGER PRIMARY KEY, tweet TEXT, label TEXT,
  conf REAL, muted INTEGER, ts TEXT DEFAULT (datetime('now')))""")

def log(tweet, res, muted):
    db.execute("INSERT INTO judge(tweet,label,conf,muted) VALUES(?,?,?,?)",
               (tweet, res["label"], res["confidence"], int(muted)))
    db.commit()

# 後日: 誤ミュート見直し
# SELECT label, AVG(conf), SUM(muted) FROM judge GROUP BY label;
```

このログがあるため、次章のPlaywright投入後に「`bait` の閾値だけ0.85へ上げる」といった調整を再生成なしで回せる。
