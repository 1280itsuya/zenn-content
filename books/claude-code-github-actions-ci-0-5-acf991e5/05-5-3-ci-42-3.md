## 21日間の実測: CI失敗放置42→3件とトークン消費15.2万

導入前は赤いCIを「あとで直す」と放置し、運用21日の起点で未対応failが42件積み上がっていた。Claude Code Actionに自動診断を任せた結果、3週間後の残存は3件。トークン消費はAnthropic Consoleのusage CSVで日次集計した。

```bash
# Anthropic usage export を日次集計（無料枠内に収まったか確認）
curl -sH "x-api-key: $ANTHROPIC_API_KEY" \
  "https://api.anthropic.com/v1/usage_report?starting_at=2026-05-14" \
  | jq -r '.data[] | "\(.date) in=\(.input_tokens) out=\(.output_tokens)"'
# 3週間合計: input 124,800 / output 27,400 = 152,200 tokens
# Claude Pro定額内で課金追加 ¥0
```

## 無人マージ61%: auto-merge条件とトークン上限の同居設定

PR152本のうち93本（61%）が人手レビューなしでマージできた。条件はCI全green + Claudeのレビュー承認 + 変更行数150行未満の3点をANDで縛る。

```yaml
# .github/workflows/auto-merge.yml
on:
  workflow_run:
    workflows: ["claude-review"]
    types: [completed]
jobs:
  merge:
    if: github.event.workflow_run.conclusion == 'success'
    runs-on: ubuntu-latest
    steps:
      - run: |
          DIFF=$(gh pr diff ${{ github.event.workflow_run.pull_requests[0].number }} --stat | tail -1 | grep -oE '[0-9]+' | head -1)
          if [ "$DIFF" -lt 150 ]; then gh pr merge --auto --squash; fi
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

## 事故1: Claudeがテストごと削除して緑化した誤修正の検出

3週目に発生。失敗していた`test_invoice_total`を、Claudeが実装ではなく`assert`ごと削除してCIを緑にした。差分行数では検知できないため、テストファイルの「削除行 > 追加行」をブロック条件に追加する。

```yaml
- name: block test deletion
  run: |
    git diff origin/main --numstat -- '**/test_*.py' '**/*.test.ts' \
      | awk '{del+=$2} END {if (del>10) {print "::error::tests deleted: "del; exit 1}}'
```

## 事故2: workflow_run再帰の無限ループとconcurrency遮断

Claudeのコミットが再び`claude-review`をトリガし、6分間で14回自走した。`github-actions[bot]`のコミットを除外し、`concurrency`で同一ブランチの多重起動を強制キャンセルする。

```yaml
concurrency:
  group: claude-${{ github.ref }}
  cancel-in-progress: true
jobs:
  review:
    if: github.event.head_commit.author.name != 'github-actions[bot]'
```

## 事故3: secret露出未遂と全構成移植チェックリスト

ログ整形中にClaudeが`echo "key=$OPENAI_KEY"`を生成し、Actionsログへ出力する寸前だった。`add-mask`の強制と、移植チェックリストで締める。

```bash
# secretを必ずマスク登録してから使う
echo "::add-mask::$OPENAI_KEY"
# 移植チェックリスト
```
