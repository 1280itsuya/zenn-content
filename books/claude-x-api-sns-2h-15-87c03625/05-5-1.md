## Claudeで短評ポスト下書きを生成するプロンプト全文

要約済みの1次情報JSONを渡し、140字以内・主観1行・出典URL付きの下書きを返させる。temperature=0.4で語尾のブレを抑えた。

```python
import anthropic

client = anthropic.Anthropic()

PROMPT = """次の1次情報を、Xの副業アカウント用ポスト下書きに変換しろ。
制約:
- 全角140字以内、絵文字は0個
- 1行目=事実の核、2行目=実務での示唆を1つ、3行目=出典URL
- 誇張・煽り・「必見」「神」などの語は禁止
- 確証のない数値は書かない
入力: {info}"""

def draft(info: str) -> str:
    msg = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=300,
        temperature=0.4,
        messages=[{"role": "user", "content": PROMPT.format(info=info)}],
    )
    return msg.content[0].text
```

## 転載と引用の線引きを40字ルールで自動化する

本文からの引用は40字以内に抑え、必ず原典URLとタイトルを末尾に付ける。引用率が閾値を超えた下書きは投稿前に弾く。

```python
def guard_quote(draft_text: str, source_text: str, url: str) -> str:
    overlap = max(
        (len(s) for s in source_text.split("。") if s and s in draft_text),
        default=0,
    )
    if overlap > 40:
        raise ValueError(f"引用過多 {overlap}字: 要約し直し")
    if url not in draft_text:
        draft_text += f"\n出典: {url}"
    return draft_text
```

## フォロワー0→340で「保存される投稿」の共通パターン

2か月・212投稿のうち保存上位30件を集計すると、保存数の73%が「数値+再現手順」を含む投稿に集中していた。煽りワード入りは平均保存4.2に対し、数値入りは18.6。

```python
import pandas as pd

df = pd.read_csv("posts.csv")  # cols: text, bookmarks
df["has_number"] = df["text"].str.contains(r"\d+(%|円|倍|分)")
top = df.sort_values("bookmarks", ascending=False).head(30)
print(top["has_number"].mean())          # 0.73
print(df.groupby("has_number")["bookmarks"].mean())
# False   4.2
# True   18.6
```

## 完全自動投稿を避ける理由と週次チェックリスト

投稿APIで日10件×3日流したテストアカウントは4日目に表示制限を受けた。人手添削を挟む運用に戻してからは制限0件。週次の確認をBashで定型化し、収集2h→15分・誤ミュート1.8%・APIコスト月¥2,400の3指標を毎週監視している。

```bash
#!/usr/bin/env bash
# 毎週日曜に実行: 収集→発信の健全性チェック
echo "[1] 誤ミュート率(目標<2%):"; python eval_mute.py --week
echo "[2] APIコスト(目標<¥2400/月):"; python cost_report.py
echo "[3] 下書き承認率(人手添削の負荷):"; python approval_rate.py
echo "[4] 保存数Top5の再投稿候補:"; python pick_evergreen.py --top 5
```

このチェックを8週続けた結果、誤ミュート1.8%を維持したままネタ切れせず週14投稿を回せている。収集で削ったノイズの分だけ、発信に回す1次情報の純度が上がる構造になっている。
