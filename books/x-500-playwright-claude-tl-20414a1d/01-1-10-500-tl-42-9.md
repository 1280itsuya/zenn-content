## 完成物：500件ミュートを10分で終える1コマンド

このBookで作るのは、PlaywrightがXのTLをスクロールし、Claudeが各アカウントを分類して「宣伝・煽り・情報商材」だけを自動ミュートするスクリプトだ。実行はこれだけ。

```bash
# .env に X_USER / X_PASS / ANTHROPIC_API_KEY を設定済み前提
python mute_runner.py --max 500 --action mute --dry-run false
# => Scanned 512 accounts / Muted 500 / Skipped 12 (follow-back) in 9m42s
```

`--dry-run true` で分類結果だけ吐かせ、目視で確認してから本番を流す運用にしている。500件で実測9分42秒、APIコストは後述する。

## ノイズ率42%→9%：2週間の実測グラフ

独自指標「ノイズ率 = 雑音ツイート数 / 直近100件」を毎朝計測した。計測スクリプトもPlaywrightで自動化している。

```python
def noise_rate(tweets: list[dict]) -> float:
    junk = [t for t in tweets if t["label"] in ("ad", "bait", "infoproduct")]
    return round(len(junk) / len(tweets) * 100, 1)

# Day1: 42.0  Day3: 31.5  Day7: 18.2  Day14: 9.0
```

ミュート直後ではなく14日かけて9%まで下がる。理由は、Xのアルゴリズムが「ミュート対象に似たアカウント」のおすすめ表示を徐々に減らすため。一括ミュートは引き金で、TLの自浄はその後に効く。

## 消した500件の内訳と誤爆7件の復旧ログ

Claudeが付けたラベルを集計すると、雑音の正体が数値で見える。

```json
{
  "ad_promo":        115,
  "politics":         90,
  "viral_bait":       55,
  "infoproduct":      48,
  "spam_reply":       42,
  "ambiguous":       150
}
```

宣伝23%・政治18%・バズ便乗11%・情報商材10%。誤爆は500件中7件で、いずれも「副業」という語に反応した正規の技術アカウントだった。復旧は`unmute`アクションでアカウントIDを渡すだけ。

```bash
python mute_runner.py --action unmute --ids data/false_positives.txt
# => Unmuted 7 accounts in 11s
```

## なぜX API v2無料枠では不可能か

「APIでやればいい」が通らない理由を1分で示す。無料枠の制約を表にすると詰みが見える。

```text
X API v2 Free tier (2026-06時点)
  - POST操作のみ / 月1,500件まで
  - mutes 取得・作成エンドポイントは Basic($200/月)以上
  - TL本文の取得は実質不可
=> 500件ミュート + 分類用の本文取得 は無料枠で実行不能
```

ブラウザ自動化なら、人間が手で操作する画面をそのまま叩くため、この課金ゲートを通らずに同じことができる。次章でPlaywrightのログイン突破とセッション永続化（Cookie再利用で2回目以降のログインを0秒に）を実装する。手で500件を消す数時間を、この1冊で10分に置き換える。
