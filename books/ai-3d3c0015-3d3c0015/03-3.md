## 収益パイプラインの全体像

稼げる自動投稿システムの核心は「生成→品質チェック→投稿→計測」の4ステージを途切れなく繋ぐことだ。どれか一つでも手作業が残ると、ループが止まった瞬間に収益も止まる。

本章では実際に稼働しているパイプラインをベースに、ゼロから組み立てる手順を示す。

---

## ステージ1：ネタ生成エージェント

まず「何を書くか」を自動で決める。テーマの選定をAIに任せることが時間節約の第一歩だ。

```python
# src/topic_planner.py
import anthropic

client = anthropic.Anthropic()

def pick_topic(niche: str) -> str:
    msg = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=256,
        messages=[{
            "role": "user",
            "content": f"ニッチ「{niche}」で今週検索急上昇中の悩みを1つ答えよ。タイトル形式で。"
        }]
    )
    return msg.content[0].text
```

`niche`には「節約術」「副業Python」などを渡す。毎朝7時にスケジューラが呼び出し、その日のテーマを決定する。

---

## ステージ2：本文生成と品質ゲート

テーマが決まったら記事本文を生成し、品質チェックを通過したもののみ次へ進める。

```python
def generate_and_gate(topic: str) -> str | None:
    draft = generate_article(topic)        # LLMで下書き生成
    score = judge_quality(draft, topic)    # LLM-as-Judge で採点
    if score < 7:
        return None                        # 低品質は廃棄
    return draft
```

`judge_quality`はタイトルと本文の一致度・具体性・文字数を0〜10で採点する別エージェントだ。スコアが閾値を下回れば静かに破棄し、翌サイクルで再挑戦する。品質ゲートを省くと、テンプレ丸出しの記事がプラットフォームに拒否されてアカウントごと停止される。

---

## ステージ3：マルチチャネル投稿

品質を通過した記事はZenn・Qiita・はてなブログへ同時投稿する。チャネルごとにAPIが異なるため、共通インターフェイスを作る。

```python
# orchestrator/run_pipeline.py
from posters import zenn, qiita, hatena

POSTERS = [zenn.post, qiita.post, hatena.post]

def publish_all(article: dict):
    for poster in POSTERS:
        try:
            poster(article)
        except Exception as e:
            log_error(poster.__name__, e)  # 失敗しても他チャネルは継続
```

Zennはgit pushで公開するため`GIT_TERMINAL_PROMPT=0`を環境変数に必ず設定する。これを忘れると認証ダイアログ待ちで2時間ハングする（実際に起きた）。

---

## ステージ4：計測と勝ちパターンの還元

投稿した翌日以降、各チャネルのAPIでview数を収集し、上位記事のタグ・構成をフィードバックループに乗せる。

```bash
python src/winner_extractor.py \
  --channel zenn --top 5 \
  --out data/winning_topics.json
```

このJSONを翌朝のテーマ選定エージェントに食わせることで、「よく読まれた構成を模倣して次の記事を書く」サイクルが回り始める。最初の1ヶ月は赤字でも、3ヶ月後には勝ちパターンが自動で蓄積され、view数が複利で伸びていく。

## Task Schedulerで朝7時に全自動起動

Windowsなら以下でサスペンドなしに毎朝実行できる。

```powershell
schtasks /create /tn "AutoMoney_7am" `
  /tr "python C:\auto-money\orchestrator\run_pipeline.py" `
  /sc daily /st 07:00 /ru SYSTEM
```

`/ru SYSTEM`を指定するとログオフ中でも動作する。ここまで組めば、眠っている間にパイプラインが記事を生成・投稿・計測し、翌朝には結果レポートがDiscordに届いている状態が完成する。
