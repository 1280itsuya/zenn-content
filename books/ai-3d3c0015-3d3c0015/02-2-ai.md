## 収益パイプラインとは何か

AIで稼ぐ仕組みは、単発のスクリプトではなく「パイプライン」として設計する必要がある。パイプラインとは、**入力→生成→検証→投稿→計測**という処理が自動でつながった一連の流れだ。各工程が独立したモジュールになっていることで、どこかが壊れても他の工程は動き続け、修正も局所化できる。

## 基本の5ステップ構造

収益パイプラインは次の5ステップで構成する。

1. **テーマ選定**：検索ボリュームや競合度から稼げるキーワードを抽出する
2. **コンテンツ生成**：LLM（Claude/GPT等）でドラフトを生成する
3. **品質ゲート**：タイトルと本文の一致・文字数・禁止ワードを自動検証する
4. **投稿**：Zenn・Qiita・はてな等のAPIで自動公開する
5. **計測フィードバック**：view数・クリック数を取得し、次のテーマ選定に還元する

この構造を1つのオーケストレーターが束ねる。

```python
# orchestrator.py の骨格
for channel in CHANNELS:
    topic = theme_selector.pick(channel)
    draft  = generator.run(topic)
    if not quality_gate.check(draft):
        continue          # 品質不合格はスキップ
    poster.publish(channel, draft)
    metrics.record(channel, topic)
```

## 設計上の3原則

**① env未設定はスキップ**：APIキーが存在しないチャネルは静かに飛ばす。`if not os.getenv("ZENN_PAT"): return` の1行で実現できる。これにより、部分的な環境でもパイプライン全体が止まらない。

**② 品質ゲートは必ず挟む**：LLMはタイトルと無関係な本文を生成することがある。`assert topic_keyword in body[:200]` のような簡易チェックでも、スパム記事の量産を防げる。

**③ 計測ループを閉じる**：投稿して終わりではなく、数日後にview数を取得して`winners.json`に記録する。上位テーマのタグやジャンルを次回の生成プロンプトに注入することで、精度が自己強化される。

## スケジューリングで「寝ている間に稼ぐ」

設計が固まったら、Windowsタスクスケジューラ（またはcron）で毎朝7時に実行を登録する。

```
schtasks /create /tn "AutoMoney" /tr "python orchestrator.py" /sc daily /st 07:00
```

これで**1日1回、人間が何もしなくても**コンテンツが生成・投稿・計測されるサイクルが回り始める。重要なのは「動かし続けること」だ。初月の収益はほぼゼロでも、3ヶ月後には検索インデックスが積み上がり、流入が始まる。パイプラインの価値は、時間とともに複利的に増大する。

## まとめ

収益を生むAIパイプラインの本質は、**5ステップの自動化＋品質ゲート＋計測フィードバック**の三位一体だ。どれか1つが欠けても、ゴミが量産されるか、稼げても改善されないかのどちらかになる。次章では、このパイプラインを実際にコードで組み上げる手順を解説する。
