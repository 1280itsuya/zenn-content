## 自動改善ループとは何か

コンテンツを量産するだけでは収益は頭打ちになる。重要なのは「何が売れたか」を自動で学習し、次の生成に反映する**フィードバックループ**の仕組みだ。人間がPDCAを回すのではなく、AIがデータを見て自律的にパラメータを調整する。これが自動改善ループの本質である。

## 計測→抽出→反映の3ステップ

ループは次の3ステップで構成する。

**Step 1: 計測**　各プラットフォームのAPIからビュー数・クリック数・売上を毎日取得し、ローカルのSQLiteに蓄積する。

```bash
python src/collect_metrics.py --date today
```

**Step 2: 勝ちパターンの抽出**　上位20%の記事タイトル・タグ・文字数をLLMに渡し、共通点を言語化させる。

```bash
python src/winner_extractor.py --top-n 10 --output data/exemplars.json
```

出力例：`{"winning_tags": ["Claude", "自動化", "副業"], "avg_length": 1800, "hook_pattern": "エラー文直引用"}`

**Step 3: 生成への反映**　次の生成サイクルでこのJSONをシステムプロンプトに注入する。前回の「勝ちフォーマット」が自動的に踏襲される。

```python
exemplars = json.load(open("data/exemplars.json"))
system_prompt = f"以下の勝ちパターンに従って記事を書け: {exemplars}"
```

## 死蔵を防ぐ「applied」フラグ管理

改善案を生成しても適用されなければ意味がない。`config/agents.json`に`applied: false`の項目が溜まり続けるのがよくある失敗だ。対策として、改善案をファイルに書いた直後に自動でコードへ反映するスクリプトを組み込む。

```bash
python src/apply_patches.py --auto-new  # net-newな設定変更のみ自動適用
```

既存ロジックの変更は人間ゲートを挟み、設定値の追加・タグ追加などは即時反映する。この二段構えで「提案したが放置」を防ぐ。

## ループの健全性を毎朝確認する

自律ループは静かにハングする。orchestratorがCPU0%で止まっていても気づかない。毎朝7時のタスク後に必ず以下を実行し、前日比でビュー数が増えているかを確認する仕組みを入れる。

```bash
python src/health_check.py --alert-if-no-new-views
```

異常を検知したらDiscordへ通知し、ハングプロセスを自動killして再起動する。これでループが途切れず回り続ける。

## 実測値がループを加速する

筆者の環境では、このループを稼働させた結果、Zennの週間ビュー数が3週間で2.4倍になった。重要なのは「気合い」ではなく**数値が数値を呼ぶ構造**を作ることだ。一度ループが自走し始めれば、人間の仕事は異常検知と大方針の調整だけになる。
