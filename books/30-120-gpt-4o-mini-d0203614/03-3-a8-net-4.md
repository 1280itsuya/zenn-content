## トピック別リンクマッピングをYAMLで一元管理する

A8.net計測リンクとふるさと納税4社（さとふる/楽天/ふるなび/ANA）を、記事トピックへ機械的に紐付ける。リンク文字列をコードに直書きするとnofollow漏れやサブID欠落が混入するため、`links.yaml`に集約してテストで検証する。

```yaml
# links.yaml — topic ごとに許可リンクを定義
furusato:
  keywords: [ふるさと納税, 返礼品, 寄付, ワンストップ]
  links:
    - {name: 楽天ふるさと納税, url: "https://hb.afl.rakuten.co.jp/...", subid: rakuten_fr}
    - {name: さとふる, url: "https://px.a8.net/svt/...", subid: satofull, rel: nofollow}
credit:
  keywords: [クレカ, 還元率, 年会費無料, ポイント]
  links:
    - {name: A8計測, url: "https://px.a8.net/svt/ejp?...", subid: card_top3, rel: sponsored}
```

## キーワード照合で文脈不一致な挿入を弾く

本文に該当トピックの語が2回以上出現しなければ挿入しない。クレカ記事にふるさと納税リンクが混ざる事故をこの閾値で防ぐ。

```python
import yaml, re
CFG = yaml.safe_load(open("links.yaml", encoding="utf-8"))

def match_topic(body: str) -> str | None:
    best, score = None, 0
    for topic, c in CFG.items():
        hit = sum(body.count(k) for k in c["keywords"])
        if hit >= 2 and hit > score:
            best, score = topic, hit
    return best  # 該当なしは None → リンク挿入せず公開
```

## テーマdrift（実測12%）を検出して公開中止する

タイトル名詞と本文トピックが食い違う記事が30本中およそ12%発生した。drift記事へリンクを差すとクリック率が落ちるため、不一致は`raise`で公開を止める。

```python
def guard_drift(title: str, body_topic: str):
    title_topic = match_topic(title * 3)  # タイトルを重み付け
    if title_topic and title_topic != body_topic:
        raise ValueError(f"drift: title={title_topic} body={body_topic} → 公開中止")
```

## 重複・nofollow漏れを止める整合性チェック

同一URLの二重挿入と`rel`属性の欠落は、それぞれ規約違反と計測ノイズの原因になる。挿入前に全リンクを走査して弾く。

```python
def validate(links: list[dict]) -> list[dict]:
    seen = set()
    for l in links:
        if l["url"] in seen:
            raise ValueError(f"重複リンク: {l['name']}")
        seen.add(l["url"])
        if l["url"].startswith("https://px.a8") and "rel" not in l:
            raise ValueError(f"nofollow/sponsored 漏れ: {l['name']}")
    return links
```

## UTM+サブIDを付与し本文末へ挿入する

サブIDに記事スラッグを埋め込むと、A8.net管理画面で「どの記事のどのリンクが踏まれたか」を後から集計できる。

```python
from urllib.parse import urlencode

def decorate(url: str, subid: str, slug: str) -> str:
    sep = "&" if "?" in url else "?"
    q = urlencode({"a8mat": f"{subid}_{slug}",
                   "utm_source": "zenn", "utm_campaign": slug})
    return f"{url}{sep}{q}"

def inject(body: str, topic: str, slug: str) -> str:
    links = validate(CFG[topic]["links"])
    lines = [f"- [{l['name']}]({decorate(l['url'], l['subid'], slug)})"
             for l in links]
    return body + "\n\n## 関連リンク\n" + "\n".join(lines)
```

## 踏まれたリンクをSQLiteで集計する

A8.netの成果レポートCSVを`a8mat`キーで突合し、記事別クリックを日次で貯める。30本運用で「上位3記事が全クリックの68%」のような偏りが見え、次の量産トピックを絞れる。

```python
import sqlite3, csv
db = sqlite3.connect("clicks.db")
db.execute("CREATE TABLE IF NOT EXISTS clicks(slug TEXT, subid TEXT, n INT)")
for row in csv.DictReader(open("a8_report.csv", encoding="cp932")):
    subid, slug = row["広告主サブID"].rsplit("_", 1)
    db.execute("INSERT INTO clicks VALUES(?,?,?)", (slug, subid, int(row["クリック数"])))
db.commit()
for r in db.execute("SELECT slug, SUM(n) FROM clicks GROUP BY slug ORDER BY 2 DESC LIMIT 5"):
    print(r)  # クリック上位5記事 → 横展開の対象
```

これで本文生成からリンク挿入、drift中止、クリック集計までが1パイプラインで閉じる。次章では公開後の投稿反映漏れを検出するヘルスチェックを実装する。
