結論から言う。本番は `wan2.2-i2v-a14b-fp8` 一本に固定する。A14Bフル精度はRTX5090 32GBではoffload必須で1step目にsegfault、5BはVRAM 14GBに収まるが顔と手の崩れが約3割多い。14B fp8だけがVRAM 29.8GBでoffloadゼロかつ崩れ最少だった。以降の全章はこのモデル前提で進む。

## A14Bフル精度がRTX5090 32GBでsegfaultする再現コード

フル精度A14Bはモデルだけで約56GB。32GBに乗せるにはCPU offloadが要るが、offload境界で1step目に落ちる。以下が手元で100%再現したコマンドだ。

```bash
# offload有効でA14Bフルを起動 → 1step目でsegfault (exit 139)
python generate.py \
  --task i2v-A14B --ckpt_dir ./Wan2.2-I2V-A14B \
  --offload_model True --t5_cpu \
  --size 832*480 --frame_num 81
# => Segmentation fault (core dumped)
```

VRAMピークは落ちる直前で31.6GB。32GBの上限に張り付いた瞬間に死ぬので、フル精度は選択肢から外す。

## 5B/14B fp8のVRAMと生成秒数を実測した3構成ベンチ表

同一プロンプト・832x480・81フレーム・25stepで計測した結果が次の通り。

```python
bench = {
    # model            VRAM_GB  sec   崩れ率(体感)
    "A14B-full":       (None,   None, "segfault"),
    "5B":              (14.2,   38.0, "高(約30%)"),
    "14B-fp8":         (29.8,   52.0, "低(最少)"),
}
for k, (v, s, q) in bench.items():
    print(f"{k:12} VRAM={v} sec={s} 崩れ={q}")
```

5Bは速いが顔と手が破綻しやすくShortsのサムネで弾かれる。14B fp8は14秒遅い代わりにoffloadゼロで安定する。

## --fp8_e4m3fnフラグでVRAMを29.8GBに収める起動設定

14Bを32GBに乗せる鍵は重みのfp8量子化だ。`--fp8_e4m3fn` を付けるだけでVRAMが約42GB→29.8GBに落ち、offloadが不要になる。

```bash
python generate.py \
  --task i2v-A14B --ckpt_dir ./Wan2.2-I2V-A14B-fp8 \
  --fp8_e4m3fn \
  --offload_model False --t5_cpu False \
  --size 832*480 --frame_num 81 --sample_steps 25
# VRAM peak: 29.8GB / offloadなしで完走
```

offload Falseのまま完走できるのがfp8の価値で、フル精度との差はここに集約される。

## sageattention vs sdpaで1本あたり11秒差を測る

attention実装の差は無視できない。sageattentionに切り替えるとstepあたりが縮み、1本52秒→41秒になった。

```python
import time, torch
from diffusers import WanImageToVideoPipeline

for attn in ["sdpa", "sageattention"]:
    pipe = WanImageToVideoPipeline.from_pretrained(
        "./Wan2.2-I2V-A14B-fp8", torch_dtype=torch.float8_e4m3fn,
        attn_implementation=attn,
    ).to("cuda")
    t = time.perf_counter()
    pipe(image=img, num_frames=81, num_inference_steps=25)
    print(attn, round(time.perf_counter() - t, 1), "sec")
# sdpa 52.0 / sageattention 41.0
```

毎日10本回すなら110秒/日の短縮。0円運用では電気代に直結する差だ。

## 832x480と1280x720のOOM境界を確定する

解像度を上げるとVRAMが跳ねる。14B fp8でも1280x720はピーク32.4GBでOOMする。本番は832x480で固定し、必要なら後段でアップスケールする。

```yaml
# config/wan22.yaml — 全章共通の確定設定
model: wan2.2-i2v-a14b-fp8
fp8_e4m3fn: true
offload_model: false
attn: sageattention
size: "832*480"      # 1280*720はpeak32.4GBでOOM
frame_num: 81
sample_steps: 25
```

この `wan2.2-i2v-a14b-fp8` が以降の全章で唯一回す本番モデルになる。5Bへのフォールバックはしない。崩れ率とVRAM実測がそれを支持している。
