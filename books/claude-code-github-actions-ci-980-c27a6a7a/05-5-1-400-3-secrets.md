## ¥1,400の内訳｜無限ループ¥900・並列課金¥400・Secrets漏洩は寸前回避

溶かした内訳は無限ループ¥900・matrix並列課金¥400・Secrets漏洩は寸前で回避の3件だ。原因は全て「workflowの発火条件」と「権限設定」の詰めの甘さに集約される。事故は2026年4月の運用2ヶ月目に集中し、Anthropic APIの日次明細をCSVで突き合わせて特定した。

```bash
# Anthropic Consoleの利用明細CSVから事故日のコストを抽出
awk -F',' '$1 ~ /2026-04-1[2-5]/ {sum+=$4} END {printf "事故3日間の課金: $%.2f\n", sum}' usage.csv
# => 事故3日間の課金: $9.31 (約¥1,400)
```

## github.actorでbot自己コミットを除外｜無限ループ¥900を止めるpush発火制御

無限ループは「botのレビュー修正コミット → pushイベント再発火 → 再レビュー」の連鎖で起きた。47回連続発火し¥900を溶かした。`github.actor`がbot名と一致する場合にjobをスキップすれば連鎖は切れる。

```yaml
jobs:
  review:
    if: github.actor != 'claude-ci-bot[bot]'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
```

## concurrencyでmatrix多重起動をキャンセル｜並列課金¥400の重複レビュー対策

並列課金は3 OS × 2 Nodeのmatrix（6並列）が全てClaude APIを叩き、1PRで6回課金された結果だ。`concurrency`で同一PRの実行を1本に絞り、古い実行をキャンセルすれば¥400→¥67に下がる。

```yaml
concurrency:
  group: claude-review-${{ github.event.pull_request.number }}
  cancel-in-progress: true
```

レビューjob自体は`strategy.matrix`から外し、テストjobとは別ファイルに分離する。

## permissions最小化とadd-maskでSecrets漏洩を防ぐ最小権限設定

APIキーがデバッグログに出る寸前だった。`echo $ANTHROPIC_API_KEY`を消し、`permissions`を読み取り最小に絞り、`add-mask`でログ上のキーを伏字化する。これで漏洩経路を3つ同時に塞ぐ。

```yaml
permissions:
  contents: read
  pull-requests: write
steps:
  - run: echo "::add-mask::${{ secrets.ANTHROPIC_API_KEY }}"
```

## Anthropic spending limitと$20アラート｜無人運用へ移す上限設定

無人運用の前提は「暴走しても上限で止まる」状態だ。Anthropic Consoleで月額上限$20とアラート閾値$15を設定し、GitHub側でも日次コストを集計してSlack通知する。これで事故が起きても被害は上限額で頭打ちになる。

```yaml
  - name: コスト集計と上限アラート
    run: |
      COST=$(curl -s https://api.anthropic.com/v1/usage \
        -H "x-api-key: ${{ secrets.ANTHROPIC_API_KEY }}" | jq '.daily_cost_usd')
      awk -v c="$COST" 'BEGIN{if(c>5)exit 1}' || \
        curl -X POST "${{ secrets.SLACK_WEBHOOK }}" -d "{\"text\":\"日次$${COST} 超過\"}"
```

上限$20・アラート$15・日次$5の3段ガードを敷けば、第4章のworkflow.yml一式をそのまま無人運用へ移行できる。
