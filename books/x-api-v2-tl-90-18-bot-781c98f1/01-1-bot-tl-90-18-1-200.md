## 90分→18分:半年運用ログから取り出した実測値

このBotを6か月回したMacBook Air上の`tl_log.sqlite`から、1日あたりのTL閲覧時間を集計した結果が以下になる。導入前の中央値は90分、導入後は18分。80%減という数字はミュート辞書1,200語がノイズツイートをTLから物理的に消した効果だ。

```bash
sqlite3 tl_log.sqlite \
  "SELECT phase, ROUND(AVG(view_sec)/60,1) AS avg_min
   FROM session GROUP BY phase;"
# before|90.4
# after |18.2
```

誤ミュート率(本来読みたかった投稿を消した割合)は手動レビューで2.3%。この数字を5%以下に抑える閾値設計を第3章で扱う。

## tweepy + X API v2 Free枠で月¥0、取得上限の実測

X API v2のFree枠は2023年の改悪後、読み取りが月100投稿・書き込みが月500投稿まで縮小した。ミュート操作は書き込み(POST /2/users/:id/muting)に該当し、TL取得はFree枠では実質使えない。そこで本BotはホームTLをWeb側のセッションで読み、ミュート反映だけをAPIで叩く分業にしている。

```python
import tweepy, os
client = tweepy.Client(
    bearer_token=os.environ["X_BEARER"],
    consumer_key=os.environ["X_API_KEY"],
    consumer_secret=os.environ["X_API_SECRET"],
    access_token=os.environ["X_ACCESS"],
    access_token_secret=os.environ["X_ACCESS_SECRET"],
)
me = client.get_me().data.id  # Free枠でも月1回程度なら消費が小さい
print(me)
```

月額コストは¥0。書き込み500/月の上限に対し、1日のミュート登録は平均6.5件、月でも約200件で枠内に収まる実測値だった。

## 自動生成ミュート辞書1,200語の上位カテゴリ

辞書は手打ちではなく、非表示にしたツイートの形態素から自動抽出される。`mute_dict.json`の上位を頻度順に出すと、宣伝系・煽り系・スパムリンク系で全体の6割を占めた。

```python
import json, collections
d = json.load(open("mute_dict.json", encoding="utf-8"))
top = collections.Counter(d).most_common(5)
for word, cnt in top:
    print(f"{word}\t{cnt}")
# 拡散希望  214
# 副業     188
# 今だけ   141
# 当選     120
# LINE登録 97
```

カテゴリ分類と辞書の自動更新ロジックは第4章で組む。

## 4モジュール構成と最終的に手に入るスクリプト一式

完成形は4ファイルに分かれる。各モジュールが第2〜5章に対応し、章を読み終えるたびに1ファイルずつ動く状態になる。

```yaml
# pyproject.toml 抜粋 — 依存はこの3つだけ
[project]
dependencies = ["tweepy>=4.14", "playwright>=1.44", "janome>=0.5"]

# 構成
modules:
  collector.py:  # 第2章 PlaywrightでTL本文を収集
  scorer.py:     # 第3章 誤ミュート率2.3%を支える閾値判定
  dict_updater.py# 第4章 1,200語辞書の自動更新
  muter.py:      # 第5章 tweepyで枠内ミュート反映
```

`git clone`して`playwright install chromium`を打てば、第5章の終わりには自分のTLでこの90分→18分が再現できる。

## 第2章へ:Playwrightで取得上限を踏まないTL収集を組む

ここまでが完成形のスナップショットだ。次章ではFree枠の読み取り制限を回避するため、API取得ではなくPlaywrightでログイン済みセッションからTLを吸い出す`collector.py`を書く。レート制限を踏まないスクロール間隔(実測で平均1.8秒)と、収集18分のうち14分を占めるDOM解析を短縮するセレクタ設計まで踏み込む。自分のTLでこの数値を再現したい場合は、この先のモジュールが揃って初めて動くため、続けて手を動かしてほしい。
