## AIに「稼ぐ仕組み」を設計させるとはどういうことか

副業で稼ぐとき、多くの人は「何を売るか」から考え始める。だが本当に先に決めるべきは「誰が動かすか」だ。あなたが手を動かし続ける仕組みは副業ではなく、時給換算すれば割の悪いアルバイトに過ぎない。

本書が提案するのは、AIに**稼ぐ仕組みそのものを設計・実行させる**アプローチだ。あなたの役割は「仕様を与える人」であり、日々の生産・投稿・改善はClaudeやPythonスクリプトに委ねる。

---

## 自動化副業の全体地図

構造はシンプルに3層に分けられる。

```
[生成層]  Claude CLI → 記事/予想/商品説明を生成
[投稿層]  Python poster → Zenn/Qiita/Boothへ自動公開
[収益層]  アフィリリンク / 商品販売 / サービス受注
```

たとえば筆者の環境では、毎朝7時にタスクスケジューラが次のコマンドを実行する。

```powershell
python orchestrator/run_morning.py --date today
```

このスクリプトがClaude CLIを呼び出し、テーマ選定→記事生成→Zennへgit push→Blueskyへ告知、までを無人で完走する。起きたら「今日の投稿完了」のDiscord通知が届いている。

---

## 4つの収益チャネルと優先順位

自動化副業には大きく4つの入口がある。

1. **コンテンツ収益**：Zenn/Qiitaへの技術記事でビュー→アフィリ収益
2. **デジタル販売**：Booth/GumroadにAIツールやテンプレを自動出品
3. **サービス受注**：ランサーズ/ランサーズへのGIG出品→AI納品
4. **予測サービス**：競艇・競輪・競馬のAI予想をnoteで有料販売

初月は**サービス受注**を最優先にすることを強く勧める。理由は「分母が1」だからだ。コンテンツ収益は1万PVで数千円だが、サービス1件なら¥3,000〜¥30,000を即日回収できる。集客は「プラットフォーム持ち」のマーケットプレイスに任せ、あなたはAIで納品品質を担保するだけでよい。

---

## 最初の一歩：環境構築とClaude CLIの導入

以下の手順で今日中に土台が完成する。

```powershell
# 1. Claude Code CLIを導入
npm install -g @anthropic-ai/claude-code

# 2. プロジェクトフォルダを作成
mkdir auto-money && cd auto-money

# 3. 環境変数ファイルを作成
copy .env.example .env
# ANTHROPIC_API_KEY= を記入（Claude Maxプランなら定額）

# 4. 最初の記事を生成してみる
claude -p "Zenn向けのPython自動化入門記事を1500字で書いて"
```

Claude Maxプランであれば月額固定でAPIコストがかからない。つまり**生成コストゼロで記事を量産できる**のが、このシステムの核心だ。

---

## 設計の思想：「動かし続ける」より「止まらない」

自動化で最も大切なのは完璧な品質より**継続稼働**だ。1本の記事が多少荒削りでも、365日公開され続ける仕組みの方が圧倒的に価値が高い。

エラーで全体が止まらないよう、各チャネルは`.env`の設定がなければスキップする設計にする。一部が壊れても残りは動く。これを「疎結合」と呼ぶ。

第2章以降では、この地図の各ポイントを実装レベルで解説していく。まずは全体像を頭に入れた上で、自分のリソース（時間・資金・スキル）に合ったチャネルを1つ選んで動かし始めることが、最短の収益化ルートだ。
