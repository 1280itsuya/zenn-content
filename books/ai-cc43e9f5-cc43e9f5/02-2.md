## 収益フローを「設計」するという発想

副業で稼ぐには、作業を繰り返すのではなく、稼ぐ仕組みそのものを設計しなければならない。AIと自動化を使えば、一度作ったフローが毎日動き続け、あなたが寝ている間にも収益を積み上げる。本章では「ツール選定→フロー構築→収益化」の具体的な設計手順を解説する。

---

## ツール選定の三原則

自動化に使うツールは「無料・API対応・ドキュメントが豊富」の三つを軸に選ぶ。

- **LLM**：Claude（Anthropic）またはGPT-4o。Claude Maxプランなら月額定額でAPI無制限に近い運用が可能。
- **スクリプト言語**：Python 3.11。`pip install anthropic python-dotenv requests`で即スタートできる。
- **投稿先**：Zenn・Qiita・はてなブログ。いずれもトークン一枚でAPIから記事を公開できる。

環境変数は`.env`ファイルで一元管理し、コードにキーを書かない習慣を最初から徹底する。

```
ANTHROPIC_API_KEY=sk-ant-...
ZENN_GITHUB_TOKEN=ghp_...
```

---

## 自動化フローの三層構造

収益フローは**生成→投稿→計測**の三層で考える。

**① 生成層**：Claudeにテーマと構成を渡し、記事本文を生成させる。プロンプトにターゲットキーワードと文字数を明示するのがコツだ。

```python
import anthropic, os
from dotenv import load_dotenv

load_dotenv()
client = anthropic.Anthropic()

def generate_article(topic: str) -> str:
    msg = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=2000,
        messages=[{"role": "user", "content": f"テーマ「{topic}」で技術記事を1500字で書いてください。"}]
    )
    return msg.content[0].text
```

**② 投稿層**：生成した本文をZenn用リポジトリにコミットし、`git push`でそのまま公開する。GitHub Actionsでスケジュール実行すれば完全無人化できる。

**③ 計測層**：Zenn・QiitaのAPIでview数を毎日取得し、上位記事のタグ・テーマを次の生成プロンプトに還元する。「勝ちパターン」を自動で学習する増幅ループがここで完成する。

---

## 収益接続：読者を「出口」へ誘導する

記事を書くだけでは収益にならない。記事末尾に必ずCTA（行動喚起）を置く。

- **アフィリエイト**：Amazonアソシエイトで関連技術書のリンクを自動挿入
- **デジタル販売**：BoothやGumroadへ誘導し、テンプレート・ツールキットを販売
- **サービス案内**：自社サービスページへのリンクを記事フッターに常設

CTAの文言も生成時にClaudeに同時出力させると、記事本文と自然につながる。

---

## 最小構成で動かす

まず「週3記事・1チャネル・1CTA」という最小構成で動かす。ツールを増やすより、一本のフローを完走させる方が先だ。計測で数値が出たら、チャネルを増やし、テーマをA/Bテストし、投稿頻度を上げる。設計は「完成してから動かす」のではなく「動かしながら育てる」が鉄則である。
