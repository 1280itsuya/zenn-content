## 収益の「レバー」を特定する

自動化副業で月収を伸ばすには、まず「どこを増やせば収益が上がるか」を数値で把握することが先決だ。以下のコマンドでチャネルごとの投稿数・ビュー数・収益を一覧する。

```bash
python src/analytics/channel_report.py --days 30
```

出力例：

```
channel          posts  views  revenue
zenn_tech           12    840    ¥0
affiliate_blog      30     12   ¥180
ai_kit               4    230   ¥740
```

この数字を見れば「`ai_kit`は投稿4本で¥740、`affiliate_blog`は30本で¥180」という歴然とした差が分かる。レバーは投稿数ではなく**チャネル選択と単価**にある。

## スケールアップの3原則

**①勝ちチャネルへリソースを集中する**  
ビュー単価が高いチャネルの生成頻度を上げる。`config/agents.json`で`daily_limit`を引き上げるだけでよい。

```json
"ai_kit": { "daily_limit": 8, "enabled": true }
```

**②単価を上げる（バンドル販売）**  
単品¥980の商品を「3点セット¥2,480」としてGumroadに登録すると、同じ流入から2.5倍の収益になる。商品ページは自動生成できる。

```bash
python src/gumroad/bundle_publisher.py \
  --products mcp_kit llms_kit claude_pack \
  --price 2480 --title "Claude自動化フルセット"
```

**③勝ちテーマを次の記事へ還元する**  
ビュー上位記事のタグ・タイトル構造を抽出し、次サイクルのプロンプトに差し込む「増幅ループ」を組む。

```python
# winner_extractor.py の核心部分
top = sorted(articles, key=lambda a: a["views"], reverse=True)[:5]
winning_tags = Counter(t for a in top for t in a["tags"]).most_common(3)
```

抽出したタグをオーケストレーターが自動で次のプロンプトへ注入するため、サイクルを重ねるほどビューが伸びる設計になる。

## 収益最大化の「組み合わせ戦略」

スケールの本質は**チャネルを並列化してリスクを分散しながら単価を上げ続けること**だ。

- **無料コンテンツ（Zenn/Qiita）**：検索流入の入口。1記事あたりのコストはLLM呼び出し約¥2。
- **有料商品（Gumroad/Booth）**：流入を直接収益に変換。バンドル化で客単価を3倍以上にできる。
- **月額サービス（ストアケア/自動化代行）**：1契約で安定した月次収入。コンテンツ集客からサービス成約へのファネルを自動で繋ぐ。

この3層を同時に稼働させると、ビューが増えるほど有料転換も積み上がる複利構造が生まれる。

## 運用の自動化：タスクスケジューラへの登録

スケールアップした構成を毎朝確実に動かすには、WindowsタスクスケジューラにPythonスクリプトを登録する。

```powershell
$action = New-ScheduledTaskAction -Execute "python" `
  -Argument "C:\auto-money\orchestrator\run.py"
$trigger = New-ScheduledTaskTrigger -Daily -At "07:00"
Register-ScheduledTask -TaskName "AutoMoney" `
  -Action $action -Trigger $trigger -RunLevel Highest
```

これでPCを開かなくても毎朝7時に全チャネルが動き、レポートがDiscordに届く。収益レバーを特定し、勝ちパターンを自動増幅する仕組みを組んだ上でスケジューラに乗せる——この順序がスケールアップの正しい手順だ。
