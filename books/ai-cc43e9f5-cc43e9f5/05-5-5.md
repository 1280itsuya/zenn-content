## 月5万円は「拡張フェーズ」の入り口

月5万円を安定して稼げるようになった時点で、あなたのシステムは「動く」ことが証明されている。次の問いは「どれだけ速く倍にできるか」だ。ここで手を動かし続けるのは間違いで、正解は**システムにシステムを足す**ことだ。

---

## ステップ1：収益チャネルを縦に伸ばす前に「勝ちパターン」を特定する

闇雲に投稿数を増やしても収益は線形にしか伸びない。まず現状の数値を抽出する。

```bash
python src/winner_extractor.py --days 30 --top 10
```

このスクリプトはZenn・Qiitaのview数上位10記事からタグ・トピック・文字数の共通項を抽出し、`data/winning_patterns.json` に出力する。勝ちパターンが出たら、それを生成プロンプトに還元する。

```python
# config/agents.json の strategist セクションを更新
"exemplar_topics": winning_patterns["top_tags"][:5]
```

これだけで次サイクルから「当たりやすい記事」が自動的に増産される。

---

## ステップ2：横展開──収益チャネルを増やす

月5万円の構成を確認し、チャネルごとの粗利率を計算する。たとえばZenn記事（広告収益）の粗利は100%だが、記事代行（B2B）は作業時間が伴う。拡張の優先順位はこうなる。

1. **Booth自助販売**（完全無在庫、手数料のみ）
2. **Gumroad教材**（一度作れば永続収入）
3. **note有料記事**（既存記事の有料化）

例として、Booth商品を追加する場合は次の手順だ。

```bash
python src/booth_lister.py \
  --title "Claude×MCP導入キット" \
  --price 2480 \
  --description data/mcp_kit_desc.md
```

出品が完了したら、Zennの技術記事末尾に誘導CTAを自動挿入するよう `posters/zenn.py` のテンプレートを更新する。これで記事→商品販売のファネルが完成し、既存の流入資産がそのまま売上に転換される。

---

## ステップ3：複利化──勝ちチャネルに予算を再投資する

収益が出たら「時間を買う」再投資が最速の複利化手段だ。具体的には次の2点に絞る。

**① 投稿頻度の増加**
Claude CLI（Maxプラン定額）をフルに使い、1日12本だった記事生成を24本に増やす。

```bash
# cron で 7時・13時・19時の3回に分けて起動
0 7,13,19 * * * cd ~/auto-money && python orchestrator/main.py
```

**② 新規チャネルの自動開拓**
`src/channel_scout.py` を週次で走らせ、view数・競合密度・収益化難易度を自動スコアリングさせる。スコア上位のチャネルだけ翌月から追加する仕組みにすれば、判断コストがゼロになる。

---

## 月10万円への道筋

月5万円→10万円のジャンプに必要なのは「倍の努力」ではなく、**収益チャネル数 × 単価** の組み合わせ最適化だ。記事収益（¥1万）＋Booth販売（¥2万）＋記事代行1件（¥3万）＋note有料（¥1万）を並走させれば、どれかが落ちても月7万円の床が残る。自動化の真価は「一点集中リスクの排除」にある。システムが止まらない限り、収益の複利は続く。
