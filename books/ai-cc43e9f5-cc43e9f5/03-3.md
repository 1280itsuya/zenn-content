## 収益の「配管」とは何か

AIで記事を量産しても、読者がいなければ売上はゼロだ。決済リンクを用意しても、そこへ誘導する動線がなければ誰も買わない。自動投稿ツールが動いても、収益につながらなければただの自己満足に終わる。

収益化とは、**集客→コンテンツ→決済**という3本のパイプを水漏れなく接続する配管工事だ。本章ではその配管を実際に組み上げる。

## 集客パイプ：検索流入を自動で引き込む

最も安定した集客源はSEOだ。ZennやQiitaは独自ドメインの権威を持ち、書いた翌日に検索上位に入ることがある。

まず記事を自動生成するスクリプトを用意する。

```bash
python src/auto_qiita_tech.py --topic "Python非同期処理" --tags "Python,asyncio,初心者"
```

生成した記事はQiita APIでそのまま投稿できる。

```python
import requests, os
headers = {"Authorization": f"Bearer {os.environ['QIITA_TOKEN']}"}
payload = {"title": title, "body": body, "tags": tags, "private": False}
requests.post("https://qiita.com/api/v2/items", json=payload, headers=headers)
```

ポイントは**タグを実在する正規タグに限定**することだ。`#AI副業`のような造語タグは検索されない。`#Python`や`#機械学習`など、月間検索数の多いタグを3〜5個に絞る。

## 決済パイプ：GumroadとBoothを即日開通する

集客に成功しても、決済口が開いていなければ機会損失になる。無料で始められるGumroadは最速の選択肢だ。

1. Gumroad.comでアカウント作成
2. 「New Product」→デジタル商品（PDFやZIP）をアップロード
3. 価格を設定して商品URLを取得

取得したURLを記事末尾のCTAに埋め込む。

```python
CTA = f"\n\n---\n詳細な設定テンプレートはこちら→ {GUMROAD_URL}\n"
body_with_cta = article_body + CTA
```

この1行を投稿スクリプトに追加するだけで、全記事にアフィリリンクが自動挿入される。

## 自動投稿パイプ：Cronで毎朝動かす

3本のパイプを繋いだら、あとはスケジューラーに任せる。Windowsならタスクスケジューラー、LinuxならCronだ。

```
# 毎朝7時に記事生成→投稿を実行
0 7 * * * cd /path/to/project && python orchestrator.py >> logs/daily.log 2>&1
```

`orchestrator.py`の中では生成→検証→投稿の順に処理が流れる。品質ゲートを挟むことで、タイトルと本文が乖離した記事を自動排除できる。

```python
if not content_matches_title(title, body):
    logger.warning(f"テーマ不一致: {title} → スキップ")
    continue
```

## 配管完成後のチェックリスト

| 確認項目 | 方法 |
|----------|------|
| 記事が実際に公開されているか | QiitaのマイページURLをブラウザで確認 |
| CTAリンクが正しく機能するか | Gumroadの商品URLを実際にクリック |
| Cronが時刻通りに動いているか | `logs/daily.log`の最終更新時刻を確認 |
| 収益がダッシュボードに反映されているか | Gumroad管理画面を週1で確認 |

配管の9割は一度組めば自動で動き続ける。最初の1時間の設定投資が、毎日の手作業を消してくれる。
