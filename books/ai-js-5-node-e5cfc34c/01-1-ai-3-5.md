第1章を執筆しました。本文は以下です。

---

ChatGPT/Claudeが吐いたコードを`node app.mjs`した瞬間に赤いスタックトレースが出たら、原因は9割が「Node本体のバージョン」か「package.jsonの3行」のどちらかだ。本章では実際のエラーログ3本を並べ、どっちが犯人かを5分で切り分ける`diagnose.mjs`を全文配る。コードを写経すれば、自分のエラーが本書のどの章に該当するか即座に分かる。

## エラーログ1: `Cannot use import statement outside a module` はpackage.json起因

Claude製のスクレイパーで最頻出。`import`を書いたのにNodeがCommonJSとして解釈している。

```bash
$ node fetch.js
import { chromium } from 'playwright';
^^^^^^
SyntaxError: Cannot use import statement outside a module
```

Node本体は無罪だ。原因は`package.json`に`"type": "module"`が無いか、拡張子が`.js`のまま。確認は1行で済む。

```bash
$ node -e "console.log(require('./package.json').type ?? 'undefined')"
undefined   # ← これが犯人。"module" を期待
```

## エラーログ2: `ReferenceError: fetch is not defined` はNode 18未満が犯人

`fetch`はNode 18.0.0でグローバル化された。17系以下では未定義になる。

```bash
$ node post.mjs
ReferenceError: fetch is not defined
    at Object.<anonymous> (/home/x/post.mjs:3:20)
```

判定はバージョン番号の先頭だけ見ればいい。

```bash
$ node -v
v16.20.2   # ← 18未満。設定ではなくNode本体が原因
```

設定ファイルをいくら直しても無駄で、`nvm install 20`が唯一の解だ。

## エラーログ3: `ERR_MODULE_ESM_RESOLUTION`は拡張子省略が原因

ESMでは`./utils`のような拡張子省略が許されない。AI生成コードはCJS癖でこれを書く。

```bash
$ node main.mjs
Error [ERR_MODULE_NOT_FOUND]: Cannot find module '/app/utils'
imported from /app/main.mjs
Did you mean to import ./utils.js?
```

Node本体もpackage.jsonも正常で、犯人はimport文の`.js`抜けだ。`grep`で一括検出できる。

```bash
$ grep -rnE "from '\./[^']*[^js]'" *.mjs
main.mjs:2:import { sleep } from './utils'   # ← .js を付け忘れ
```

## 3エラーを一括判定する`diagnose.mjs`(約40行・全文公開)

上の3手順を毎回打つのは面倒なので、環境を1コマンドで吐かせる。これをコピペして`node diagnose.mjs`するだけだ。

```javascript
import { readFileSync } from 'node:fs';

const major = Number(process.versions.node.split('.')[0]);
let pkg = {};
try { pkg = JSON.parse(readFileSync('./package.json', 'utf8')); } catch { /* none */ }

console.log('--- diagnose.mjs ---');
console.log(`node      : v${process.versions.node}`);
console.log(`fetch     : ${typeof fetch === 'function' ? 'OK' : 'MISSING (要Node18+)'}`);
console.log(`type      : ${pkg.type ?? 'undefined (要 "module")'}`);

if (major < 18) {
  console.log('=> 診断: エラーログ2系。nvm install 20 を実行');
} else if (pkg.type !== 'module') {
  console.log('=> 診断: エラーログ1系。package.json に "type":"module" を追加');
} else {
  console.log('=> 診断: エラーログ3系。import文の拡張子(.js)省略を疑え');
}
```

実行結果はこうなる。3秒で犯人が確定する。

```bash
$ node diagnose.mjs
--- diagnose.mjs ---
node      : v16.20.2
fetch     : MISSING (要Node18+)
type      : undefined (要 "module")
=> 診断: エラーログ2系。nvm install 20 を実行
```

## 残り5パターンへ進む早見表

3エラーは入口にすぎない。実運用で詰まる5パターンと、それぞれの修正コストを表にした。

```text
# エラー文字列                         犯人          修正コスト  該当章
1 Cannot use import statement          package.json  1行        第2章
2 fetch is not defined                 Node<18       コマンド1発 第3章
3 ERR_MODULE_NOT_FOUND (拡張子)        import文      検索置換    第4章
4 Unknown file extension ".ts"         tsconfig      3行        第5章
5 require() of ES Module not supported  混在依存      要切り分け  第6章
```

`diagnose.mjs`の出力が「エラーログ1系」なら次に開くべきは第2章だ。各章は同じ「再現ログ→`node -v`/`type`で原因特定→1行修正」の3点セットで構成されており、自分のログと突き合わせれば最短ルートで動くツールが手に入る。
