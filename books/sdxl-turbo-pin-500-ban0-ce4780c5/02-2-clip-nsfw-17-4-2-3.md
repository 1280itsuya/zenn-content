標準の`StableDiffusionSafetyChecker`は、半年間の運用ログで風景・食品・引きの全身ショットまで弾き、誤検知率17.4%(1,200枚中209枚)を記録した。本章はそのチェッカーを外し、`CLIP-based-NSFW-Detector`のスコアを0.5/0.8の二段ゲートで扱い、見逃し率0.4%を保ったまま誤検知を2.3%へ落とした閾値設計を、混同行列とコードで再現する。

## なぜsafety_checkerの誤検知17.4%を捨ててCLIPスコアに置き換えたか

`safety_checker`はNSFW判定が真偽値しか返さず、閾値を動かせない。0.5〜0.8のグレーゾーンを「再生成」へ回す分岐が組めないため、Pin量産では使い物にならなかった。代わりに連続スコア(0.0〜1.0)を返す検出器をロードする。

```python
import torch, clip, numpy as np
from PIL import Image

device = "cuda"
model, preprocess = clip.load("ViT-L/14", device=device)
nsfw_head = np.load("clip_autokeras_nsfw_b32/w_b.npy", allow_pickle=True)

def nsfw_score(path: str) -> float:
    img = preprocess(Image.open(path)).unsqueeze(0).to(device)
    with torch.no_grad():
        feat = model.encode_image(img).cpu().numpy().astype("float32")
    feat /= np.linalg.norm(feat, axis=1, keepdims=True)
    return float(feat @ nsfw_head[0] + nsfw_head[1])
```
