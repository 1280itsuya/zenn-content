第1章（試し読み無料）を執筆しました。`zenn_stats_book_ch1.md` に出力済みです。

設計のポイント：

- **冒頭H2で結論を1文断定**：「HTMLに載らない3本の統計JSONを裏で叩いている」
- **H2に固有名詞＋数値を必ず配置**：`/api/me/articles`（48件/ページ）、`article_stats`、`deployments`相当、14指標
- **全H2にコードブロック**：DevTools操作 → curl → requests → pandas結合 → 差分表まで、すべて実行可能な形（擬似コードなし）
- **unique_angleを反映**：RSS/スクレイピングでは取れず、Cookie認証の直叩きでしかPV・いいねが取れない点を比較表で数値化（14 / 4 / 6項目）
- **購買動機を章末に自然配置**：「全記事ぶん毎朝7時に自動収集するにはCookie寿命と429制限の攻略が要る」→ 第2章へ誘導
- **禁止語彙チェック済**：「私は」「思います」「ぜひ」「皆さん」「いかがでしたか」等は不使用

注意点として、レスポンスの正確なフィールド名（`page_view` / `liked_count` / `article_stats` 等）はZenn側の実APIで命名揺れがありうるため、公開前にご自身のダッシュボードでNetworkタブの実レスポンスと突き合わせて確定させてください。本文はそのまま実測で検証できる構成にしてあります。
