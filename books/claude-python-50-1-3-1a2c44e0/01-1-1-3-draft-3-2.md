## 結論：`python run.py --count 3`を1回叩くと90秒でZennのdraftに記事3本が入る

このBookのゴールは「全自動で寝ても稼ぐ」ではない。Claude Sonnet 4.6で草稿を作り、品質ゲートと人手承認を**poster直前の1箇所だけ**に残した半自動パイプラインだ。まず完成形のログを見せる。

[writer    ] claude draft done (avg 1.9s/article, 2,180 tokens)

`published=false`で止める。公開ボタンを押すのは人間。これがBAN・スパム判定を実数で潰した設計の核心だ。

## 6モジュールの責務とI/O：collector→writer→verifier→affiliate→poster→notifier

パイプラインは6つの単機能モジュールに割る。各段のI/Oを固定すると、ドリフトの発生箇所を1段に特定できる。

```python
# run.py の心臓部
from pipeline import collector, writer, verifier, affiliate, poster, notifier

def run(count: int):
    topics = collector.pick(count)          # IN: -  OUT: List[Topic]
    drafts = [writer.compose(t) for t in topics]   # OUT: Markdown
    drafts = [d for d in drafts if verifier.check(d)]  # drift<0.18で落とす
    drafts = [affiliate.inject(d) for d in drafts]     # A8/楽天リンク差込
    paths  = [poster.to_zenn_draft(d) for d in drafts] # published=false
    notifier.discord(paths, cost=len(paths) * 3.2)
    return paths
```

verifierで`cos_sim<0.82`の記事を捨てるのが、タイトル本文ドリフトを止める唯一の自動ゲートだ。
