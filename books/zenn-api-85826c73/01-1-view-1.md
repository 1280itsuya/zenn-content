# 第1章 完成形デモ: 全記事のview推移を1枚で見る最終ダッシュボード

結論から言うと、本書を読み切ると、Zennの全Book・全記事のview / like / コメント / フォロワーを**日次の折れ線で1画面**に描き、GitHub Actionsが毎朝7時に自動更新するダッシュボードが手に入る。Zennには公式統計APIが存在せず、ダッシュ画面を毎日開いて30本分を手で写経すると約20分/日 = 月10時間が溶ける。この章ではその完成形を先に見せ、なぜ非公開APIに踏み込むのかを示す。

## 完成イメージ: Next.js 14 + Recharts の view 推移グラフ

最終成果物のコア部分はこれだけだ。`/api/stats` が返す日次配列を `LineChart` に流し込み、30本でも1枚に収める。

```tsx
// app/components/ViewTrend.tsx
'use client';
import { LineChart, Line, XAxis, YAxis, Tooltip } from 'recharts';

export function ViewTrend({ data }: { data: { date: string; views: number }[] }) {
  return (
    <LineChart width={880} height={320} data={data}>
      <XAxis dataKey="date" /><YAxis />
      <Tooltip />
      <Line type="monotone" dataKey="views" stroke="#3ea8ff" dot={false} />
    </LineChart>
  );
}
```

Zennブランドカラー `#3ea8ff` でview推移を描くと、どの記事がいつ伸びたかが一目で分かる。

## リポジトリ構成: data / scripts / app の3層分離

完成リポジトリは「収集(Python)」「保存(JSON)」「描画(Next.js)」を疎結合にしている。第2章以降はこの各ディレクトリを順に埋める作業になる。

```bash
zenn-dashboard/
├── scripts/
│   ├── login.py          # 第3章: Playwrightでログイン→Cookie取得
│   └── collect.py        # 第4章: 非公開API /api/dashboard を叩く
├── data/
│   └── 2026-06-02.json   # 日次スナップショット(gitに追記)
├── app/                  # 第6章: Next.js + Recharts
└── .github/workflows/
    └── daily.yml         # 第7章: 毎朝7時 cron
```

`data/` に日付別JSONを追記し続ける設計なので、過去差分から「前日比view」も後から復元できる。

## 取得する非公開エンドポイントの実測レスポンス

ダッシュボードの全数値は、ログイン後にしか叩けない非公開エンドポイント1本から来る。第4章で特定手順を解説するが、レスポンス形はこうだ。

```python
# scripts/collect.py (抜粋)
import requests
r = requests.get(
    "https://zenn.dev/api/dashboard/articles",
    cookies=cookies,  # 第3章で取得したログインCookie
)
for a in r.json()["articles"]:
    print(a["title"], a["page_view_count"], a["liked_count"])
# => Zenn統計を非公開APIで取得... 1342 87
```

`page_view_count` が公開画面には出ない総view数で、これを毎日保存することが本書の心臓部だ。

## 手作業集計が破綻する数値: 30本 × 20分 = 月10時間

なぜ自動化するのか。手集計の限界を数字で出す。記事30本を1本40秒で転記しても20分、土日も休めば1.7時間/週が蒸発する。

```python
articles, sec_per_article = 30, 40
daily_min = articles * sec_per_article / 60     # 20.0 分
monthly_h = daily_min * 30 / 60                 # 10.0 時間
print(f"{daily_min:.0f}分/日 → {monthly_h:.0f}時間/月")
# => 20分/日 → 10時間/月
```

月10時間は、新しいBookを1冊書ける時間だ。自動化の投資回収は初月で終わる。

## 毎朝7時に自走させる GitHub Actions

最後のピースは無人運用だ。`schedule` で毎朝7時(JST=UTC22時)に収集スクリプトを回し、JSONをコミットする。

```yaml
# .github/workflows/daily.yml
on:
  schedule:
    - cron: '0 22 * * *'   # JST 07:00
jobs:
  collect:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: pip install -r requirements.txt
      - run: python scripts/collect.py
        env:
          ZENN_EMAIL: ${{ secrets.ZENN_EMAIL }}
          ZENN_PASS: ${{ secrets.ZENN_PASS }}
      - run: |
          git add data/ && git commit -m "stats $(date +%F)" && git push
```

ここまでが完成形だ。第2章ではChrome DevToolsで非公開エンドポイントと認証Cookieを実測特定し、第3章でPlaywrightによるログイン自動化、第4章で`/api/dashboard`の全フィールドを叩き切る。`secrets.ZENN_PASS`をどう安全に渡すか、Cookieの有効期限切れにどう自動再ログインで対処するか――公式APIが無いZennで毎朝7時に止まらず動かす実装の全コードは、次章以降で1行ずつ組み上げていく。
