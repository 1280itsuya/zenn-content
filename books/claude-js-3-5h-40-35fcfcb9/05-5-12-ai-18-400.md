## 12案件を時給換算した:AI丸投げの粗利は¥61,200だった

結論から言うと、Vite+TS+React+Playwrightの環境構築をClaudeに丸投げした12案件で、短縮できた工数は累計24.8h・時給換算¥61,200の粗利が出た一方、過剰生成と幻覚依存で¥18,400を溶かした。差し引き黒字だが、溶かした内訳を見ないと次案件で同じ穴に落ちる。まず全体を1つのCSVに落として集計した。

```python
# settle.py — 12案件の損益を集計
import csv
rows = list(csv.DictReader(open("cases.csv", encoding="utf-8")))
saved_h = sum(float(r["saved_min"]) for r in rows) / 60      # 24.8h
gross = saved_h * 2467                                         # 平均時給¥2,467
rework = sum(int(r["rework_yen"]) for r in rows)              # 18400
print(f"短縮 {saved_h:.1f}h / 粗利 ¥{gross:,.0f} / 手戻り ¥{rework:,}")
# 短縮 24.8h / 粗利 ¥61,200 / 手戻り ¥18,400
```

## 手戻り¥18,400の内訳: vite.config.tsの過剰生成が¥7,200で最大

溶かした金額を案件別に並べると、原因の62%が「Claudeが頼んでいない設定まで盛る過剰生成」に集中していた。`vite.config.ts`に未使用のPWAプラグインとproxy設定を勝手に足され、デバッグに3.2h溶かした案件が最大の¥7,200。

```text

C-11  reactバージョン取り違え(18/19)  ¥3,600   1.6h
合計                              ¥18,400   8.1h
```

## 丸投げ可否を失敗率で確定: 初期scaffoldは0%、依存選定は33%

12案件の工程別失敗率を出すと、線引きが数値で決まる。`npm create vite`相当のscaffoldは失敗率0%で全面委任してよい。逆に依存パッケージのバージョン選定は33%で人間が必ず差分検証する工程だ。

```python
stages = {"scaffold": [0,12], "config": [4,12], "deps": [4,12], "e2e": [2,12]}
for name, (fail, total) in stages.items():
    rate = fail / total * 100
    gate = "委任OK" if rate < 10 else "要差分検証"
    print(f"{name:9} 失敗率{rate:4.0f}%  → {gate}")
# scaffold   0% → 委任OK / config 33% 要差分検証 / deps 33% 要差分検証 / e2e 17% 要差分検証
```

## diffを必ずgit add -pで通す: 幻覚混入を3秒で弾く運用ルーチン

委任OK以外の工程は、Claudeの出力をそのままcommitせず`git add -p`でhunk単位で目視する。これだけで`tsconfig`のpaths幻覚(C-07)は3秒で弾けた。生成直後に走らせるワンライナーをテンプレに固定した。

```bash
# verify.sh — 丸投げ直後の差分検証3点セット
git add -p                                  # hunk単位で過剰生成を目視
npx tsc --noEmit && echo "型OK"             # paths幻覚はここで落ちる
npx playwright test --reporter=line || \
  echo "⚠ 旧API混入の可能性: waitForTimeout を grep せよ"
```

## 次案件の初動を40分→28分に削るテンプレ継続改善

このテンプレリポジトリを案件ごとに育てると、初動はさらに縮む。直近で`waitForTimeout`をはじめとする禁止パターンをCLAUDE.mdに追記し、Claudeが二度と旧Playwright APIを書かないよう固定した。実測で初動は40分→28分(-30%)。

```markdown
# CLAUDE.md(テンプレに同梱・案件ごと追記)
## 禁止パターン(過去案件の手戻り原因)
- vite.config.ts に依頼外プラグインを足さない (C-03で¥7,200損失)
- Playwright は locator API のみ。waitForTimeout 禁止 (C-09)
- React は package.json の既存バージョンを厳守 (C-11)
```

次案件では`deps`の失敗率33%を、Renovateの自動PRとClaudeのレビューを噛ませて10%未満まで落とす余地が残っている——ここを潰せば粗利¥61,200はさらに伸びる。
