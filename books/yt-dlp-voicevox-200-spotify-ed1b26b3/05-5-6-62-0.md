## VOICEVOXパラメータ変更前後の離脱率比較: speed_scale 1.0への切り替えで8.7pt改善

| 設定 | 週平均完聴率 | 30秒以内離脱率 | 計測週数 |
|---|---|---|---|
| speed_scale=**1.2**, intonation=1.0 | 53.1% | 28.4% | 7週 |
| speed_scale=**1.0**, intonation=1.0 | 61.8% | 17.2% | 10週 |
| speed_scale=**1.0**, intonation=1.3 | 62.4% | 16.9% | 9週 |

`intonation_scale`の1.3への引き上げは誤差範囲。支配的な変数は読み上げ速度だった。本番パラメータをそのまま貼る。

## Spotify Audience Network: 月1万リスナーが閾値・現状の79倍という現実

収益化の計算を正直に示す。

```python
# monetization_check.py
PEAK_WEEKLY_LISTENERS = 127        # 第23週の実測値
MONTHLY_LISTENERS_ESTIMATE = PEAK_WEEKLY_LISTENERS  # 月次集計での近似値

SPOTIFY_AN_THRESHOLD = 10_000      # Audience Network申請条件(月間リスナー)
SPOTIFY_CPM_USD = 15               # 推定CPM(上限値)
USD_TO_JPY = 155

gap = SPOTIFY_AN_THRESHOLD / MONTHLY_LISTENERS_ESTIMATE
revenue_at_threshold = (SPOTIFY_AN_THRESHOLD / 1000) * SPOTIFY_CPM_USD * USD_TO_JPY

print(f"現状の不足倍率: {gap:.0f}x")               # 79x
print(f"閾値到達時の月収上限: ¥{revenue_at_threshold:,.0f}")  # ¥23,250
```

月2.3万円の上限に到達するために79倍の規模が必要という事実は隠さない方がいい。Spotifyのストリーミング広告で月3万を狙うなら月2万リスナーが現実的な目標で、現状比157倍になる。Podcast単体での短期収益化は構造上難しい。

## Claude APIでエピソードイントロを動的生成する拡張コード

現状の課題は定型イントロによる最初30秒の離脱率(16.9%)で、ここをさらに削れる余地がある。Claude APIで記事ごとに異なるイントロを生成し、VOICEVOXで合成してMP3の先頭に結合する。

```python
# claude_intro_gen.py
import anthropic, subprocess, tempfile, os, json, requests

client = anthropic.Anthropic()

def generate_intro(article_text: str, episode_num: int) -> str:
    resp = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=200,
        messages=[{
            "role": "user",
            "content": (
                f"以下の技術記事を、Podcastの冒頭30秒（口語・体言止め禁止）に変換。"
                f"『第{episode_num}回』で始め、記事中の具体的な数値か手順名を1個必ず入れること。\n\n"
                f"{article_text[:800]}"
            )
        }]
    )
    return resp.content[0].text

def synth_wav(text: str, out_path: str, params: dict) -> None:
    q = requests.post("http://localhost:50021/audio_query",
                      params={"text": text, "speaker": 3}).json()
    q.update(params)
    wav = requests.post("http://localhost:50021/synthesis",
                        params={"speaker": 3},
                        data=json.dumps(q)).content
    with open(out_path, "wb") as f:
        f.write(wav)

def prepend_intro(intro_wav: str, body_wav: str, output_mp3: str) -> None:
    subprocess.run([
        "ffmpeg", "-y",
        "-i", f"concat:{intro_wav}|{body_wav}",
        "-acodec", "libmp3lame", "-b:a", "128k",
        output_mp3
    ], check=True)

# 使用例
PARAMS = {"speedScale": 1.0, "intonationScale": 1.3,
          "pitchScale": 0.0, "volumeScale": 1.0,
          "prePhonemeLength": 0.1, "postPhonemeLength": 0.1}

intro_text = generate_intro(open("article.txt").read(), episode_num=27)
synth_wav(intro_text, "/tmp/intro.wav", PARAMS)
synth_wav(open("body.txt").read(), "/tmp/body.wav", PARAMS)
prepend_intro("/tmp/intro.wav", "/tmp/body.wav", "ep027.mp3")
```

`generate_intro`の出力は毎回記事の核心数値を拾うため、「今回は〇〇という手順で〇〇を実現した」という具体フックが自動で入る。30秒以内離脱率が16.9%からさらに下がるかどうかは第7章のABテスト設計で検証する。
