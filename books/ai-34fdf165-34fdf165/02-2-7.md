## 第2章　自動化の設計図：稼げる仕組みを最短でつくる7ステップ

自動化の失敗パターンはほぼ共通している。「とりあえずスクリプトを書いてみる」から始め、気づけば動かない処理の山だけが残る。稼げる自動化には設計図が要る。ここでは最短で収益に繋がる7つのステップを順に解説する。

---

**ステップ1：収益ゴールを数値で決める**

「稼ぎたい」では動けない。「月3万円／1記事あたり広告クリック単価5円／月6,000PV必要」と逆算する。目標が数値になれば、何を自動化すべきかが見える。

**ステップ2：ボトルネックを1つ特定する**

全部を自動化しようとすると全部が中途半端になる。まず手作業の中で最も時間を奪っている工程を1つ選ぶ。例えば「毎日のブログ投稿タイトル考案に30分かかっている」なら、そこだけを狙い撃ちにする。

**ステップ3：最小構成でプロトタイプを動かす**

完璧なシステムより、動く粗削りなものを優先する。Claude APIで記事タイトルを生成するだけなら10行で足りる。

```python
import anthropic
client = anthropic.Anthropic()
msg = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=256,
    messages=[{"role": "user", "content": "節約をテーマにブログタイトルを5つ提案して"}]
)
print(msg.content[0].text)
```

まずこれを動かす。完成形は後でいい。

**ステップ4：出力を「投稿できる形式」に整える**

生成テキストをそのまま使えるケースは少ない。Markdown形式で出力させる、文字数を制限する、タグを自動付与するといった後処理を組み込む。この工程を省くと「生成はできるが公開できない」状態が続く。

**ステップ5：スケジューラに組み込む**

手動で実行するうちは自動化ではない。WindowsならタスクスケジューラにPythonスクリプトを登録し、毎朝7時に動かす。

```
schtasks /create /tn "AutoBlog" /tr "python C:\auto\run.py" /sc daily /st 07:00
```

これで人間が寝ていても処理が走る。

**ステップ6：失敗を検知・通知する**

自動化が「静かに死んでいる」状態は最も危険だ。例外をキャッチしてDiscordのWebhookに通知を飛ばすだけで、問題を翌朝ではなく即座に把握できる。エラーログをファイルに書き出す処理も必ずセットで入れる。

**ステップ7：計測して改善ループを回す**

投稿数・PV・クリック数を毎日CSVに記録する。1週間分のデータが溜まれば、どのカテゴリが読まれているかが分かる。その結果をプロンプトに反映させることで、生成品質が時間とともに上がっていく。これが「稼ぎ続ける仕組み」の正体だ。

---

7ステップを貫く原則は「動かして計測してから改善する」だ。設計に時間をかけすぎず、まず最小限を動かす。収益はその先にある。
