## 自動化パイプラインとは何か

収益を「作る」から「流れさせる」に発想を変えると、月収の天井が消える。自動化パイプラインとは、コンテンツ生成・投稿・収益回収を一本の流れに繋いだ仕組みだ。手動作業が介在するたびに流れは止まり、収益は頭打ちになる。

## パイプラインの3レイヤー構造

効率的なパイプラインは3層で設計する。

**生成層**：LLMがテーマを受け取り、記事・商品説明・SNS投稿文を出力する。  
**投稿層**：各プラットフォームのAPIへ自動送信する。  
**収益層**：アフィリリンク・有料コンテンツへの誘導を埋め込む。

この3層を分離して実装すると、どこかが壊れても他の層は動き続ける。

## 実装の最小セット

Pythonで最小構成を作るなら以下の流れになる。

```bash
# 依存インストール
pip install anthropic python-dotenv requests

# 環境変数を設定
echo "ANTHROPIC_API_KEY=sk-ant-xxx" >> .env
```

```python
import anthropic, os
from dotenv import load_dotenv
load_dotenv()

client = anthropic.Anthropic()

def generate_article(theme: str) -> str:
    msg = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=1500,
        messages=[{"role": "user", "content": f"{theme}について800字の解説記事を書いて"}]
    )
    return msg.content[0].text
```

生成した本文をそのままZenn/Qiitaの投稿APIへ渡す。投稿関数を差し替えるだけで配信先が増えるのがレイヤー分離の恩恵だ。

## スケジュール実行で「寝ている間に動かす」

Windows Task Schedulerに登録すれば、毎朝7時に自動起動できる。

```powershell
schtasks /create /tn "AutoBlog" /tr "python C:\auto-money\orchestrator\main.py" `
  /sc daily /st 07:00 /ru SYSTEM
```

夜間に記事を生成し、朝に投稿・夕方に成果を確認するというリズムを組むと、本業を持つ人でも無理なく回せる。

## 詰まりやすい3つの落とし穴

**①ハング問題**：外部API待ちでプロセスが止まる。`subprocess`のtimeoutを必ず設定し、ログに書いて早期検知する。

**②クレデンシャル漏洩**：APIキーをソースにハードコードするとGit pushで流出する。`.env`ファイルは必ず`.gitignore`に入れること。

**③テンプレート汚染**：プロンプトが固定文言のまま量産すると、検索エンジンにスパム判定される。毎回テーマと実例を差し込む変数化が必須だ。

## 運用指標で改善を回す

パイプラインは作って終わりではない。週次でこの3指標を確認する。

| 指標 | 目標 |
|------|------|
| 投稿成功率 | 95%以上 |
| 記事あたりPV | 前週比+5% |
| CTA経由クリック数 | 月次累計で右肩上がり |

数値が動かない記事のタグ・タイトルを変えてA/Bテストし、勝ちパターンをプロンプトに還元する。このフィードバックループが「自動化が勝手に賢くなる」状態を作る。

設計→実行→計測→改善の4ステップを週1サイクルで回せれば、3ヶ月後には収益の土台が出来上がっている。
