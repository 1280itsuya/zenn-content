## 収益を生む自動化ワークフローの設計と実装

自動化の失敗パターンの多くは「何かを自動化した」で終わることだ。収益に直結しない作業を効率化しても、手間が減るだけで売上は増えない。本章では、**「収益ゴールから逆算してワークフローを設計する」**という原則を軸に、実装まで一気に解説する。

## 収益逆算でワークフローを定義する

まず「何が収益を生むか」を特定する。たとえば技術ブログであれば、PV数ではなくアフィリエイトクリックや有料コンテンツへの誘導が収益源だ。以下の順で分解する。

1. **収益イベント**を特定する（例：Gumroadでの購入、アフィリリンククリック）
2. そのイベントを起こす**直前アクション**を洗い出す（例：記事を読む、CTA を見る）
3. 直前アクションを増やす**量産タスク**を自動化する（例：SEO 記事の生成・投稿）

このように「収益 → 行動 → タスク」の順で遡ることで、自動化の優先度が明確になる。

## 基本ワークフローの実装

Python と Claude API で、最小構成の収益ワークフローを実装する。

```python
# workflow.py
import anthropic, os, pathlib

client = anthropic.Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

def generate_article(topic: str) -> str:
    msg = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=2048,
        messages=[{
            "role": "user",
            "content": f"次のトピックで1000字のZenn技術記事を書いてください：{topic}"
        }]
    )
    return msg.content[0].text

def run_daily(topics: list[str]):
    for topic in topics:
        article = generate_article(topic)
        pathlib.Path(f"articles/{topic[:20]}.md").write_text(article)
        print(f"✓ 生成完了: {topic[:20]}")

if __name__ == "__main__":
    run_daily(["Claude APIで副業を始める方法", "MCPサーバー入門"])
```

これをタスクスケジューラ（Windows）またはcron（Linux）で毎朝7時に実行する。

```
# cron 設定例
0 7 * * * cd ~/auto-money && python workflow.py >> logs/daily.log 2>&1
```

## 品質ゲートで"空回り"を防ぐ

生成した記事をそのまま投稿すると、テンプレ感満載の低品質コンテンツになりやすい。**LLM-as-Judge**で自動検証を挟む。

```python
def quality_check(article: str, topic: str) -> bool:
    result = client.messages.create(
        model="claude-haiku-4-5-20251001",
        max_tokens=64,
        messages=[{
            "role": "user",
            "content": f"記事がトピック「{topic}」と一致し、具体例を含むか。YESまたはNOのみ答えよ。\n\n{article[:500]}"
        }]
    )
    return "YES" in result.content[0].text
```

チェックを通過した記事だけを投稿キューに入れることで、ペナルティリスクを下げながら量産できる。

## 投稿・計測・改善のループを回す

ワークフローは「生成 → 投稿 → 計測 → 改善」の4ステップで循環させる。Zennやdev.toは投稿APIを持つため、PVデータを取得して**ビュー数の多いトピック軸で次の記事を生成**する閉ループが作れる。この増幅ループこそが、単なる自動化と「収益を生む自動化」の決定的な差だ。
