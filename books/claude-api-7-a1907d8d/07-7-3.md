## 第7章：月3万円到達後にやること——パイプラインの拡張ロードマップとコスト最適化

月3万円を超えると、パイプラインは「趣味の実験」から「小さな事業」に変わる。この段階でやるべきことは、収益を10万円へ伸ばす施策と、スケールに伴うコスト増を抑える最適化の二本柱だ。

---

## コスト最適化：まず手を付けるべき3点

### 1. Prompt Cachingを有効化する

同じシステムプロンプトを繰り返し送っているなら、Anthropicの**Prompt Caching**が即効性ある。キャッシュヒット時のinputトークンコストは通常の10分の1以下になる。

```python
response = anthropic.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    system=[
        {
            "type": "text",
            "text": LONG_SYSTEM_PROMPT,
            "cache_control": {"type": "ephemeral"}  # ここを追加するだけ
        }
    ],
    messages=[{"role": "user", "content": user_input}]
)
```

**落とし穴**：キャッシュTTLは5分。バッチ処理でリクエスト間隔が開くとキャッシュが切れ、逆にキャッシュ書込コスト（通常の1.25倍）だけ取られる。連続実行する設計にすること。

### 2. Haiku→Sonnetの使い分けを徹底する

記事の下書き・タグ抽出・要約など「精度より速度」のタスクはHaiku 4.5に任せる。コストは約20分の1。

```python
def choose_model(task_type: str) -> str:
    heavy = {"final_edit", "quality_check", "client_article"}
    return "claude-sonnet-4-6" if task_type in heavy else "claude-haiku-4-5-20251001"
```

月3万円規模だとAPI代が月3,000〜8,000円になり始める。ここを半分にするだけで利益率が大きく改善する。

### 3. 実行環境をローカルからクラウドへ移す

自宅PCでTask Schedulerを回している場合、PCの電源断・スリープでパイプラインが止まる。月5万円を目指す前にサーバー移行が必須だ。

| 選択肢 | 月額目安 | 向いているケース |
|---|---|---|
| AWS EC2 t3.micro | ¥1,200〜 | 24h常駐・複数チャンネル |
| GitHub Actions | ¥0〜 | 日次バッチ・軽量タスク |
| Railway / Render | ¥700〜 | デプロイ手間を省きたい |

**AWS移行の最小手順**：

```bash
# 1. EC2にSSH後、依存を入れる
pip install anthropic python-dotenv

# 2. .envはSSMパラメータストアで管理（平文ファイルを置かない）
aws ssm get-parameter --name "/auto-money/ANTHROPIC_API_KEY" --with-decryption \
  --query Parameter.Value --output text

# 3. systemdでオーケストレーターを常駐化
sudo systemctl enable auto-money-pipeline
sudo systemctl start auto-money-pipeline
```

## 10万円へ向けた拡張ロードマップ

### ステップ1：チャンネルを並列化する（月3〜5万円）

単一チャンネルが安定して月3万円を出せた段階で、同じパイプラインを別ジャンルに横展開する。コードの変更はプロンプトとポスター設定のみ。

```
channels/
  tech_blog/      # 既存
  finance_blog/   # 横展開1（プロンプトを差し替えるだけ）
  english_dev/    # 横展開2（Dev.to向け英語チャンネル）
```

### ステップ2：勝ちパターンを計測して増幅する（月5〜8万円）

闇雲に量産しても伸びない。view数・クリック率が高かった記事のタグ・構成・冒頭パターンをfeedbackとしてシステムプロンプトへ還元する。

```python
# winner_extractor.py の要点
top_articles = get_top_articles(platform="zenn", limit=10)
winning_tags = extract_common_tags(top_articles)
winning_hooks = [a["intro"][:100] for a in top_articles]
# これをシステムプロンプトのexemplarsセクションに挿入
```

### ステップ3：サービス1件で月額収益を積む（月8〜10万円）

記事量産だけでは天井が低い。月額サブスク（MEO返信代行・AI導入支援など）を1件でも受注すると、その月の最低保証収益になる。パイプラインが作れた技術力は、そのままB2Bサービスに転用できる。
