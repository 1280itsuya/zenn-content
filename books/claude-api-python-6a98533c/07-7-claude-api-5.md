## 第7章　Claude APIパイプラインを副業として月5万円に伸ばすロードマップ

---

## パイプライン完成直後が「最も危険な時期」

自動生成が動くと、つい「あとは待つだけ」と思いがちだ。だが現実は違う。記事を量産しても、流入経路がなければ収益はゼロのままになる。筆者は13サイクル分のコンテンツを自動公開したが、収益が動いたのは「どこから人が来るか」を設計し直した後だった。

本章では、月5万円到達までを **3フェーズ**に分けて整理する。

---

## フェーズ1（1〜2週目）：流入経路を1本つなぐ

最初にやるべきことは、記事が人の目に触れる経路を「1本だけ」確実に動かすことだ。ZennやQiitaなど、プラットフォーム側が検索インデックスを持つ場所を選ぶ。

```python
# zenn_poster.py の最小構成
import os
from dotenv import load_dotenv

load_dotenv()  # ← これを省くと.envが読まれず静かにスキップされる

ZENN_PAT = os.getenv("ZENN_GITHUB_PAT")
if not ZENN_PAT:
    raise RuntimeError("ZENN_GITHUB_PAT が未設定。.envを確認してください")
```

**落とし穴：** `load_dotenv()` を書き忘れると、エラーも出ずに投稿がスキップされる。パイプライン内の各ファイルに必ず記述すること。

---

## フェーズ2（3〜4週目）：勝ちテーマを自動増幅する

実際のview数が取れたら、上位記事のタグとテーマを次のサイクルに還元する仕組みを入れる。

```python
# winner_extractor.py（抜粋）
def extract_winners(articles: list[dict], top_n: int = 5) -> list[str]:
    """view数上位N件からタグを抽出してstrategy.jsonに書き戻す"""
    sorted_articles = sorted(articles, key=lambda x: x["views"], reverse=True)
    winning_tags = []
    for article in sorted_articles[:top_n]:
        winning_tags.extend(article.get("tags", []))
    return list(set(winning_tags))
```

週次でこのスクリプトを実行し、生成プロンプトのテーマプールを上位タグで更新する。「書いたまま放置」ではなく、数値が生成にフィードバックされる構造を作ることが2週目の核心だ。

---

## フェーズ3（2ヶ月目以降）：マネタイズ経路を複数持つ

記事からの流入が安定したら、収益を複数の口から受け取れるようにする。

| 経路 | 単価の目安 | 自動化度 |
|------|-----------|---------|
| Zenn有料本 | ¥500〜¥1,500 | 半自動 |
| Gumroad教材 | ¥980〜¥2,980 | 全自動 |
| ランサーズ記事代行 | ¥3,000〜¥10,000/本 | 手動（入口のみ自動） |

**重要な判断：** 記事代行は「他人の流入を借りる」ため、自前SEOが育つ前でも収益が出る。月5万円の最短ルートは、コンテンツ自動化と代行の並走だ。

---

## 外注化の判断基準

以下のいずれかに該当した時点で、その作業を外注に出す判断をする。

1. 同じ手作業を**週3回以上**繰り返している
2. 自分の時給換算で**¥500未満**の作業になっている
3. 失敗しても**翌日リカバリーできる**範囲の業務

外注先への引き渡しには、パイプラインのログ出力を読めるドキュメントを1枚用意するだけでよい。コードは渡さなくてよい。

---

## 次の72時間でやること

1. `git log --oneline -10` でパイプラインの最終稼働を確認する
2. Zenn/Qiitaのview数を手動で確認し、上位3記事のタグをメモする
3. `winner_extractor.py` を走らせ、`strategy.json` のテーマを更新する
4. Gumroadに1商品だけ登録し、記事末尾にリンクを1本追加する

月5万円は大きく見えるが、分解すると「流入1本×マネタイズ1口×週次改善」の繰り返しだ。パイプラインはすでに動いている。あとは数値を見て、刺さった経路を太くするだけでいい。
