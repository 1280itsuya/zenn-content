結論から言うと、直近6案件でAI併用前後の環境構築を分単位で計測した結果、平均2時間28分が短縮された。本章では生ログ・時給換算・損益分岐の本数・運用チェックリストを公開する。

## 6案件の構築時間ログをBash + jqで自動計測する

計測は手動申告ではなく、`date +%s` で開始終了を刻むラッパーで自動化した。案件IDと種別を引数に取り、JSON1行を追記する。

```bash
#!/usr/bin/env bash
# track.sh <id> <kind> <mode:ai|manual>
start=$(date +%s)
echo "計測開始: $1 ($2/$3)"
read -p "構築完了したらEnter" _
end=$(date +%s)
min=$(( (end - start) / 60 ))
jq -n --arg id "$1" --arg kind "$2" --arg mode "$3" --argjson m "$min" \
  '{id:$id,kind:$kind,mode:$mode,min:$m}' >> build_log.jsonl
echo "所要 ${min}分 を build_log.jsonl に記録"
```

6案件の結果(分)はReact/Vite案件が手動142→AI38、Next.js案件が手動168→AI52。`jq -s 'group_by(.mode)|map({mode:.[0].mode,avg:(map(.min)|add/length)})' build_log.jsonl` で平均は手動151分・AI併用28分に収束した。

## 平均2時間28分短縮を時給3,500円でPython集計する

短縮分を副業時給3,500円で金額化する。pandasで案件種別ごとに差分を取り、月8案件想定で月額を試算した。

```python
import pandas as pd
df = pd.read_json("build_log.jsonl", lines=True)
piv = df.pivot_table(index="kind", columns="mode", values="min", aggfunc="mean")
piv["saved_min"] = piv["manual"] - piv["ai"]
avg_saved = piv["saved_min"].mean()     # = 148分 ≒ 2時間28分
monthly = avg_saved / 60 * 3500 * 8     # 月8案件
print(f"月時短換算: {monthly:,.0f}円")   # 月時短換算: 69,066円
```

148分 × 3,500円 × 8案件 = 月69,066円分の時間が浮く計算になる。

## Claude/Copilot月額と損益分岐の本数を計算する

GitHub Copilot月10ドル(約1,500円)、Claude Pro月20ドル(約3,000円)、合計4,500円。1案件あたり148分=8,633円分の時短なので、損益分岐を計算する。

```python
cost = 1500 + 3000          # 月額合計 4,500円
per_job = 148/60 * 3500     # 1案件の時短額 ≒ 8,633円
break_even = cost / per_job
print(f"損益分岐: {break_even:.2f}案件")  # 損益分岐: 0.52案件
```

月0.52案件、つまり1案件こなせばClaude/Copilotの投資は回収される。副業の手取りに対して回収リスクは限りなく低い。

## Node 22更新時の再検証をBashで一発実行する

AI生成の`package.json`はNode 18前提のままLTSが22へ上がると壊れる。納品後の事故を防ぐため、`engines`整合とビルド通過を1コマンドで再検証する。

```bash
#!/usr/bin/env bash
node -v | grep -q "v22" || { echo "Node 22未満: 再検証中止"; exit 1; }
npm ci && npm run build || { echo "ビルド破綻: 第2章の修正プロンプト適用"; exit 1; }
echo "Node 22 / Vite ビルド通過: 納品OK"
```

## プロンプト資産をgit tagでバージョン管理する運用ルール

構築フェーズで効いたプロンプトは`prompts/`にYAMLで保存し、案件種別ごとにタグ分岐させる。これが第2〜4章の破綻7パターン対策の再利用資産になる。

```yaml
# prompts/vite-react.yml
version: 3
kind: vite-react          # next/express/vite-react で分岐
node: "22"
checks: [tsc-noemit, vite-build, eslint]
notes: "破綻#3(型不整合)対策の検証スクリプトを必ず同梱"
```

`git tag prompt-v3 && git push --tags` でバージョンを固定し、案件着手時に`git switch`相当でテンプレを切り替える。これで初動3時間→40分が継続的に再現できる。
