この章では、インストールから初めての対話までを一気に進めます。私は自宅のWindows 11マシンでClaude Codeを毎日動かしています。本書の手順もWindowsを主軸にし、MacとLinuxの差分は補足に留めます。

### 2.1 インストール前の準備

必要なものは次の3つです。

- Windows 10以降のPC
- ターミナル(Windows Terminalを推奨します)
- Claudeのアカウント(サブスクリプションまたはAPI)

npm経由で入れる場合のみNode.js 18以上が必要です。

### 2.2 インストール

PowerShellを開き、次のコマンドを実行します。

```powershell
irm https://claude.ai/install.ps1 | iex
```

これが公式のインストーラです。npmに慣れている方は次でも構いません。

```powershell
npm install -g @anthropic-ai/claude-code
```

インストール後、ターミナルを開き直してからバージョンを確認します。

```powershell
claude --version
```

バージョン番号が表示されれば成功です。診断コマンド `claude doctor` も覚えておいてください。

【Mac/Linuxの場合】インストールコマンドが `curl -fsSL https://claude.ai/install.sh | bash` に変わるだけです。以降の手順に大きな差はありません。

### 2.3 認証——初回起動とログイン

作業したいプロジェクトのフォルダに移動して起動します。

```powershell
cd C:\work\my-project
claude
```

初回はテーマ選択のあと、ログイン方法を聞かれます。ブラウザが開くので、画面の案内に従ってください。選択肢は大きく2つです。

- Claude.aiアカウント(ProやMaxなどの定額プラン)
- Anthropic Console(APIの従量課金)

個人ならまず定額プランが目安です。あとから `/login` で切り替えられます。

### 2.4 許可プロンプト——AIが勝手に何でも実行するわけではない

最初に知ってほしい仕組みが「許可プロンプト」です。Claude Codeは、ファイルの書き換えやコマンド実行といった「環境を変更する操作」の前に、必ずあなたに確認を求めます。

画面に実行内容が表示され、おおよそ次の選択肢が出ます。

- はい(今回だけ許可)
- はい、このセッション中は同種の操作を確認しない
- いいえ(拒否する)

つまり、AIが勝手にファイルを消したり、勝手にコマンドを打ったりすることはありません。読み取りは確認なしで進みますが、書き込みと実行の門番はあなたです。「いいえ」を選んでも会話は途切れません。「別の方法でやり直して」と指示し直せます。

許可ルールは設定で細かく制御できますが、それは第3章で扱います。最初の30分は、確認内容を一つひとつ読んで「AIが何をしようとしているか」を観察してください。それ自体が最良の入門になります。

### 2.5 最初の30分で試す3つの対話

既存のプロジェクトフォルダで `claude` を起動してください。以下はそのまま写経できるプロンプトです。

#### 対話1: 既存コードの説明を頼む(5分)

```text
このプロジェクトの構成を説明してください。
主要なファイルの役割と、処理がどこから始まるかを
初心者にも分かるように教えてください。
```

Claude Codeが自分でファイルを開き、読み、要約します。チャットAIのようにコードを貼り付ける必要はありません。これがエージェント型の体験です。

#### 対話2: 小さなバグ修正を頼む(15分)

心当たりのある小さな不具合を、次の型で伝えます。

```text
(ファイル名)の(関数名)が、(入力の例)のとき
(実際の挙動)になります。期待は(正しい挙動)です。
原因を調べて、修正は最小限にしてください。
```

ポイントは「再現条件・実際の挙動・期待する挙動」の3点セットです。修正の直前に許可プロンプトが出るので、差分を読んでから許可してください。

#### 対話3: テストを実行してもらう(10分)

```text
このプロジェクトのテストを実行してください。
失敗があれば原因を説明してください。
修正はまだしないでください。
```

ここでも許可プロンプトが出ます。「修正はまだしないで」と行動範囲を区切る言い方は、この先ずっと使える基本テクニックです。

### 2.6 つまずきポイントQ&A

**Q1. `claude` と打つと「認識されていません」と言われます。**

A. PATHが通っていません。まずターミナルを開き直してください。npm版なら `npm config get prefix` で表示されるフォルダがPATHにあるか確認します。ネイティブ版のインストール先は `%USERPROFILE%\.local\bin` です。

**Q2. ログインしたはずなのに認証エラーになります。**

A. `/logout` してから `/login` でやり直すのが最短です。ブラウザが開かない場合は、ターミナルに表示されるURLを手動で貼り付けます。社内プロキシ環境では管理者への確認が必要なことがあります。

**Q3. ターミナルで日本語が文字化けします。**

A. まずWindows Terminalを使ってください。旧来のコマンドプロンプト単体より日本語表示が安定します。それでも化ける場合は文字コードをUTF-8にします。

```powershell
chcp 65001
```

フォントも日本語対応のものに設定してください。

**Q4. 動作がおかしいときの共通手順はありますか。**

A. `claude doctor` で診断、ターミナル再起動、再インストール、の順で試してください。

### 2.7 まとめ

- インストールはPowerShellのコマンド1行です。
- 書き込みと実行の前には必ず許可プロンプトが出ます。
- 最初の30分は「説明・小さな修正・テスト実行」の3本です。
