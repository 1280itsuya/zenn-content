## 仕組みを「育てる」という発想

単発の自動化スクリプトで満足していては、副業収益は月数千円で頭打ちになる。稼ぎ続けるために必要なのは、**仕組みが次の仕組みを生む複利構造**だ。記事が読まれれば検索データが蓄積し、そのデータが翌日の記事テーマを最適化する。このサイクルを24時間止めずに回すのが本章のゴールである。

## 24時間エンジンの全体像

自律運用パイプラインは大きく4層に分かれる。

```
[生成層]  → Claude CLI でコンテンツ生成
[投稿層]  → Zenn / Qiita / Bluesky へ自動公開
[計測層]  → view数・クリック数を毎日収集
[改善層]  → 勝ちパターンを抽出して翌日の生成に還元
```

Windowsであればタスクスケジューラ、Linuxではcronで各層を時刻起動する。

## Windowsタスクスケジューラでの設定例

PowerShellで以下を実行すると、毎朝7時に生成→投稿が自動起動する。

```powershell
$action = New-ScheduledTaskAction `
  -Execute "python" `
  -Argument "C:\auto-money\src\orchestrator.py" `
  -WorkingDirectory "C:\auto-money"

$trigger = New-ScheduledTaskTrigger -Daily -At "07:00"

Register-ScheduledTask `
  -TaskName "AutoMoneyMorning" `
  -Action $action `
  -Trigger $trigger `
  -RunLevel Highest
```

これだけで「起きたら記事が公開されている」状態が完成する。

## 計測層：viewを自動収集する

投稿して終わりでは改善ができない。Zenn APIとQiita APIを叩き、記事ごとのview数を毎日 `data/views_log.csv` に追記する。

```python
# src/view_collector.py（抜粋）
import requests, csv, datetime

def collect_zenn_views(username: str):
    url = f"https://zenn.dev/api/articles?username={username}&order=latest"
    articles = requests.get(url).json()["articles"]
    today = datetime.date.today().isoformat()
    with open("data/views_log.csv", "a", newline="") as f:
        w = csv.writer(f)
        for a in articles:
            w.writerow([today, "zenn", a["slug"], a["liked_count"]])
```

1週間分が溜まれば、どのタグ・どのタイトル構造がviewを稼いでいるか統計が取れる。

## 改善層：勝ちパターンを生成に還元する

`winner_extractor.py` は週次でCSVを集計し、上位3記事のタグとキーワードを `config/winning_topics.json` に書き出す。生成エージェントはこのJSONを毎朝読み込み、次の記事テーマを決定する。

```json
// config/winning_topics.json（自動更新）
{
  "top_tags": ["Python", "Claude", "自動化"],
  "best_title_pattern": "〇〇を自動化した話",
  "avg_view_top3": 312
}
```

これが**複利の核心**だ。記事が読まれるほどデータが精度を上げ、精度が上がるほど次の記事が読まれやすくなる。

## 無人課金ゼロの安全設計

深夜や外出中に有料APIを暴走させないため、無料枠（Claude CLI Max定額）は全自動、有料処理は承認キューを必須にする。

```python
# 有料処理は approval_queue.json に積むだけ
def queue_paid_task(task: dict):
    with open("data/approval_queue.json", "a") as f:
        json.dump(task, f, ensure_ascii=False)
        f.write("\n")
```

朝に人間が確認し、承認した分だけ実行する。これで**無人課金ゼロ**を保ちながら自律性を最大化できる。

## 複利化のチェックリスト

- [ ] タスクスケジューラで7時・13時・19時の3本を登録した
- [ ] view収集が毎日CSVに追記されている
- [ ] `winning_topics.json` が週次で自動更新されている
- [ ] 有料処理は承認キュー経由になっている

仕組みは一度作れば24時間働き続ける。あなたの仕事は、その仕組みを週1回チェックして磨き続けることだけだ。
