## 結論：systemd timerで毎朝7時・1体5円の自動投稿に到達した

3か月の運用で確定したコストは、電気代込みで**1体あたり4.8円・月30体で約144円**。内訳はGPU(RTX 4090)消費電力0.35kWh×31円＋CLIP推論のCPU時間。cronより`systemd timer`を選んだ理由は、失敗時の`journalctl`一括追跡とリトライ制御が効くため。

```ini
# /etc/systemd/system/civitai-pipeline.timer
[Timer]
OnCalendar=*-*-* 07:00:00
Persistent=true
[Install]
WantedBy=timers.target
```

```bash
systemctl enable --now civitai-pipeline.timer
systemctl list-timers civitai-pipeline.timer  # NEXT列で7:00を確認
```

## 失敗1・重複投稿でshadowban：sha256ハッシュ台帳で根絶

同一構図を3日連続投稿しエンゲージメントが1/5へ激減。Civitai側のshadowban判定をハッシュ重複排除で回避する。

```python
import hashlib, json, pathlib
LEDGER = pathlib.Path("posted.json")
def is_dup(img_bytes: bytes) -> bool:
    h = hashlib.sha256(img_bytes).hexdigest()
    seen = json.loads(LEDGER.read_text()) if LEDGER.exists() else []
    if h in seen: return True
    seen.append(h); LEDGER.write_text(json.dumps(seen)); return False
```

## 失敗2・CLIP閾値0.32が高すぎて候補ゼロ：分位点で動的化

固定閾値0.32は週の3割で候補ゼロを返した。上位30%を採る分位点方式へ変更したdiff。

```diff
- picks = [x for x in scored if x.clip >= 0.32]
+ import numpy as np
+ cutoff = np.quantile([x.clip for x in scored], 0.70)
+ picks = [x for x in scored if x.clip >= cutoff]  # 常に上位30%
```

## 失敗3・メタデータ欠落で再現不能：sidecar JSON強制

seed/promptを保存し忘れBuzzした構図を再現できなかった。投稿時にsidecarを必須化する。

```python
def save_meta(path, seed, prompt, clip):
    pathlib.Path(path + ".json").write_text(json.dumps(
        {"seed": seed, "prompt": prompt, "clip": round(clip,4)},
        ensure_ascii=False))
```

## 失敗4&5・Civitai API v1廃止とNSFWタグ漏れ：契約テストで先回り

API仕様変更で`/v1/images`が410を返し3日分の投稿が無言で失敗。著作権・nsfwタグ漏れで2体が削除された。投稿前に契約テストとタグ検証を挟む。

```python
import requests
def post_image(f, meta, token):
    assert {"nsfw","license"} <= meta.keys(), "必須タグ欠落"
    r = requests.post("https://civitai.com/api/v1/images",
        headers={"Authorization": f"Bearer {token}"},
        files={"file": f}, data=meta, timeout=30)
    if r.status_code == 410:
        raise RuntimeError("API改廃: エンドポイント要更新")
    r.raise_for_status(); return r.json()["id"]
```
