# llms.txtでサイトをAI検索（ChatGPT・Perplexity）に正確に拾わせる実装ガイド

## llms.txtとは

`robots.txt`がクローラー向けなら、`llms.txt`はLLM向けのサイト案内ファイルだ。2024年にAnswer.AIのJeremy Howardが提唱したMarkdown形式の仕様で、サイトルートに置くだけでPerplexityやChatGPT（Bing検索経由）のインデクサーが参照する。

公式仕様: [llmstxt.org](https://llmstxt.org)

---

## 最小構成

`/llms.txt`（サイトルート直下）に以下を置く。

```markdown
# サービス名

> 一文でサービスの目的を書く。AIが回答するとき最初に使われる部分。

サービスの補足説明をここに書く。箇条書きより地の文のほうが
LLMに取り込まれやすい傾向がある。

## ドキュメント

- [APIリファレンス](https://example.com/docs/api): エンドポイント一覧と認証方法
- [クイックスタート](https://example.com/docs/start): 5分で動かす手順

## 最新記事

- [llms.txt実装ガイド](https://example.com/blog/llms-txt): 2026年版
```

`>` で始まる行がサービスの要約で、AI回答に最も反映される。`## セクション名` でグループ化し、`- [ページ名](URL): 説明` 形式でページを列挙する。

## Next.js / Python での動的生成

ページが増えるたびに手書きするのは現実的でない。APIルートで自動生成する。

**Next.js（App Router）**

```ts
// app/llms.txt/route.ts
export async function GET() {
  const posts = await fetchLatestPosts(10); // CMSやDBから取得

  const lines = [
    `# ${SITE_NAME}`,
    "",
    `> ${SITE_DESCRIPTION}`,
    "",
    "## 最新記事",
    "",
    ...posts.map((p) => `- [${p.title}](${p.url}): ${p.excerpt}`),
  ];

  return new Response(lines.join("\n"), {
    headers: { "Content-Type": "text/plain; charset=utf-8" },
  });
}
```

**Python / FastAPI**

```python
@app.get("/llms.txt", response_class=PlainTextResponse)
async def llms_txt():
    posts = fetch_latest_posts(limit=10)
    lines = [f"# {SITE_NAME}", "", f"> {SITE_DESCRIPTION}", "", "## 最新記事", ""]
    for p in posts:
        lines.append(f"- [{p.title}]({p.url}): {p.excerpt}")
    return "\n".join(lines)
```

## つまずきポイント

**1. Content-TypeはREST用途のままにしない**

既存のAPIルートに追記して `application/json` で返してしまうケースがある。必ず `text/plain; charset=utf-8` で返すこと。

**2. URLは絶対パスで書く**

`/docs/api` ではなく `https://example.com/docs/api`。LLMがファイルをオフライン参照するとき、相対パスは解決できない。

**3. robots.txtで弾かない**

```
# 悪い例（llms.txtも取得できなくなる）
User-agent: *
Disallow: /
Allow: /sitemap.xml
```

少なくとも `Allow: /llms.txt` を明示しておく。

**4. 更新日時のヒントを添える**

```markdown
<!-- last-updated: 2026-06-17 -->
```

HTMLコメントはMarkdownでも記述できる。インデクサーが鮮度を判断するヒントになる。

## 動作確認

まずファイルが取得できるか確認する。

```bash
curl -s https://example.com/llms.txt | head -20
```

次にPerplexityで「example.comについて教えて」と質問し、回答が自分のサービス説明と一致するか確かめる。即座に反映されるわけではなく、インデクサーの再クロールを待つ必要がある点は留意しておく。

---

## まとめ

- サイトルートの `/llms.txt` に置くだけで対応コストは低い
- `>` の要約行に最も力を入れる。一文でサービスを説明し切ること
- ページ数が多いサービスはAPIルートで動的生成する
- `Content-Type: text/plain`・絶対URL・robots.txt除外の3点が鉄則

仕様はまだドラフト段階だが、Perplexityが実際に参照し始めていることは確認されている。既存サービスであれば数十行の追加で済むため、AI検索対応の入口として最初に手を付ける価値がある。
