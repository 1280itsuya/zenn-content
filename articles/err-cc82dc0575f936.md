## 発生条件

- `null` が代入されたオブジェクト変数に対してメソッド呼び出しやフィールドアクセスを行ったとき
- メソッドの戻り値が `null` になり得るにもかかわらず、戻り値を検査せずに続けて操作したとき
- 配列やコレクションの要素が `null` のまま、インデックスアクセス後に操作したとき

## 原因

`Exception in thread "main" java.lang.NullPointerException` は、JVM が `null` 参照を経由してオブジェクトを操作しようとした際にスローされる。以下は最小の再現例だ。

```java
public class Main {
    public static void main(String[] args) {
        String value = getValue(); // null が返る可能性がある
        System.out.println(value.length()); // NPE: value が null のためメソッド呼び出し不可
    }

    static String getValue() {
        return null; // 意図せず null を返している
    }
}
```

実行すると次のようなスタックトレースが出力される。

```
Exception in thread "main" java.lang.NullPointerException
    at Main.main(Main.java:4)
```

Java 14 以降では **Helpful NullPointerException** が有効になっており、`Cannot invoke "String.length()" because "value" is null` のように、どの変数が `null` だったかをメッセージに含めてくれる。これにより原因箇所の特定が大幅に楽になる。Java 13 以前やメッセージが簡素な場合は、スタックトレースの行番号を手がかりに変数の状態をデバッガで確認する。

## 修正方法

### パターン1: 事前に null チェックする

```java
public class Main {
    public static void main(String[] args) {
        String value = getValue();
        if (value != null) {
            System.out.println(value.length());
        } else {
            System.out.println("value is null");
        }
    }

    static String getValue() {
        return null;
    }
}
```

シンプルな `if` による null ガードは可読性が高く、条件分岐での処理切り替えが必要な場合に適している。

### パターン2: Optional を活用する

```java
import java.util.Optional;

public class Main {
    public static void main(String[] args) {
        Optional<String> value = Optional.ofNullable(getValue());
        value.ifPresentOrElse(
            v -> System.out.println(v.length()),
            () -> System.out.println("value is null")
        );
    }

    static String getValue() {
        return null;
    }
}
```

`Optional` を使うと、呼び出し元に「この値は存在しない可能性がある」という意図をシグネチャで明示できる。`orElse` や `orElseGet` でデフォルト値を返す設計にすることで、`Exception in thread "main" java.lang.NullPointerException` の発生経路そのものを潰せる。

### パターン3: メソッドから null を返さない設計にする

```java
import java.util.Collections;
import java.util.List;

public class Main {
    public static void main(String[] args) {
        List<String> items = getItems();
        System.out.println(items.size()); // null の心配が不要
    }

    static List<String> getItems() {
        // null の代わりに空コレクションを返す
        return Collections.emptyList();
    }
}
```

コレクションを返すメソッドでは `null` ではなく空コレクションを返すのが定石だ。呼び出し側のチェックコードが不要になり、コード全体がシンプルになる。根本的な解決策として、メソッドの設計段階から `null` を返さないよう徹底することが最も効果的だ。
