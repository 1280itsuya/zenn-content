## 発生条件

- `git status` や `git log`、`git add` などの Git コマンドを、Git で管理されていないディレクトリで実行したとき
- リポジトリのルートより上の親ディレクトリに移動した状態でコマンドを叩いたとき
- `git clone` に失敗した後の空フォルダや、ZIP 展開したプロジェクトフォルダ内で作業しているとき

## 原因

エラーメッセージ `fatal: not a git repository (or any of the parent directories): .git` は、Git がカレントディレクトリから親ディレクトリへ順番に `.git` フォルダを探索した結果、どこにも見つからなかった場合に発生する。

たとえば、以下のような状況が典型例だ。

```bash
# プロジェクトのつもりで作ったが git init をしていない
mkdir my-project
cd my-project
git status
# => fatal: not a git repository (or any of the parent directories): .git
```

```bash
# clone したリポジトリのルートより外で作業してしまっている例
cd ~
git log
# => fatal: not a git repository (or any of the parent directories): .git
```

Git は `.git` ディレクトリを「リポジトリの目印」として扱う。この `.git` フォルダが存在しない限り、どのような Git コマンドも実行できない。ZIP でダウンロードしたコードは `.git` が含まれないことが多く、同じエラーが出る。

## 修正方法

### 新規リポジトリとして初期化する場合

```bash
cd my-project       # 対象のディレクトリに移動
git init            # .git フォルダを作成してリポジトリを初期化
git add .
git commit -m "initial commit"
```

`git init` を実行すると `.git` ディレクトリが生成され、以降の Git コマンドが使えるようになる。

### 既存のリポジトリに移動する場合

```bash
# リポジトリがどこにあるか確認する
find ~ -name ".git" -maxdepth 4 -type d 2>/dev/null

# 見つかったらそのディレクトリへ移動
cd ~/repos/my-project
git status   # 今度は正常に動作する
```

### リモートから clone する場合（ZIP 展開後などに多い）

```bash
# ZIP 展開したフォルダは .git を持たないので clone し直す
rm -rf my-project-main           # ZIP 展開フォルダを削除
git clone https://github.com/user/my-project.git
cd my-project
git status   # fatal: not a git repository は解消される
```

要点は3つだ。まず `pwd` で自分がどこにいるかを確認する。次に `ls -la` で `.git` フォルダの有無を確認する。`.git` がなければ `git init` するか、正しいリポジトリのルートへ `cd` するかのどちらかで必ず解消できる。`fatal: not a git repository` というメッセージが出たら「`.git` がない場所でコマンドを打った」と即座に判断してよい。
