## 発生条件

- `Array.map()` でJSX要素を返す際に、各要素に `key` prop を付けていない
- `key` に配列のインデックスを使っており、要素の追加・削除・並び替えが起きる
- `key` に重複する値（例: 同じ文字列定数）を指定している

## 原因

[React](https://www.amazon.co.jp/s?k=React%20%E5%85%A5%E9%96%80%20%E6%9C%AC&tag=1280itsuya22-22)は仮想DOMの差分計算時にリスト内の各要素を識別するために `key` を使用する。`key` が存在しないか重複していると、再レンダリング時にどの要素が変わったかを正確に判断できず、コンソールに以下の警告が出る。

```
Warning: Each child in a list should have a unique "key" prop.
```

最小再現コード：

```jsx
function ItemList({ items }) {
  return (
    <ul>
      {items.map((item) => (
        <li>{item.name}</li>  // key が無い
      ))}
    </ul>
  );
}
```

インデックスを使った場合でも、要素の並び替えや削除が伴うと意図しない再レンダリングが発生する：

```jsx
// 並び替えがある場合は不具合の原因になる
{items.map((item, index) => (
  <li key={index}>{item.name}</li>
))}
```

`index` を `key` にすると、要素の順序が変わったとき Reactは「位置が同じなら同じ要素」と誤判断し、状態（inputの内容など）が意図しない要素に残留することがある。

## 修正方法

データが持つ安定した一意の識別子（`id` など）を `key` に使う：

```jsx
function ItemList({ items }) {
  return (
    <ul>
      {items.map((item) => (
        <li key={item.id}>{item.name}</li>
      ))}
    </ul>
  );
}
```

データに `id` がない場合は、データ生成時に付与する：

```js
import { v4 as uuidv4 } from 'uuid';

const items = rawItems.map((item) => ({
  ...item,
  id: uuidv4(),  // 配列生成時に一度だけ付与する
}));
```

注意点：`uuidv4()` をレンダリング中（`map()` の中）で呼ぶと、再レンダリングのたびに新しい値が生成されてしまい、かえってパフォーマンスが悪化する。`key` の生成はデータ側で行い、レンダリング側では参照するだけにする。

`index` を `key` に使っても警告自体は消えるが、**並び替えや要素の追加・削除が一切ない静的なリスト**に限って許容される。それ以外のケースでは `id` などの安定したキーを使うのが原則。
