# GitHub ActionsとClaude APIでブログ記事を毎日自動投稿する（サーバー不要・無料枠で動く）

個人ブログの継続投稿は思った以上に手間がかかる。本記事では、GitHub Actionsのスケジューラー＋Claude APIを組み合わせて、毎朝記事を自動生成・投稿するパイプラインを構築する手順を解説する。VPSもコンテナも不要で、GitHub無料枠で動く。

---

## 全体構成

```
GitHub Actions (cron) → Python スクリプト → Claude API (記事生成) → Zenn/はてな (投稿)
```

Actionsのワークフローが毎朝起動し、Pythonスクリプトがテーマを決めてClaude APIに記事本文を生成させ、プラットフォームのAPIへ投稿する。

---

## 1. リポジトリ準備

```
.
├── .github/workflows/daily_post.yml
├── src/
│   └── post_article.py
├── topics.txt          # 投稿テーマを1行1件で列挙
└── requirements.txt
```

`requirements.txt`:
```
anthropic>=0.28.0
requests
python-dotenv
```

---

## 2. ワークフローファイル

```yaml
# .github/workflows/daily_post.yml
name: Daily Blog Post

on:
  schedule:
    - cron: '0 22 * * *'  # UTC 22:00 = JST 07:00
  workflow_dispatch:       # 手動実行も可

jobs:
  post:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-python@v5
        with:
          python-version: '3.12'

      - run: pip install -r requirements.txt

      - run: python src/post_article.py
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          ZENN_GITHUB_TOKEN: ${{ secrets.ZENN_GITHUB_TOKEN }}
```

## 3. 記事生成スクリプト

```python
# src/post_article.py
import os, pathlib, datetime, anthropic

def pick_topic() -> str:
    topics = pathlib.Path("topics.txt").read_text(encoding="utf-8").splitlines()
    day_index = datetime.date.today().toordinal() % len(topics)
    return topics[day_index].strip()

def generate_article(topic: str) -> str:
    client = anthropic.Anthropic()
    message = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=2000,
        messages=[{
            "role": "user",
            "content": (
                f"エンジニア向けの実用的な技術記事を書いてください。\n"
                f"テーマ: {topic}\n"
                f"要件: Markdown形式・1500字前後・コード例を含む・誇大表現なし"
            )
        }]
    )
    return message.content[0].text

def push_to_zenn(title: str, body: str):
    """Zennはリポジトリpushで公開されるためファイル書き込みだけでよい"""
    slug = datetime.date.today().strftime("%Y%m%d")
    path = pathlib.Path(f"articles/{slug}.md")
    path.parent.mkdir(exist_ok=True)
    path.write_text(
        f"---\ntitle: {title}\nemoji: 📝\ntype: tech\ntopics: []\npublished: true\n---\n\n{body}",
        encoding="utf-8"
    )
    os.system(f'git config user.email "bot@example.com"')
    os.system(f'git config user.name "Article Bot"')
    os.system(f'git add articles/ && git commit -m "post: {slug}" && git push')

if __name__ == "__main__":
    topic = pick_topic()
    body = generate_article(topic)
    title = body.splitlines()[0].lstrip("# ").strip()
    push_to_zenn(title, body)
    print(f"投稿完了: {title}")
```

## 4. Secretsの設定

GitHubリポジトリの **Settings → Secrets and variables → Actions** で以下を登録する。

| キー名 | 値 |
|---|---|
| `ANTHROPIC_API_KEY` | Anthropic Consoleで発行したキー |
| `ZENN_GITHUB_TOKEN` | `contents: write` 権限のPAT |

---

## つまずきポイント

**git pushが認証エラーになる**  
ワークフロー内でpushするには、PAT（`ZENN_GITHUB_TOKEN`）をリモートURLに埋め込む必要がある。

```bash
git remote set-url origin https://x-access-token:${ZENN_GITHUB_TOKEN}@github.com/YOUR/REPO.git
```

スクリプト冒頭で `os.system(...)` の前に実行しておくと確実。

**cronが動かない（Actionsの非アクティブ化）**  
リポジトリへの直近コミットが60日以上ない場合、GitHubはスケジュールワークフローを自動停止する。`workflow_dispatch` を使った手動実行を定期的に行うか、`topics.txt` を週次で更新する運用が安全。

**記事が空ファイルになる**  
Claude APIのレスポンスが空の場合に無条件でpushするとZennに空記事が作られる。生成後にバリデーションを挟むこと。

```python
if len(body.strip()) < 300:
    raise ValueError(f"生成結果が短すぎます: {len(body)}字")
```

## 運用コスト感

Claude Sonnet 4.6を使った場合、1記事あたりの入出力トークンはおおよそ2,000〜3,000トークン程度になる。実際のコストはAnthropicの公式料金ページで確認してほしい。月30記事程度であれば無視できる水準になることが多い。

## まとめ

- **GitHub Actions cronで毎朝自動起動**し、テーマローテーションで投稿内容を変える
- Zennはリポジトリpushが即公開になるためAPIなしで完結する
- git push認証とワークフロー自動停止の2点が主なつまずき所
- バリデーションを入れて空記事・短すぎる記事を防ぐ
