# [ドメイン](https://px.a8.net/svt/ejp?a8mat=4B3XB4+4VMW8I+50+2HU3GX)取得+DNS設定を最短で終わらせる手順と費用感

## 結論

ドメイン取得からDNS浸透まで、手順さえ知っていれば**30分以内**に終わる。費用は年間数百円〜2,000円前後が相場だが、登録ドメインの種類とレジストラによって初年度と更新費が大きく異なる点に注意が必要だ。

## なぜここを最初に理解するか

[個人開発](https://www.amazon.co.jp/s?k=%E5%80%8B%E4%BA%BA%E9%96%8B%E7%99%BA%20%E6%9C%AC&tag=1280itsuya22-22)でVPSを立てたとき、「IPアドレスで動いているけどドメインを割り当てたい」という場面は必ず来る。このとき詰まるポイントは2つだ。

1. **レジストラ選び**（どこでドメインを買うか）
2. **DNSレコードの設定**（どのIPに向けるか）

この2ステップを混同したまま作業すると、設定ミスで丸一日溶かすことになる。順番と役割を先に把握しておくと迷わない。

## 費用感（2026年時点・変動するため必ず公式で確認）

ドメイン料金はレジストラ・TLD・タイミングによって変動する。以下はあくまで目安であり、公式サイトの最新料金を必ず確認してほしい。

| TLD | 初年度の目安 | 更新の目安 |
|-----|------------|----------|
| `.com` | 数百円〜1,500円前後 | 1,500円〜2,000円前後 |
| `.net` | 数百円〜1,500円前後 | 1,500円〜2,000円前後 |
| `.dev` | 2,000円〜3,000円前後 | 同程度 |
| `.jp` | 2,000円〜4,000円前後 | 同程度 |

**初年度激安・更新は高め**というパターンが多い。長期運用するサービスなら更新費を必ず確認すること。

### 主要レジストラ

- **お名前.com** — 国内最大手。UIが情報過多なのでオプション追加に注意。
- **ConoHa VPS / Xserver VPS / さくらのVPS** — ホスティング事業者がドメイン取得窓口も持っており、同社でVPSとドメインを一元管理できる。移管作業が減るのが利点。
- **Cloudflare Registrar** — `.com` を原価提供（約1,000円前後）。DNS管理も同一画面で完結するため、個人開発では選択肢として有力。

## 最短手順

### Step 1: ドメインを取得する（5〜10分）

1. レジストラのサイトでドメイン名を検索し空きを確認
2. カートに入れて購入（[クレジットカード](https://px.a8.net/svt/ejp?a8mat=4B3XB4+5N0U2A+38L8+BX3J5)即時反映が最速）
3. Whois公開代行を有効にする（個人情報保護のため必須）

お名前.comなら購入完了直後にDNS管理画面にアクセスできる。

### Step 2: VPSのIPアドレスを確認する（1分）

```bash
# ConoHa / Xserver / さくら等のコンパネ、またはSSHログイン後
ip addr show eth0 | grep 'inet '
```

グローバルIPアドレス（例: `203.0.113.10`）を手元に控えておく。

### Step 3: DNSレコードを設定する（5〜10分）

レジストラのDNS管理画面（またはCloudflareのダッシュボード）で以下を追加する。

| タイプ | ホスト名 | 値 | TTL |
|--------|---------|-----|-----|
| A | `@` | VPSのIPアドレス | 3600 |
| A | `www` | VPSのIPアドレス | 3600 |

`@` はルートドメイン（`example.com`）を指す。`www` を別サブドメインとして運用する場合は両方設定しておく。

### Step 4: 浸透を確認する（5〜60分）

DNSの反映（浸透）には数分〜数時間かかる。以下のコマンドで確認できる。

```bash
# 引けるか確認
dig example.com A +short

# 特定のDNSサーバーに問い合わせる（Cloudflareの場合）
dig @1.1.1.1 example.com A +short
```

`dig` が自分のVPS IPを返せば完了。浸透前は古いキャッシュを返すことがある。TTLを短く（300秒など）設定しておくと切り替え時に便利だ。

### Step 5: Webサーバー側の設定（Nginxの例）

```nginx
server {
    listen 80;
    server_name example.com www.example.com;
    root /var/www/html;
    index index.html;
}
```

ここまで設定できれば `http://example.com` でアクセスできる。HTTPS化は Let's Encrypt の `certbot` で追加する。

## よくある詰まりポイント

**TTLが長くてキャッシュが残る**
変更前にTTLを小さくしておくのが定石。デフォルト3600秒（1時間）のままだと切り替え確認に時間がかかる。

**AレコードとCNAMEの混在**
ルートドメイン（`@`）にCNAMEは設定できない（RFC非準拠）。ルートにはAレコードを使うこと。

**DNSとドメインを別業者で管理する場合**
レジストラ側でネームサーバーをCloudflare等に変更し、以後のDNSレコード編集はCloudflareで行う。ここを混同すると「設定したのに反映されない」になる。

## まとめ

- ドメイン取得は**レジストラ選び→購入→DNS設定**の3ステップ
- 費用は初年度と更新費が異なる場合が多いため**更新費を必ず確認**する
- ConoHaやXserver、さくらなどVPS提供社でドメインも取ると管理が一元化できる
- `dig` コマンドで浸透を確認し、Nginxの `server_name` と一致させれば完了

手順自体は単純で、慣れれば15分で終わる作業だ。最初に役割分担（レジストラとDNSは別概念）を把握しておくと、どこで詰まっているかがすぐ判断できる。
