# [ドメイン](https://px.a8.net/svt/ejp?a8mat=4B3XB4+4VMW8I+50+2HU3GX)取得 + DNS設定の最短手順と費用【個人開発者向け】

## 結論：最短ルートはこれ

1. **お名前.com または Cloudflare Registrar** でドメインを取得
2. **Cloudflare** にネームサーバーを移管してDNS管理
3. VPS（ConoHa / Xserver / さくら等）のIPをAレコードで登録

これだけ。難しい設定は不要で、DNSの反映まで含めても数時間以内に完了する。

---

## 費用の目安（料金は変動するため必ず公式を確認）

ドメイン料金はTLD（トップレベルドメイン）によって大きく異なる。

| TLD | 傾向 |
|-----|------|
| `.com` | 汎用性が高く人気。初年度割引が多い |
| `.dev` | 開発者向け、Google管理。HTTPS必須 |
| `.jp` | 国内信頼性高め。やや高め |
| `.tech` / `.io` | スタートアップ・開発系で人気 |

お名前.comでは初年度キャンペーン価格が適用されることが多いが、**2年目以降の更新料金が高くなるケースがある**。Cloudflare Registrarは原価販売（仕入れ値そのままで販売）を謳っており、更新料金が安定しやすい特徴がある。いずれも公式サイトで最新価格を確認すること。

## 手順①：ドメインを取得する

### お名前.com の場合

```
1. https://www.onamae.com/ でドメイン検索
2. カートに追加→会員登録→支払い
3. 取得完了メールを受け取る（数分〜数十分）
```

Whois情報公開代行は必ず有効にしておく（個人情報保護のため）。

### Cloudflare Registrar の場合

既存のCloudflareアカウントがあれば「Domain Registration」から検索・購入できる。DNS管理も同一画面で完結するため、後述の移管手順が不要になる点が楽だ。

---

## 手順②：CloudflareでDNSを管理する（推奨）

レジストラのDNS管理画面でも設定できるが、CloudflareのDNSは**応答速度・UI・無料SSL（CDN経由）**の面で優れているため移管を推奨する。

```
1. Cloudflare（https://cloudflare.com）でアカウント作成
2. 「Add a Site」でドメインを追加
3. 無料プランを選択
4. Cloudflareが提示する2つのネームサーバーをメモする
   例: xxx.ns.cloudflare.com / yyy.ns.cloudflare.com
5. お名前.comの管理画面 → 「ネームサーバーの変更」で上記2つを入力
6. 反映まで最大48時間（実際は数時間以内が多い）
```

ネームサーバー変更後、Cloudflareのダッシュボードがアクティブ状態になれば移管完了。

---

## 手順③：VPSのIPアドレスをAレコードで登録

ConoHa VPS・Xserver VPS・さくらのVPSのいずれも、コントロールパネルでサーバーのグローバルIPを確認できる。

```
Cloudflare ダッシュボード → DNS → Records → 「Add record」

Type  : A
Name  : @        （ルートドメイン用）または www
IPv4  : YOUR_VPS_IP
TTL   : Auto
Proxy : 「Proxied」（オレンジ雲）にすると Cloudflare CDN 経由になる
```

サブドメインを使う場合（`api.example.com` など）は同様にAレコードを追加する。

**CNAMEとAレコードの使い分け：**
- IPが固定のVPS → **Aレコード**
- Vercel・Netlify等のPaaS → **CNAMEレコード**（各サービスの指示に従う）

---

## 手順④：動作確認

```bash
# DNS反映確認
dig example.com A +short

# または
nslookup example.com 8.8.8.8
```

期待するVPSのIPが返ってくれば成功。

---

## よくある落とし穴

**キャッシュで古いIPが返る**  
TTL期間中は古い応答がキャッシュされる。`dig`に`+nocache`を使うか、別の端末・DNSサーバーで確認する。

**`www`と裸ドメインの両方を設定する**  
`@`（example.com）と`www`（www.example.com）は別レコード。片方だけ設定して「アクセスできない」というミスが多い。

**`.dev`ドメインはHTTPS必須**  
`.dev`はHSTSプリロードリストに含まれており、HTTPでのアクセスはブラウザにブロックされる。SSL証明書の設定（Let's Encrypt等）を先に済ませること。

---

## まとめ

ドメイン取得からDNS設定までの最短手順は「取得→Cloudflare移管→Aレコード登録→確認」の4ステップ。費用はTLDと更新タイミング次第で変動するため、初年度価格だけでなく更新料金も必ず確認してから決めること。Cloudflare Registrarを使えばDNS管理まで一元化でき、運用コストを最小化できる。
