## 発生条件

- 関数の戻り値が `None` になりうるのに、戻り値に対して直接 `[key]` や `[index]` でアクセスしている
- 辞書検索（`dict.get(key)`）やリスト操作（`re.search()` など）が失敗して `None` を返したのに、結果をそのまま添字アクセスしている
- APIレスポンスや外部ファイルの読み込みで、データが存在しないケースを考慮していない

## 原因

`TypeError: 'NoneType' object is not subscriptable` は、`None` に対して `[]` によるサブスクリプトアクセスを試みたときに発生します。

```python
import re

text = "hello world"

# re.search が None を返すケース
match = re.search(r"\d+", text)  # マッチしないので None が返る
print(match[0])  # TypeError: 'NoneType' object is not subscriptable
```

```python
data = {"user": {"name": "Alice"}}

# キーが存在しない場合、get() は None を返す
profile = data.get("profile")
print(profile["age"])  # TypeError: 'NoneType' object is not subscriptable
```

辞書の `.get()` メソッドや `re.search()` などは、対象が見つからない場合に `None` を返します。その `None` に対して `[...]` でアクセスしようとすると、[Python](https://www.amazon.co.jp/s?k=Python%20%E5%85%A5%E9%96%80%20%E6%9C%AC&tag=1280itsuya22-22) は添字操作をサポートしない型（`NoneType`）に対する操作として例外を送出します。

## 修正方法

値が `None` でないことを事前にチェックするか、デフォルト値を与えて `None` が伝播しないようにします。

```python
import re

text = "hello world"

match = re.search(r"\d+", text)
if match is not None:
    print(match[0])
else:
    print("マッチなし")
```

```python
data = {"user": {"name": "Alice"}}

# get() にデフォルト値を渡して None を防ぐ
profile = data.get("profile", {})
print(profile.get("age", "不明"))
```

```python
# walrus 演算子（Python 3.8+）を使う書き方
import re

text = "hello world"

if match := re.search(r"\d+", text):
    print(match[0])
```

要点は2つです。

1. **`None` チェックを必ず挟む** — `if value is not None:` で確認してからアクセスします。`if value:` は `0` や空文字列も偽になるため、`None` の判定には `is not None` を使う方が明確です。
2. **デフォルト値でフォールバック** — `dict.get(key, {})` のようにデフォルトを渡すか、関数側で `None` を返さない設計にすることで、呼び出し側に `None` が漏れるのを防げます。

`'NoneType' object is not subscriptable` というメッセージが出たら、エラー行のひとつ手前で代入している変数を `print()` や `assert` で確認し、その時点で `None` になっていないか追うのが最短の原因特定方法です。
