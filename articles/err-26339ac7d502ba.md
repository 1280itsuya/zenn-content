## 発生条件

- JVM がクラスローダーで `Class.forName()` や `ClassLoader.loadClass()` を呼んだとき、対象クラスが **クラスパス上に存在しない** 場合
- 依存ライブラリを `build.gradle` や `pom.xml` に追加し忘れた、またはスコープ（`provided`/`compileOnly` 等）が本番実行時に除外されている場合
- JAR を手動クラスパス指定で実行する際に `-cp` オプションや `CLASSPATH` 環境変数の記述が漏れている場合

## 原因

`java.lang.ClassNotFoundException` は、実行時にクラスが見つからなかったことを示す検査例外です。コンパイルは通るが実行時に落ちる点が厄介で、特に「ビルドツールの依存定義に追加したつもり」で実は追加できていないケースが多いです。

以下は Gradle プロジェクトで依存を書き忘れた状態の最小再現例です。`com.google.gson.Gson` を使おうとしているのに `gson` ライブラリが依存に含まれていないため、実行時に `ClassNotFoundException` が発生します。

```java
// Main.java
import com.google.gson.Gson;

public class Main {
    public static void main(String[] args) throws Exception {
        // build.gradle に gson の依存が無い状態で実行すると
        // java.lang.ClassNotFoundException: com.google.gson.Gson が投げられる
        Class<?> clazz = Class.forName("com.google.gson.Gson");
        Gson gson = (Gson) clazz.getDeclaredConstructor().newInstance();
        System.out.println(gson.toJson("hello"));
    }
}
```

```
// build.gradle（問題のある状態）
dependencies {
    // gson が無い
}
```

実行すると次のようなスタックトレースが出ます。

```
Exception in thread "main" java.lang.ClassNotFoundException: com.google.gson.Gson
    at java.base/jdk.internal.loader.BuiltinClassLoader.loadClass(BuiltinClassLoader.java:641)
    at java.base/jdk.internal.loader.ClassLoaders$AppClassLoader.loadClass(ClassLoaders.java:188)
    at java.base/java.lang.ClassLoader.loadClass(ClassLoader.java:526)
    at java.lang.Class.forName0(Native Method)
    at java.lang.Class.forName(Class.java:421)
    at Main.main(Main.java:7)
```

## 修正方法

**Gradle の場合：** `build.gradle` に不足している依存を追加します。

```groovy
// build.gradle
dependencies {
    implementation 'com.google.gson:gson:2.10.1'
}
```

**Maven の場合：** `pom.xml` に `<dependency>` ブロックを追加します。

```xml
<!-- pom.xml -->
<dependencies>
    <dependency>
        <groupId>com.google.gson</groupId>
        <artifactId>gson</artifactId>
        <version>2.10.1</version>
    </dependency>
</dependencies>
```

依存を追加後、`./gradlew build` または `mvn package` を実行してキャッシュを含めて再ビルドします。

**スコープによる実行時除外に注意：** `compileOnly`（Gradle）や `provided`（Maven）スコープは、コンパイルは通るが実行時 JAR に含まれません。`javax.servlet` 等のサーブレット API でよく起きます。アプリケーションサーバーが提供しない環境で実行するなら `implementation`/`compile` スコープに変更してください。

```groovy
// 誤り：実行時にクラスパスから外れる
compileOnly 'javax.servlet:javax.servlet-api:4.0.1'

// 修正：サーバーが提供しない場合は implementation に変更
implementation 'javax.servlet:javax.servlet-api:4.0.1'
```

**Fat JAR（uber JAR）を使う場合：** `java -jar` で実行する JAR は、依存クラスを内包する Fat JAR として作成する必要があります。Gradle なら `shadowJar` プラグイン、Maven なら `maven-assembly-plugin` や `maven-shade-plugin` を使って全依存をバンドルすることで、`java.lang.ClassNotFoundException` の再発を防げます。
