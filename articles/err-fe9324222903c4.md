## 発生条件

- プライベートリポジトリのイメージを `docker pull` または `docker run` しようとしたとき
- イメージ名やタグのスペルミス・存在しないタグを指定したとき
- Docker Hub や ECR、GHCRなどのレジストリにログインしていない状態でプルしようとしたとき

## 原因

`docker: Error response from daemon: pull access denied for` は、デーモンがイメージの取得を拒否したことを示すエラーである。主な原因は次の3つに集約される。

**1. 未ログイン状態でのプライベートイメージ取得**

```bash
# ログインせずにプライベートイメージをプル
docker pull myorg/private-app:latest
# => docker: Error response from daemon: pull access denied for myorg/private-app, ...
```

**2. イメージ名・タグの誤り**

```bash
# タグが存在しない、またはリポジトリ名が間違っている
docker run ghcr.io/myorg/myapp:v1.0.0
# => docker: Error response from daemon: pull access denied for ghcr.io/myorg/myapp, ...
```

**3. CI/CD環境での認証情報未設定**

```yaml
# GitHub Actions などで credentials を渡し忘れた例
- name: Run container
  run: docker run myorg/private-app:latest
  # レジストリへのログインステップが抜けている
```

レジストリ側からは「そのイメージへのアクセス権がない」または「イメージが存在しない」のどちらも同じ `pull access denied` として返るため、エラー文だけでは原因の切り分けができない点が混乱を招きやすい。

## 修正方法

**ステップ1: 対象レジストリにログインする**

```bash
# Docker Hub の場合
docker login

# GitHub Container Registry の場合
echo $GITHUB_TOKEN | docker login ghcr.io -u USERNAME --password-stdin

# AWS ECR の場合
aws ecr get-login-password --region ap-northeast-1 \
  | docker login --username AWS --password-stdin \
    123456789012.dkr.ecr.ap-northeast-1.amazonaws.com
```

**ステップ2: イメージ名とタグを確認する**

```bash
# 存在するタグの一覧を確認（Docker Hub の場合）
curl -s "https://hub.docker.com/v2/repositories/myorg/myapp/tags/" \
  | python -m json.tool | grep '"name"'

# ローカルにあるイメージ一覧を確認
docker images | grep myapp
```

**ステップ3: CI/CD での認証設定（GitHub Actions の例）**

```yaml
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Log in to GHCR
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Run container
        run: docker run ghcr.io/myorg/myapp:latest
```

`pull access denied for` が出た際は、まず `docker login` を実行して認証状態を確立し、次に `docker images` や Registry の UI でイメージ名・タグが正確かどうかを照合する順序で確認するとよい。パブリックイメージのつもりがプライベートに設定されているケースも多いため、レジストリ側の公開設定も併せて確認しておくこと。
