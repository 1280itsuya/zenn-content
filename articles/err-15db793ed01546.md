## 発生条件

- Node.js で `import` 文を使ったファイルを実行した際、`package.json` に `"type": "module"` が設定されていない場合
- `.js` 拡張子のファイルで ES Modules の構文を使い、CommonJS 環境（デフォルトの Node.js）で動かそうとした場合
- `ts-node` や `jest` などのツールが内部で CommonJS としてファイルを評価するのに、ソースが `import` 構文を使っている場合

## 原因

Node.js はデフォルトで CommonJS モジュールとしてファイルを解釈します。そのため、`import` 文を含むファイルを何の設定もなく実行すると `Cannot use import statement outside a module` が発生します。

以下が最小再現コードです。

```js
// index.js
import fs from 'fs';

console.log(fs.readFileSync('./package.json', 'utf8'));
```

```json
// package.json（問題のある状態）
{
  "name": "my-app",
  "version": "1.0.0",
  "scripts": {
    "start": "node index.js"
  }
}
```

```
$ node index.js
SyntaxError: Cannot use import statement outside a module
```

`"type"` フィールドが存在しない場合、Node.js は `.js` ファイルを CommonJS として扱うため、ESM 専用の `import` 構文を認識できません。

## 修正方法

### 方法1: `package.json` に `"type": "module"` を追加する

プロジェクト全体を ESM として扱う場合の対処です。

```json
// package.json（修正後）
{
  "name": "my-app",
  "version": "1.0.0",
  "type": "module",
  "scripts": {
    "start": "node index.js"
  }
}
```

この設定を加えると、`.js` ファイルが ESM として評価され、`import` 文がそのまま動作します。

### 方法2: `import` を `require` に書き換える

CommonJS のまま維持したい場合や、既存コードへの影響を最小限にしたい場合はこちらを選びます。

```js
// index.js（修正後）
const fs = require('fs');

console.log(fs.readFileSync('./package.json', 'utf8'));
```

### 方法3: ファイル拡張子を `.mjs` に変更する

`package.json` を変更せずに特定ファイルだけ ESM にしたい場合は、拡張子を `.mjs` にすることで Node.js が ESM として解釈します。

```
$ mv index.js index.mjs
$ node index.mjs
```

どの方法を選ぶかはプロジェクトの構成によります。新規プロジェクトであれば `"type": "module"` を設定して ESM に統一するのが一般的です。既存の CommonJS プロジェクトに一部だけ `import` を混在させている場合は、`require` への書き換えか `.mjs` 化が現実的な選択肢になります。なお、`"type": "module"` を設定したプロジェクトで CommonJS を使いたいファイルがある場合は、その拡張子を `.cjs` にすることで混在が可能です。
