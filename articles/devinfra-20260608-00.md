## 結論：[個人開発](https://www.amazon.co.jp/s?k=%E5%80%8B%E4%BA%BA%E9%96%8B%E7%99%BA%20%E6%9C%AC&tag=1280itsuya22-22)の最初の1台は「ConoHa VPS か Xserver VPS」、長期運用なら「さくらのVPS」

個人開発でVPSを選ぶとき、月数百円〜千円台の最小プランならどこも実用十分です。差が出るのは**初期構築の手間**と**転送量・スナップショットなどの周辺機能**です。結論から言うと、

- とにかく早く立てたい → **ConoHa VPS**（管理画面とテンプレが速い）
- 知名度と安定運用のバランス → **Xserver VPS**
- 長く塩漬けで持つ＋枯れた安心感 → **さくらのVPS**
- [ドメイン](https://px.a8.net/svt/ejp?a8mat=4B3XB4+4VMW8I+50+2HU3GX)とまとめて契約したい → **お名前.com**（VPSよりドメインで使う前提が無難）

> ⚠️ 料金は各社のキャンペーン・為替・スペック改定で頻繁に変わります。本文の金額は契約時点の一例で、**必ず公式の最新料金を確認**してください。

## 根拠：実務で効いた3観点

### 1. 料金感（最小構成の体感）
1〜2GBメモリの最小プランは、各社おおむね**月数百円〜1,000円台前半**のレンジに収まります。私が個人ブログ＋小さなAPIを動かしている構成は、長期契約割引込みで**例: 月額700円前後**で回せています。注意点は2つ。

- **長期一括前払いで単価が大きく下がる**ため、1ヶ月単価だけ見ると割高に見える
- 増強（メモリ追加・上位移行）の手数料や手順は各社差があるので、**伸びしろも含めて選ぶ**

### 2. SSH
どのサービスでもやることは同じで、最初にすべきは公開鍵運用への切り替えです。

```bash
# 鍵を作る（ローカル）
ssh-keygen -t ed25519 -C "vps-dev"

# 公開鍵を初回だけパスワードで送り込む
ssh-copy-id -i ~/.ssh/id_ed25519.pub root@<VPSのIP>
```

接続できたら `/etc/ssh/sshd_config` で最低限ここを締めます。

```text
PermitRootLogin no
PasswordAuthentication no
```

ConoHaやXserverは**作成時に公開鍵を登録**できるので、初回からパスワードログインを使わずに済むのが地味に楽でした。`~/.ssh/config` にホストを書いておくと運用が一気に軽くなります。

```text
Host mydev
  HostName <VPSのIP>
  User deploy
  IdentityFile ~/.ssh/id_ed25519
```

以後は `ssh mydev` だけで入れます。

### 3. 転送量・DNS
個人開発規模では**転送量で課金上限に当たることはまずない**のが実感で、ブログ・小規模APIなら気にしなくて問題ありませんでした。動画配信や大量画像を扱うなら、各社の転送量条件を契約前に確認してください。

DNSは「VPSの会社」と「ドメインの会社」を分けて考えるのがコツです。お名前.comでドメインだけ取り、VPSは別社、という構成も普通に組めます。最小限はAレコードを1本向けるだけです。

```text
example.com.     A     <VPSのIP>
www.example.com. CNAME example.com.
```

反映には数分〜時間がかかるので、切り替えは余裕を持って。

## 具体手順：契約から公開まで最短ルート

1. **プラン選定**：最小（1〜2GB）で契約。足りなければ後で上げる
2. **OS**：迷ったら Ubuntu LTS。情報量が多くトラブル解決が速い
3. **SSH鍵を作成時に登録** → 初回からパスワードログイン無効
4. **作業ユーザー作成**（`adduser deploy` ＋ sudo付与）、root直ログイン禁止
5. **ファイアウォール**：22/80/443 のみ開放（`ufw` でOK）
6. **DNS**：ドメイン会社側でAレコードをVPSのIPに向ける
7. **TLS**：Nginx＋Let's Encrypt（`certbot`）で自動更新まで設定

ここまでで小さなサービスは公開できます。VPS本体の差より、**鍵・ユーザー・ファイアウォール・DNSの初期設計**のほうが運用の快適さを左右する、というのが3社を触っての結論です。まず1台を最小プランで立て、手順を自分の `config` に固定してしまうのが一番の近道です。
