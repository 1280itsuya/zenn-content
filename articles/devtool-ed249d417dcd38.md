## GitHub Copilot とは

GitHub Copilot は、GitHub と OpenAI が共同開発したAI補完ツールで、VS Code・JetBrains・Neovim など主要エディタと統合して動作する。コードを書きながらリアルタイムで次の行や関数全体を提案してくれるため、定型処理やボイラープレートを繰り返しがちな業務開発者に特に向いている。TypeScript・Python・Go などの主要言語を日常的に書くエンジニアであれば、導入直後から恩恵を感じやすい。

## 起動速度

GitHub Copilot の補完応答はエディタ入力に追従するレベルで速い。関数名を書き始めた瞬間に候補が灰色のゴーストテキストとして表示され、体感的なラグはほぼない。ただし補完候補がサーバーサイドで生成されるため、ネットワーク品質が悪い環境では提案が遅れることがある。オフライン環境での動作は基本的に期待できない点は留意が必要だ。

## 補完精度

GitHub Copilot が特に強いのは、定型パターンの多いコードだ。たとえば `useEffect` のクリーンアップ関数、Expressのルートハンドラ、SQLのCRUDクエリなど「書き方が決まっている」コードは驚くほどそのまま使える補完が出てくる。一方、ドメイン固有のロジックや複雑な状態管理、自社のレガシーAPIとの連携コードなどは的外れな提案が混ざることが多い。コメントで意図を明確に書くと精度が上がる傾向はあるが、過信は禁物で最終的には自分で内容を読んで確認する習慣が不可欠だ。

## 月額コスト

GitHub Copilot の料金は個人向けが月額 $10（または年払いで $100）。学生および人気OSSメンテナは無料で利用できる公式プログラムがある。2024年以降、一部機能については月間使用量に上限を設けた無料枠が提供されるようになっており、試してから有料に移行するハードルは下がった。チームや企業向けには別途 Business / Enterprise プランがあり、ポリシー管理やコードスニペットの送信制御などが追加される。個人開発の用途では個人プランで十分なケースがほとんどだ。

## 使ってみた所感

副業で小さなWebアプリを作っているとき、GitHub Copilot が特に役立ったのはテストコードの記述だった。実装関数に対して `// test for` とコメントするだけで、それなりに網羅的なユニットテストの骨格を提案してくれる。テストは書くべきとわかっていてもつい後回しになりがちな作業なので、ここで手が止まらないのは体験として大きい。もちろん提案されたテストをそのままコピーするのではなく、境界値やエラーケースを自分で補う必要はある。それでも「最初の一歩」を出してくれるという意味で、一人で副業開発を回しているエンジニアには実用的なツールだと感じた。
