## 結論：X(旧Twitter)のリサーチは「収集だけ」を切り出せば、日2時間が15分に落ちる

副業で情報発信をしていると、ネタ探しのためにXを眺める時間が一番のコストになる。私は毎朝この作業に約2時間かけていたが、Claude（claude-opus-4）と X API v2 を組み合わせた収集スクリプトに置き換え、人間の作業を**15分**まで圧縮した。内訳は「ざっと目視で最終チェック5分＋投稿企画10分」だけだ。

## 何を自動化したか

やったことはシンプルで、3ステップに分けただけ。

1. **X API v2 の `recent search`** で、対象キーワード15件分の直近24時間ツイートを取得（無料枠でも月1万件まで読める）
2. 取得した最大300件を Claude に投げ、`要約・感情ラベル・バズ要因` の3項目をJSON厳格出力で返させる
3. 「いいね500以上かつ自分のニッチに関連」だけを Notion に自動追記

ポイントは、Claudeに「探させない」こと。検索とフィルタはAPIの数値条件で機械的に削り、**判断が要る部分だけ**を言語モデルに任せる。これでトークン消費は1日あたり約4万、コストは**月600円前後**に収まった。

## 効果（実測値）

| 項目 | 手動 | 自動化後 |
|---|---|---|
| 所要時間/日 | 約120分 | 約15分 |
| 拾えるツイート数 | 50件程度 | 300件 |
| 月コスト | 0円(時間) | 約600円 |

時間あたりの企画本数は3倍になり、空いた時間を執筆に回せるようになった。
