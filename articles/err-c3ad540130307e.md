## 発生条件

- [Python](https://www.amazon.co.jp/s?k=Python%20%E5%85%A5%E9%96%80%20%E6%9C%AC&tag=1280itsuya22-22)スクリプト内で `import requests` を実行したとき、`requests` パッケージがインストールされていない環境で実行した場合
- 仮想環境（venv）を複数使い分けており、`pip install requests` を行ったvenvとは別のvenvが有効化された状態でスクリプトを起動した場合
- システムPythonとvenv内Pythonが混在しており、意図しないPythonインタープリタがスクリプトを実行した場合

## 原因

`ModuleNotFoundError: No module named 'requests'` は、Pythonがモジュール検索パス上に `requests` パッケージを見つけられない場合に発生する。

最小再現コードは以下の通り。

```python
# main.py
import requests  # ← ここで ModuleNotFoundError が発生

response = requests.get("https://example.com")
print(response.status_code)
```

このエラーが出た際に確認すべきなのは、どのPythonインタープリタが使われているかと、そのインタープリタに `requests` がインストールされているかの2点。

```bash
# 現在使用されているPythonのパスを確認
which python   # macOS/Linux
where python   # Windows

# インストール済みパッケージ一覧で requests を探す
pip list | grep requests
```

`requests` が表示されなければ未インストール、もしくは別環境にインストールされている。

## 修正方法

venvを使っている場合は、対象のvenvを有効化した上で `pip install requests` を実行する。

```bash
# venvを作成（初回のみ）
python -m venv .venv

# venvを有効化
# macOS/Linux
source .venv/bin/activate

# Windows（PowerShell）
.venv\Scripts\Activate.ps1

# Windows（コマンドプロンプト）
.venv\Scripts\activate.bat

# 有効化後にインストール
pip install requests
```

有効化が成功していれば、プロンプトの先頭に `(.venv)` のようなvenv名が表示される。この状態でインストールすることで、同じvenv内のスクリプトから `import requests` が通るようになる。

```python
# インストール後は正常に動作する
import requests

response = requests.get("https://example.com")
print(response.status_code)  # 200
```

依存パッケージを `requirements.txt` で管理しているプロジェクトであれば、以下のようにまとめてインストールできる。

```bash
pip install -r requirements.txt
```

`requirements.txt` に `requests` が記載されていない場合は追記が必要。

```text
requests==2.32.3
```

バージョンを固定することで、チームや本番環境での再現性が高まる。
