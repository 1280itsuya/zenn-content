## 発生条件

- `terraform plan` や `terraform apply` 実行時に、使用するリソースに対応するプロバイダーが `required_providers` ブロックに記述されていない場合
- `provider` ブロック自体が `main.tf` などの設定ファイルに存在しない場合
- モジュールを別ディレクトリに分割した際、子モジュール側に必要な `provider` 設定が引き継がれていない場合

## 原因

`Error: Provider configuration not present` は、Terraformがリソースをどのプロバイダーで管理するか判断できないときに発生する。以下は最小再現コードの例だ。

```hcl
# main.tf（問題あり）
resource "aws_instance" "example" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"
}
```

このコードでは `terraform {}` ブロックも `provider "aws" {}` ブロックも存在しない。`terraform init` を実行しても、どのプロバイダープラグインをダウンロードすべきかTerraformが判断できないため、`plan` や `apply` の段階で `Error: Provider configuration not present` が出力される。

また、`required_providers` だけを書いて `provider` ブロックを省略するケースや、ワークスペースを切り替えた後に `terraform init` を再実行し忘れるケースでも同様のエラーが起きる。

## 修正方法

`terraform {}` ブロック内に `required_providers` を追加し、対応する `provider` ブロックを記述してから `terraform init` を実行する。

```hcl
# main.tf（修正後）
terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
}

provider "aws" {
  region = "ap-northeast-1"
}

resource "aws_instance" "example" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"
}
```

修正後は必ず以下のコマンドを順番に実行する。

```bash
terraform init
terraform plan
```

`terraform init` によってプロバイダープラグインが `.terraform/` ディレクトリにダウンロードされ、Terraformがプロバイダー設定を認識できる状態になる。これを省略すると修正後も同じエラーが継続する点に注意が必要だ。

子モジュールを使っている場合は、ルートモジュール側の `required_providers` と `provider` ブロックが正しく定義されているかを確認する。子モジュール内でプロバイダーを明示的に渡す必要がある場合は、モジュール呼び出し時に `providers` 引数を使う。

```hcl
# ルートモジュールから子モジュールへプロバイダーを渡す例
module "network" {
  source = "./modules/network"

  providers = {
    aws = aws
  }
}
```

`Error: Provider configuration not present` が出たときはまず `required_providers` の有無、次に `provider` ブロックの有無、最後に `terraform init` の実行済み状態を順に確認するのが確実な手順だ。
