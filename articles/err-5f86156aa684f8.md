## 発生条件

`Warning: Undefined array key` が発生するのは以下のような状況です。

- 配列に存在しないキーを直接参照したとき（例：`$arr['foo']` で `'foo'` キーが存在しない）
- フォームやAPIレスポンスなど外部入力を `$_POST['key']` や `$data['field']` で無検証アクセスしたとき
- ループや条件分岐でキーが設定されなかった場合に、後続処理でそのキーを参照したとき

PHP 8.0 以降、未定義キーへのアクセスは `Notice` から `Warning` に格上げされたため、以前は黙認されていたコードが警告を出すようになっています。

## 原因

以下の最小コードで `Warning: Undefined array key "status"` が再現できます。

```php
<?php
$response = ['name' => 'Alice'];

// 'status' キーが存在しないのに直接参照している
echo $response['status'];
```

このコードでは `$response` に `'status'` キーが存在しないため、PHPは `Undefined array key "status"` の Warning を発生させ、`null` を返します。フォーム送信で特定フィールドが省略された場合や、APIレスポンスの構造が変わった場合に実際のコードで頻繁に起きます。

```php
<?php
// $_POST から直接取得するパターンも同様
function handleForm(array $post): string {
    return $post['username']; // 'username' が送信されていなければ Warning
}
```

## 修正方法

存在チェックを挟む方法が複数あります。状況に応じて使い分けてください。

**`??`（null合体演算子）でデフォルト値を指定する（推奨）**

```php
<?php
$response = ['name' => 'Alice'];

// キーが存在しなければ 'unknown' を使う
$status = $response['status'] ?? 'unknown';
echo $status; // 'unknown'
```

**`isset()` で存在確認してから参照する**

```php
<?php
$response = ['name' => 'Alice'];

if (isset($response['status'])) {
    echo $response['status'];
} else {
    echo 'status not found';
}
```

**`array_key_exists()` を使う（値が `null` の場合も検出したいとき）**

```php
<?php
$data = ['flag' => null];

// isset() は null を「存在しない」と見なすが、array_key_exists() はキー自体を確認する
if (array_key_exists('flag', $data)) {
    $flag = $data['flag']; // null が正当な値として扱われる
}
```

`isset()` と `??` は値が `null` のキーに対してはキーが存在しないと同様に扱います。`null` 自体が有効な値として設定される可能性があるなら `array_key_exists()` を選んでください。

外部入力（`$_POST`・`$_GET`・JSONデコード結果など）を扱う箇所では、`Undefined array key` の Warning が出ないよう、すべてのキー参照に `??` またはチェック関数を添えるのがシンプルで保守しやすい対策です。
