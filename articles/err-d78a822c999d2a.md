## 発生条件

- 同じ変数に対して不変借用（`&T`）が生きている間に、可変借用（`&mut T`）を取ろうとしたとき
- ループや関数呼び出しをまたいで不変参照を保持したまま、同じコレクションを変更しようとしたとき
- `Vec` などのインデックス操作で `v[i]` の参照を保持しつつ `v.push()` や `v[j] = ...` を行ったとき

## 原因

Rust の借用チェッカーは「不変借用が存在する間は可変借用を許可しない」というルールを強制する。以下のコードは `error[E0502]: cannot borrow as mutable because it is also borrowed as immutable` を引き起こす典型例だ。

```rust
fn main() {
    let mut v = vec![1, 2, 3];

    let first = &v[0];      // 不変借用が始まる
    v.push(4);              // ここで可変借用 → E0502
    println!("{}", first);  // firstはまだ生きている
}
```

`first` は `v` への不変参照を保持している。その状態で `v.push(4)` は `v` の可変借用を要求するため、借用チェッカーが拒否する。`Vec` の再アロケーションが起きると `first` が指す先がダングリングポインタになるため、このルールはメモリ安全性を守るための必然的な制約だ。

同様のパターンはスライスのインデックス操作でも頻出する。

```rust
fn main() {
    let mut data = vec![10, 20, 30];

    let x = &data[0];   // 不変借用
    data[1] = 99;       // 可変借用 → E0502
    println!("{}", x);
}
```

## 修正方法

最も単純な修正は、**不変借用のスコープを先に終わらせてから可変借用を行う**ことだ。

```rust
fn main() {
    let mut v = vec![1, 2, 3];

    // 不変借用が必要な処理を先に完結させる
    let val = v[0];         // 値をコピーして借用を終了させる
    v.push(4);              // 不変借用が消えたので可変借用が可能
    println!("{}", val);
}
```

`v[0]` の型が `Copy` を実装していれば値をコピーすることで借用を即座に終わらせられる。

スライスの2つの可変部分を同時に操作したい場合は `split_at_mut` を使う。これは1つの `&mut [T]` を重ならない2つの `&mut [T]` に安全に分割するための標準的な手段だ。

```rust
fn main() {
    let mut data = vec![1, 2, 3, 4, 5];

    // インデックス0と2を同時に可変参照したい場合
    let (left, right) = data.split_at_mut(2);
    left[0] = 100;
    right[0] = 200;  // data[2] に相当
    println!("{:?}", data); // [100, 2, 200, 4, 5]
}
```

`split_at_mut` はインデックスが重ならないことを実行時に保証し、unsafe を使わずに複数の可変参照を得られる。`error[E0502]` に直面したときは、まず「不変借用のライフタイムを短くできるか」「値のコピーで済むか」を検討し、それが難しいスライス操作には `split_at_mut` を検討するとよい。
