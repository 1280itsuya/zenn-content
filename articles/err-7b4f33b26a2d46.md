## 発生条件

- `node app.js`（または `npm start`）で起動した直後に `Error: Cannot find module 'express'` が出る
- `git clone` 直後や ZIP 展開直後など、まだ `npm install` を実行していない状態でアプリを動かそうとした
- モノレポやサブディレクトリ構成で、`node_modules` の無い場所を作業ディレクトリにして起動した、または `node_modules` が壊れている

## 原因

Node.js は `require('express')` を実行した際、カレントから親へ向かって `node_modules/express` を探索します。どこにも見つからないと `Cannot find module 'express'` を投げて停止します。つまり「コードの誤り」ではなく「依存パッケージが解決できない」という環境側の問題です。

最小再現コードは以下です。`express` をインストールせずに実行すると必ず再現します。

```js
// app.js
const express = require('express'); // ここで解決に失敗する
const app = express();

app.get('/', (req, res) => res.send('ok'));
app.listen(3000);
```

```bash
# express が無い状態で実行
node app.js
# => Error: Cannot find module 'express'
```

`package.json` に `express` が記載されていても、`node_modules` が無ければ同じエラーになります。記載の有無と「実際にインストール済みか」は別物だという点に注意してください。

## 修正方法

最も基本的な対処は、正しい作業ディレクトリで `express` をインストールすることです。

```bash
# package.json があるディレクトリへ移動してから実行
npm install express

# package.json に依存が書かれている場合はまとめて入れる
npm install
```

`node_modules` が壊れている疑いがある場合は、一度削除してから入れ直します。

```bash
rm -rf node_modules package-lock.json
npm install
```

要点は次の3つです。

- **作業ディレクトリを確認する**: モノレポやサブパッケージでは、`package.json` のある正しい階層で `npm install` を実行する。別の階層で入れても `Cannot find module 'express'` は解消されない。
- **インストール結果を確認する**: `node_modules/express` が実在するか、`npm ls express` でバージョンが表示されるかを確認する。
- **package.json へ記録する**: `npm install express` は依存を `dependencies` に追記する。これにより他の環境で `npm install` だけで再現でき、同じエラーの再発を防げる。

これで `require('express')` が解決され、エラーは解消されます。
