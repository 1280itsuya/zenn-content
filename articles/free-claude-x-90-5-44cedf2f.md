## 結論：Xのタイムラインは「読む」より先に「捨てる」

毎朝のAI業界キャッチアップに90分かけていたのを、Claudeに「ノイズ判定」をさせる仕組みで実測5分まで圧縮した。やったことは単純で、Xのフォロー中ツイートをAPIで取得し、Claude（claude-haiku-4-5）に1件ずつ「自分の関心軸に対して読む価値があるか」を0〜100でスコアリングさせ、閾値未満を自動で非表示にしただけだ。

3週間運用した数字がこれ。

| 指標 | 導入前 | 導入後 |
|---|---|---|
| 1日の処理ツイート数 | 約420件 | 約420件（変わらず） |
| 自分が実際に開いた数 | 全部スクロール | 18件 |
| 所要時間 | 88分 | 5.2分 |
| 判定コスト | — | 1日 約¥4 |

420件読むのをやめたのではなく、Claudeに419件→18件まで間引かせた。残った18件の精度（読んでよかった率）は83%だった。

## 仕組みは3パーツだけ

1. **収集**：X API v2 の `GET /2/users/:id/timelines/reverse_chronological` で直近のツイートを取得。無料枠でも回る範囲に絞る。
2. **判定**：Claude にプロンプトで関心軸（例：「LLMの実装・価格・ベンチマーク。ポエムと採用報告は不要」）を渡し、各ツイートを `{score, reason, tags}` のJSONで返させる。Haikuで十分、1000件でも数円。
3. **出力**：スコア70以上だけをMarkdownのダイジェストにして、毎朝7時にタスクスケジューラで自分のSlackへ投げる。

肝は「要約」ではなく「ミュート判定」にClaudeを使うこと。要約させると幻覚と読み込みコストが増えるが、捨てる/残すの二値なら精度もコストも安定する。

## ノイズ判定プロンプトの骨格

```python
SYSTEM = """あなたは情報フィルターです。
以下のツイートを、ユーザーの関心軸に照らして0-100で採点。
- 関心軸: {axis}
- 宣伝/バズ狙い/中身のない感想は30以下
- 一次情報・コード・数値を含むものは加点
JSONのみ返す: {"score": int, "reason": str}"""
```

`reason` を必ず返させるのがコツで、後から閾値を調整するとき「なぜ70点なのか」が読めると、自分の関心軸の言語化そのものが改善していく。
