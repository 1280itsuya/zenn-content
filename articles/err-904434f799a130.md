## 発生条件

- ターミナルで打ち込んだコマンド名が `PATH` に含まれるディレクトリ内に存在しない
- ソフトウェアをインストールしていない、またはインストールが途中で失敗している
- コマンド名をタイポしている（例：`pytohn`、`nod` など）

## 原因

シェルはコマンドを実行する際、環境変数 `PATH` に列挙されたディレクトリを順番に検索する。該当するバイナリが見つからなかった場合に `command not found` が発生する。

以下は典型的な再現例：

```bash
# node がインストールされていない、または PATH に含まれていない状態
$ node -v
bash: node: command not found

# タイポの例
$ pytohn script.py
bash: pytohn: command not found
```

```bash
# 現在の PATH を確認
$ echo $PATH
/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin
# node の実体がどこにあるか（インストール済みなら表示される）
$ which node
# 何も返らない = PATH に無い、または未インストール
```

`which コマンド名` が何も返さない場合、そのコマンドは現在のシェル環境から見えていない。

## 修正方法

### 1. インストールされているか確認してからインストール

```bash
# Node.js の例（Ubuntu/Debian）
$ which node || sudo apt-get install -y nodejs

# [Python](https://www.amazon.co.jp/s?k=Python%20%E5%85%A5%E9%96%80%20%E6%9C%AC&tag=1280itsuya22-22) の例（macOS + Homebrew）
$ which python3 || brew install python
```

### 2. PATH にパスを追加する

コマンドは存在するが PATH に含まれていない場合、シェルの設定ファイルに追記する：

```bash
# ~/.bashrc または ~/.zshrc に追記
export PATH="$HOME/.local/bin:$PATH"

# 追記後、設定を反映
source ~/.zshrc
```

追記するパスは `which コマンド名` や `find / -name コマンド名 2>/dev/null` で実体の場所を確認してから指定する。

### 3. タイポを修正する

```bash
# 誤り
$ pytohn script.py   # command not found

# 正しいコマンド名に修正
$ python3 script.py
```

### 4. バージョン管理ツール経由のインストール後にシェルを再起動

`nvm`、`pyenv`、`rbenv` などでインストールした場合、新しいシェルセッションを開くか `exec $SHELL` を実行しないと PATH が更新されていないことがある：

```bash
# nvm で Node をインストールした直後
$ nvm install 20
$ exec $SHELL   # シェルを再初期化して PATH を更新

$ node -v
v20.x.x   # 今度は見つかる
```

`command not found` が出たら、まず `which` でコマンドの存在を確認し、なければインストール、あれば `echo $PATH` でパスが通っているかを確認する、という順序で切り分けるとすばやく解決できる。
