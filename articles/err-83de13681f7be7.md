## 発生条件

- PHPスクリプトの実行中にメモリ使用量が `memory_limit` の設定値を超えたとき
- 大きなCSVやJSONファイルを一括で配列に読み込んだとき
- 再帰処理やループ内でオブジェクトを大量生成し、ガベージコレクタが追いつかないとき

## 原因

`Fatal error: Allowed memory size exhausted` は、PHPプロセスが `php.ini` で定められたメモリ上限に達した際に発生する致命的エラーです。典型的なパターンとして、大きなファイルを `file()` や `json_decode(file_get_contents(...))` で一括ロードするケースが挙げられます。

```php
<?php
// 数十万行のCSVを一度に配列へ読み込む
$lines = file('/var/data/huge.csv');  // ここで Allowed memory size exhausted

foreach ($lines as $line) {
    process($line);
}
```

`file()` はファイル全体をメモリ上の配列として保持するため、ファイルサイズに比例してメモリを消費します。100MBのファイルなら実質的に100MB以上のヒープを占有します。

## 修正方法

対処は大きく2つです。

### 1. memory_limit を引き上げる（暫定対処）

```php
<?php
// スクリプト先頭で上限を増やす
ini_set('memory_limit', '512M');

$lines = file('/var/data/huge.csv');
foreach ($lines as $line) {
    process($line);
}
```

`php.ini` を直接変更する場合:

```ini
; php.ini
memory_limit = 512M
```

上限引き上げは即効性がありますが、ファイルがさらに大きくなれば再発します。根本的な解決にはなりません。

### 2. ストリーム処理に切り替える（推奨）

```php
<?php
// ファイルを1行ずつ読み込み、メモリに全行を保持しない
$handle = fopen('/var/data/huge.csv', 'r');
if ($handle === false) {
    throw new RuntimeException('ファイルを開けませんでした');
}

while (($line = fgets($handle)) !== false) {
    process($line);
}

fclose($handle);
```

`fgets()` は1行分のデータだけをメモリに載せて処理するため、ファイルサイズに関わらずメモリ使用量はほぼ一定に保たれます。

JSONの場合は `json_decode(file_get_contents(...))` の代わりに、`ext-json` の streaming parser や [`JsonStreamingParser`](https://github.com/salsify/jsonstreamingparser) のようなライブラリを検討してください。

```php
<?php
// 大きなJSONLファイル（1行1JSONオブジェクト）をストリーム処理する例
$handle = fopen('/var/data/records.jsonl', 'r');

while (($line = fgets($handle)) !== false) {
    $record = json_decode(trim($line), true);
    if ($record !== null) {
        process($record);
    }
}

fclose($handle);
```

まず `ini_set('memory_limit', ...)` で動作を確認し、その後ストリーム処理へ移行するのが現実的な進め方です。`Allowed memory size exhausted` が繰り返し発生する場合、上限を増やすだけでは対症療法にとどまるため、データの読み込み方を見直すことを優先してください。
