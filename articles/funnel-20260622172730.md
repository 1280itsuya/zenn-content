# Claude CodeにMCPサーバーを追加してファイル操作・コマンド実行を自動化する最小手順

Claude Codeは標準でもかなり使えるが、MCPサーバーを追加すると「Claude自身がツールを呼ぶ」範囲が広がり、繰り返し作業を委譲できる。この記事では公式MCPサーバーを2本追加して動作確認するまでの手順を示す。

## 前提

- Claude Code CLI インストール済み（`claude` コマンドが通る）
- Node.js 18以上（`npx` が使える）
- macOS / Linux / WSL（Windows PowerShellは後述の注意あり）

## 手順1: 設定ファイルの場所を確認する

MCPサーバーはプロジェクトごとの `.claude/settings.json` か、グローバルの `~/.claude/settings.json` に登録する。

```bash
# プロジェクトローカルに作る場合
mkdir -p .claude
touch .claude/settings.json
```

## 手順2: filesystem MCPサーバーを追加する

Anthropic公式の `@modelcontextprotocol/server-filesystem` は、指定ディレクトリ以下のファイル読み書きをClaudeに許可する。

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/absolute/path/to/your/project"
      ]
    }
  }
}
```

`/absolute/path/to/your/project` は実際のプロジェクトルートに変える。相対パスは動作しないことがあるので絶対パス推奨。

## 手順3: fetch MCPサーバーを追加する（任意）

外部URLのHTMLをClaudeに取得させたい場合は `@modelcontextprotocol/server-fetch` を追加する。

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/your/project"]
    },
    "fetch": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-fetch"]
    }
  }
}
```

## 手順4: 動作確認

`claude` を起動してMCPが認識されているか確認する。

```bash
claude
# 起動後に /mcp と入力するとサーバー一覧が表示される
```

`filesystem` と `fetch` が `connected` 状態になっていれば成功。`failed` のときは次の手順でデバッグする。

## つまずきポイントと対処

**npxがパスを解決できない**
`command` を `"npx"` にしていてもシェルのPATHが引き継がれない場合がある。その場合はフルパスを指定する。

```json
"command": "/usr/local/bin/npx"
```

**Windowsでコマンドが見つからない**
PowerShellでは `npx` の代わりに `npx.cmd` と書く必要がある。

```json
"command": "npx.cmd"
```

**アクセス拒否エラー**
`server-filesystem` は第三引数のパス以外には触れない。ルート `/` を指定するのはセキュリティリスクになるので、プロジェクトルートを必要最小限の範囲で指定する。

**サーバーがすぐ落ちる（exit 1）**
`npx -y` の `-y` を省くと初回に確認プロンプトが出てClaude Codeとのやり取りが詰まる。`-y` は必須。

## 実際に使う例

設定後、Claude Codeに次のように頼める。

```
src/以下の.tsファイルをすべてリストアップして、console.logが残っているファイル名を教えて
```

このときClaudeはMCPのfilesystemツールを使ってファイルを読みにいく。手動でgrepする手間が省ける。

## まとめ

1. `.claude/settings.json` に `mcpServers` ブロックを書く
2. `npx -y` でサーバーパッケージを指定する（絶対パス推奨）
3. 起動後 `/mcp` で接続状態を確認する
4. エラーは `command` のパスと引数の `-y` 有無を最初に疑う

公式リポジトリ `modelcontextprotocol/servers` にはGitHub・Slack・PostgreSQL向けサーバーなども公開されている。同じ形式で追記するだけで拡張できる。
