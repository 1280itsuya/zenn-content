## 発生条件

- `kubectl apply` でマニフェストを適用した直後、Pod の STATUS が `ImagePullBackOff` になる
- プライベートレジストリのイメージを Pull しようとしたが、認証情報が Kubernetes に渡っていない
- イメージ名やタグを typo した、または対象タグが削除・リネームされてレジストリに存在しない

## 原因

`ImagePullBackOff` は kubelet がコンテナイメージの取得に失敗し、指数バックオフで再試行している状態を示すステータスです。`kubectl describe pod <pod名>` の Events 欄に `Failed to pull image` と表示されます。

典型的な問題が起きるマニフェストは次のとおりです。

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: myapp
spec:
  containers:
    - name: myapp
      image: myregistry.example.com/myapp:lastest  # typo: "lastest" → "latest"
  # imagePullSecrets が未設定のままプライベートレジストリを参照
```

この例では二つの問題があります。タグ名の typo（`lastest`）と、プライベートレジストリへの認証情報が欠落していることです。

## 修正方法

**Step 1: イメージ名・タグを正しく修正する**

```bash
# 利用可能なタグを事前確認してからマニフェストに記載する
docker pull myregistry.example.com/myapp:latest
```

```yaml
# 修正後マニフェスト
containers:
  - name: myapp
    image: myregistry.example.com/myapp:latest  # タグを正確に指定
```

**Step 2: プライベートレジストリ用の Secret を作成し `imagePullSecrets` を設定する**

```bash
# レジストリ認証情報を Secret として登録する
kubectl create secret docker-registry myregistry-secret \
  --docker-server=myregistry.example.com \
  --docker-username=<USERNAME> \
  --docker-password=<PASSWORD> \
  --docker-email=<EMAIL>
```

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: myapp
spec:
  containers:
    - name: myapp
      image: myregistry.example.com/myapp:latest
  imagePullSecrets:
    - name: myregistry-secret  # 作成した Secret を参照
```

修正後に Pod を再作成し、ステータスが `ImagePullBackOff` から `Running` に変わることを確認します。

```bash
kubectl delete pod myapp
kubectl apply -f pod.yaml
kubectl get pod myapp -w
```

Public レジストリ（Docker Hub）の場合でも、レートリミットや一時的なネットワーク障害で同エラーが出ることがあります。その場合は `kubectl describe pod <pod名>` の Events に表示されるエラーメッセージを確認し、`pull rate limit exceeded` のような文言が含まれていれば認証済みアカウントの Secret を追加することで回避できます。
