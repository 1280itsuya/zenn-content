# GitHub ActionsとClaude APIでZenn記事を毎朝自動投稿するCI/CDパイプライン構築

個人ブログの更新が続かない原因の多くは「書く習慣がない」ではなく「デプロイが面倒」か「ネタ出しで詰まる」のどちらかだ。本記事では GitHub Actions のスケジュール実行と Anthropic の Claude API を組み合わせ、毎朝決まった時間に Zenn へ記事を自動投稿するパイプラインを実装する。

---

## 構成の全体像

```
GitHub Actions (cron 毎朝7時)
  └─ Python スクリプト
       ├─ Claude API でテーマ選定 + 本文生成
       ├─ articles/*.md として保存
       └─ git push → Zenn の GitHub 連携で自動公開
```

Zenn は GitHub リポジトリと連携できるため、`articles/` 以下に Markdown を push するだけで公開される。CI 側でやることは「生成 → commit → push」の3ステップだけだ。

---

## 事前準備

1. `zenn-cli` で記事リポジトリを初期化し GitHub に push する
2. Zenn の管理画面でリポジトリ連携を有効化
3. Anthropic の API キーを取得し、GitHub リポジトリの **Settings → Secrets** に `ANTHROPIC_API_KEY` として登録
4. push 用に Personal Access Token (PAT) を発行し `GH_PAT` として登録（`contents: write` 権限のみ付与する。Secrets に登録後は値の確認ができないため、発行直後にコピーすること）

## Python スクリプト（`scripts/generate_article.py`）

```python
import anthropic
import datetime
import re
import subprocess
import sys

TOPICS = [
    "GitHub Actionsのマトリックスビルドでテスト時間を短縮する",
    "Pythonの型ヒントを段階的に導入する実践手順",
    "Docker Composeで本番と開発環境の差分を最小化する",
]

def pick_topic() -> str:
    idx = datetime.date.today().toordinal() % len(TOPICS)
    return TOPICS[idx]

def generate(topic: str) -> str:
    client = anthropic.Anthropic()
    message = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=2048,
        messages=[{
            "role": "user",
            "content": (
                f"現役エンジニア向けに次のテーマで技術記事を書いてください。\n"
                f"テーマ: {topic}\n"
                f"要件: Markdown形式・1500字前後・コード例を含む・誇大表現なし\n"
                f"1行目を '# タイトル' で始めること。"
            )
        }]
    )
    return message.content[0].text

def make_slug(topic: str) -> str:
    date_str = datetime.date.today().strftime("%Y%m%d")
    safe = re.sub(r"[^\w]", "-", topic[:20], flags=re.ASCII)
    return f"{date_str}-{safe.lower().strip('-')}"

def save(content: str, slug: str) -> str:
    header = (
        "---\n"
        f"title: \"{content.splitlines()[0].lstrip('# ')}\"\n"
        "emoji: \"🛠\"\n"
        "type: \"tech\"\n"
        "topics: []\n"
        "published: true\n"
        "---\n\n"
    )
    body = "\n".join(content.splitlines()[1:]).lstrip("\n")
    path = f"articles/{slug}.md"
    with open(path, "w", encoding="utf-8") as f:
        f.write(header + body)
    return path

if __name__ == "__main__":
    topic = pick_topic()
    print(f"Topic: {topic}", file=sys.stderr)
    content = generate(topic)
    slug = make_slug(topic)
    path = save(content, slug)
    print(path)
```

## GitHub Actions ワークフロー（`.github/workflows/auto_post.yml`）

```yaml
name: auto-post

on:
  schedule:
    - cron: "0 22 * * *"   # UTC 22:00 = JST 翌7:00
  workflow_dispatch:         # 手動実行もできるようにする

jobs:
  post:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          token: ${{ secrets.GH_PAT }}

      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"

      - run: pip install anthropic

      - name: Generate article
        id: gen
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          path=$(python scripts/generate_article.py)
          echo "path=$path" >> "$GITHUB_OUTPUT"

      - name: Commit and push
        env:
          GIT_TERMINAL_PROMPT: "0"
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add articles/
          git diff --cached --quiet && echo "No changes" && exit 0
          git commit -m "auto: ${{ steps.gen.outputs.path }}"
          git push
```

`GIT_TERMINAL_PROMPT: "0"` は認証ダイアログ待ちによるハングを防ぐために必須。PAT を `actions/checkout` の `token` に渡すことで push が通る。

## つまずきポイント

**cron が JST で動かない**
GitHub Actions の `schedule` は UTC 固定。「JST 7時 = UTC 22時（前日）」なので `0 22 * * *` とする。

**Zenn の `published: true` を忘れると下書きになる**
フロントマターの `published` が `false` または欠落していると Zenn に公開されない。スクリプト側で必ず `true` を埋めること。

**同じ記事が push されない**
`git diff --cached --quiet` で差分ゼロを検出し `exit 0` するガードを入れておかないと、Actions がエラーで終了する。

**API コストの見積もり**
claude-sonnet-4-6 は 1 記事生成あたりおおよそ 1,000〜2,000 トークンの出力になる。公式の料金ページで最新単価を確認したうえで月間コストを見積もること（本記事では数値を記載しない）。

## 拡張の方向

- `TOPICS` リストを CSV や Notion データベースから読み込むと外部から管理できる
- 生成後に別の Claude 呼び出しで「タイトルと本文が一致しているか」を検証するステップを挟むと品質が上がる
- Slack Incoming Webhook を最後のステップで叩けば投稿通知を受け取れる

まずは `workflow_dispatch` で手動実行して動作を確認し、問題がなければ `schedule` を有効化するのが安全な進め方だ。
