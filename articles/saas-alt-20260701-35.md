## 結論（おすすめ1つ）

**Outline** に乗り換えることを勧める。

理由は三つ。まず、OSS かつ Docker でセルフホスト可能なため、Notion へのベンダーロックインを断ち切れる。次に、Markdown がファーストクラス市民として扱われており、エディタの挙動が「書くことを邪魔しない」。そして Slack / GitHub / Linear との公式連携が標準搭載されており、開発チームの既存ワークフローにそのまま接続できる。

## 比較表（料金/無料枠/移行コスト/対応言語）

| ツール | 料金体系 | 無料枠 | Notion からの移行コスト | 主な対応言語 |
|---|---|---|---|---|
| **Outline** | クラウド版有料・セルフホスト無料 | セルフホストは完全無料 | 低（HTML→MD変換スクリプト1本） | 英語・日本語ほか多言語UI |
| **Obsidian** | 個人無料・商用/同期有料 | 公式料金ページで要確認 | 中（手動フォルダ整理が必要） | 英語UI（日本語コンテンツは問題なし） |
| **GitBook** | 無料枠あり・チームプラン有料 | 公式料金ページで要確認 | 低〜中（Gitリポジトリ連携あり） | 英語UI |
| **Confluence** | ユーザー数課金 | 小規模無料枠 公式要確認 | 高（概念モデルが大きく異なる） | 英語・日本語UI |
| **Slab** | ユーザー数課金 | 公式料金ページで要確認 | 低（Notionインポート対応） | 英語UI |

## 移行手順

### 1. Notion からエクスポート

Notion の設定画面から「すべてのコンテンツをエクスポート」を実行し、**HTML 形式**を選択する。Markdown 形式は画像パスが壊れやすいため HTML を推奨。

```bash
# ダウンロードした ZIP を展開
unzip notion-export.zip -d ./notion-export
```

### 2. HTML → Markdown 変換

```bash
# pandoc がなければ先にインストール
# macOS: brew install pandoc
# Ubuntu: sudo apt install pandoc

# notion-export 以下の HTML を再帰的に MD へ変換
find ./notion-export -name "*.html" | while read f; do
  out="${f%.html}.md"
  pandoc "$f" -f html -t markdown_strict --wrap=none -o "$out"
  echo "converted: $out"
done
```

### 3. Outline をセルフホストで起動

```yaml
# docker-compose.yml（最小構成）
version: "3"
services:
  outline:
    image: outlinewiki/outline:latest
    env_file: .env
    ports:
      - "3000:3000"
    depends_on:
      - postgres
      - redis
  postgres:
    image: postgres:15
    environment:
      POSTGRES_USER: outline
      POSTGRES_PASSWORD: outline
      POSTGRES_DB: outline
  redis:
    image: redis:7
```

```bash
# 必要な環境変数を設定してから起動
cp .env.example .env
# .env の SECRET_KEY / DATABASE_URL / REDIS_URL を埋める
docker compose up -d
```

### 4. ドキュメントのインポート

```bash
# Outline の REST API 経由でバルクインポート
# 変換済み MD ファイルを1件ずつ POST する例
for f in ./notion-export/**/*.md; do
  curl -s -X POST "https://your-outline.example.com/api/documents.import" \
    -H "Authorization: Bearer $OUTLINE_API_TOKEN" \
    -F "file=@$f" \
    -F "publish=true" \
    -F "collectionId=$TARGET_COLLECTION_ID"
  sleep 0.5  # レートリミット回避
done
```

### 5. 画像の移行確認

```bash
# MD 内の画像参照が broken になっていないか確認
grep -rE '!\[.*\]\((?!http)' ./notion-export/ | head -20
# ローカルパスが残っている場合は手動で Outline にアップロードするか
# CDN URL へ置換する
```

## 向き不向き

**向いているチーム・ケース**

- 5〜50人規模のプロダクト開発チームで、社内 Slack / GitHub を日常的に使っている
- データを自社サーバーに置きたい（SOC 2 / GDPR 対応やセキュリティポリシーが厳しい）
- Markdown で書きたいが Notion のブロックエディタの癖に疲れた[エンジニア](https://www.amazon.co.jp/s?k=%E3%82%A8%E3%83%B3%E3%82%B8%E3%83%8B%E3%82%A2%20%E6%9C%AC&tag=1280itsuya22-22)
- GitBook は「公開 API ドキュメント」に特化させ、内部 wiki は Outline で分けたいチーム
- コスト管理をしたいスタートアップ（セルフホストなら月額ゼロで運用可能）

**避けるべきケース**

- デザイナーや非エンジニアが大多数を占めるチーム：Outline の UI は開発者向けに最適化されており、リッチなビジュアル編集を求めるメンバーには馴染みにくい
- 既に Atlassian スタック（Jira + Confluence）を全社導入済みの大企業：Confluence との連携深度は代替ツールで再現しにくい
- Notion のデータベース機能（カンバン・カレンダー・ギャラリービュー）を本格活用している：Outline はドキュメント特化であり、Notion のデータベース的用途には対応しない
- インフラ管理コストをゼロにしたい少人数チーム：セルフホストは Docker 管理が必要なため、クラウド版を選ぶなら公式料金ページで費用対効果を確認すること
