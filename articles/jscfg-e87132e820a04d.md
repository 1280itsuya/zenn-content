## 発生条件

- TypeScript プロジェクトで `import` したモジュールの型定義ファイル（`.d.ts`）が存在しない、または `tsconfig.json` の `paths` / `baseUrl` が未設定のとき
- `@types/xxx` パッケージがインストールされていない状態で、型定義を持たない npm パッケージを利用したとき
- モノレポ構成などで `tsconfig.json` の `include` / `exclude` 設定が不正確なため、参照先ファイルが TypeScript コンパイラの対象外になっているとき

## 原因

典型的な最小再現を以下に示す。

```json
// tsconfig.json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@utils/*": ["src/utils/*"]
    }
  }
}
```

```ts
// src/components/Button.tsx
import { formatDate } from "@utils/date"; // エラー: Cannot find module '@utils/date' or its corresponding type declarations
```

このエラーは主に次の2つの要因で発生する。

**1. `paths` を設定したが `moduleResolution` が対応していない**

`moduleResolution` のデフォルト値は TypeScript のバージョンや `module` オプションによって変わる。`node` モードでは `paths` エイリアスを解決できるが、`bundler` モードを指定しているのに対応するリゾルバが設定されていないケースがある。また、Vite や webpack 側のエイリアス設定と `tsconfig.json` の `paths` が食い違っていると、ビルドツールは解決できてもエディタやコンパイラがエラーを出す。

**2. 型定義ファイルが存在しない**

`@types/xxx` が未インストールの場合、または純粋な JavaScript ライブラリで型定義が同梱されていない場合、TypeScript は型情報を見つけられない。この場合はプロジェクト内に `xxx.d.ts` を自分で用意するか、`@types/xxx` をインストールする必要がある。

## 修正方法

**パスエイリアスのケース**

```json
// tsconfig.json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@utils/*": ["src/utils/*"]
    },
    "moduleResolution": "bundler"
  },
  "include": ["src"]
}
```

`moduleResolution` を実際のビルド環境（Vite なら `"bundler"`、Node.js 系なら `"node16"` や `"nodenext"`）に合わせる。`include` が `src` を含んでいることも確認する。

Vite を使っている場合は `vite.config.ts` 側にも同じエイリアスを追加する。

```ts
// vite.config.ts
import { defineConfig } from "vite";
import path from "path";

export default defineConfig({
  resolve: {
    alias: {
      "@utils": path.resolve(__dirname, "src/utils"),
    },
  },
});
```

**型定義が存在しないパッケージのケース**

```bash
npm install -D @types/xxx
```

`@types` パッケージが存在しないライブラリには、プロジェクト内に宣言ファイルを追加する。

```ts
// src/types/xxx.d.ts
declare module "xxx" {
  export function someFunction(arg: string): void;
}
```

`tsconfig.json` の `include` にこのファイルが含まれる場所を指定しておくことで、コンパイラが認識する。エラーメッセージに出るモジュール名と `declare module "..."` の文字列を完全一致させることが重要で、スラッシュやサブパスの有無でも別モジュールとして扱われる点に注意する。
