# [Claude](https://www.anthropic.com/) Code常駐用に選ぶ格安VPSの条件

## 結論：メモリ4GB・常時SSH・固定IPの3点を満たす最安プランを選ぶ

Claude Code を VPS に常駐させて「毎朝のタスク実行」「PR レビュー」「定期バッチ」を自走させたい[個人開発](https://www.amazon.co.jp/s?k=%E5%80%8B%E4%BA%BA%E9%96%8B%E7%99%BA%20%E6%9C%AC&tag=1280itsuya22-22)者向けに、まず結論から書く。選定基準はシンプルで、**(1) メモリ 4GB 以上、(2) SSH 常時接続が安定、(3) IP が固定**。この 3 つを満たす最安プランを選べば十分で、CPU コア数やストレージ速度は二の次でいい。国内なら ConoHa VPS / Xserver VPS / さくらのVPS あたりが候補、[ドメイン](https://px.a8.net/svt/ejp?a8mat=4B3XB4+4VMW8I+50+2HU3GX)を一緒に取るなら お名前.com も視野に入る。

## 根拠：Claude Code の常駐は「軽いが落ちると困る」ワークロード

Claude Code 自体の処理はクラウド側（Anthropic の API）で走るため、VPS に求められるのは生の演算力ではない。ボトルネックになるのは次の点だ。

- **メモリ**：Node ランタイム＋エディタ的な常駐プロセス＋自分のビルド・テストが同居する。2GB だと `npm install` や型チェックで OOM しやすい。4GB あると安定する。
- **ネットワークの常時性**：cron や常駐ループで API を叩き続けるので、SSH セッションや outbound 接続が切れない安定性が効く。深夜帯に勝手に切られる環境は不向き。
- **IP 固定**：SSH 鍵運用・Webhook の許可リスト・DNS の A レコードを張る前提なら、IP が変わらないことが必須。多くの VPS は固定 IP だが、契約前に必ず確認する。

逆に「時間課金で使うときだけ起動」より、**月額固定で常時起動しっぱなし**のほうが常駐用途には向く。料金は各社・各時期で変動し、キャンペーンや為替・電気代で改定されるため、ここでは具体額を断定しない。**契約直前に必ず公式の料金ページで最新の月額を確認すること**を強くすすめる。コストを抑えたいなら 4GB プランの長期割引（12〜36 ヶ月一括）を比較するのが定石だ。

## サービスの選び分け

- **ConoHa VPS**：管理画面が分かりやすく、時間課金と月額上限の併用ができる。試用して合わなければ畳みやすい。
- **Xserver VPS**：レンタルサーバの運用実績が長く、サポート窓口を重視する人向け。
- **さくらのVPS**：枯れていて情報が多く、トラブル時に日本語の事例を探しやすい。
- **お名前.com**：VPS よりドメイン取得との同梱が利点。常駐ホスト名に独自ドメインを当てたいときに一緒に契約すると DNS 管理が一本化できる。

どれも 4GB 前後のプランがあり、機能差より「管理画面の好み」と「最新料金」で選んで問題ない。

## 具体手順

1. **プラン契約**：4GB / Ubuntu LTS（22.04 か 24.04）を選ぶ。OS は迷ったら Ubuntu。
2. **SSH 鍵を登録**：ローカルで `ssh-keygen -t ed25519` を作り、公開鍵をコンソールから登録。パスワード認証は無効化する。
   ```bash
   ssh -i ~/.ssh/id_ed25519 ubuntu@<固定IP>
   ```
3. **常駐ユーザーと最小化**：root 直は避け作業ユーザーを作成。`ufw` で 22 番（と必要なら 80/443）だけ開ける。
4. **Node と Claude Code を導入**：公式手順に従って Node LTS を入れ、Claude Code CLI をインストール。API キーは `~/.bashrc` ではなく専用の env ファイルに置きパーミッションを 600 にする。
5. **DNS を張る（任意）**：独自ドメインを使うなら、お名前.com 等の管理画面で **A レコード → 固定 IP** を設定。反映は数分〜最大数十分。
6. **常駐化**：cron か systemd timer で毎朝のタスクを起動。落ちても再実行されるよう `Restart=on-failure` を付ける。

この構成なら、最小コストで Claude Code を 24 時間走らせ続けられる。まずは 4GB の最安プランで始め、メモリ不足やビルド時間が気になってから上位プランへ乗り換えれば無駄がない。
