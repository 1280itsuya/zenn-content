## Cline とは

Cline は VSCode の拡張機能として動作する AI コーディングエージェントで、ファイルの読み書き・ターミナルコマンドの実行・ブラウザ操作まで自律的にこなす。「補完候補を選ぶ」のではなく「タスクを投げて任せる」スタイルのため、設計段階のコードをゼロから実装させたい個人開発者や、繰り返しのスキャフォールディング作業を自動化したい副業エンジニアに特に向く。オープンソース(MITライセンス)で公開されており、使用モデルは自前の API キーで自由に切り替えられる。

## 起動速度

Cline は VSCode の拡張機能として常駐するため、エディタを開いた瞬間から使用可能な状態になっている。専用デスクトップアプリの起動待ちや別プロセスの立ち上げは一切ない。サイドパネルにチャット欄が出るだけで、最初の指示を打ち込むまでの手順は実質ゼロだ。初回の応答速度は選択したバックエンドモデルと API の遅延に依存するが、Cline 自体がボトルネックになる場面はほぼ経験しない。

## 補完精度

Cline が特に力を発揮するのは、複数ファイルをまたぐ修正や、既存コードの構造を読んだうえで新機能を追加するタスクだ。たとえば「この FastAPI ルートに認証ミドルウェアを追加して、既存のテストも更新して」のような指示を自然文で与えると、関連ファイルを自律的に探索し、差分を提示したうえで実行まで進む。一方、ドメイン固有の業務ロジックが暗黙知として存在するコードベースでは、Cline が誤った前提でファイルを書き換えるケースがある。指示に「変更前に計画を出して確認を待て」と明示する習慣を持つと、このリスクを大幅に抑えられる。単一行の補完速度よりも「タスク完遂力」で評価すべきツールだ。

## 月額コスト

Cline 自体は無料でインストールできるが、動作には外部 LLM の API キーが必要で、コストはそのモデルの従量課金に従う。Anthropic の Claude API を使う場合、claude-sonnet 系モデルで入力・出力それぞれ数ドル/百万トークンという価格帯になる。小規模な個人タスクを毎日こなしても月数百円〜数千円の範囲に収まることが多いが、大量のコンテキストを読ませるタスクを繰り返すと消費が膨らむ。Cline のサイドパネルにはセッションごとのトークン消費量が表示されるため、コストの可視化はしやすい。固定月額プランは存在せず、完全な従量制である点は事前に把握しておきたい。

## 使ってみた所感

副業で小規模な Web サービスを一人で運用している状況で、Cline を使って「週次の集計スクリプトを改修し、結果を Markdown レポートとして出力する機能を追加する」作業を任せたことがある。既存スクリプトを読ませ、自然文で仕様を伝えるだけで、Cline は関連モジュールを自力で辿り、テストコードも含めて一通りの実装を完成させた。自分でやれば 1〜2 時間かかる作業が、レビューと軽微な修正だけで済んだ。ただし Cline が生成したコードをそのまま使うのではなく、差分を一行ずつ確認する姿勢は崩さないようにしている。自律性が高い分、想定外の変更が紛れ込んでいないかのチェックが品質担保の肝になる。
