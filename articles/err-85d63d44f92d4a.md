## 発生条件

- Node.js で `import ... from ...` を含む `.js` ファイルを `node` で直接実行したとき。
- `package.json` に `"type": "module"` の指定が無く、ファイルが CommonJS として解釈されているとき。
- 拡張子を `.cjs` にしている、あるいはトランスパイルせず ES Modules 構文をそのまま走らせているとき。

## 原因

Node.js は対象ファイルを「CommonJS」か「ES Module」のどちらで読むかを、拡張子と `package.json` の `type` フィールドで判定します。CommonJS と判定されたファイルに `import` 文があると `SyntaxError: Cannot use import statement outside a module` が投げられます。`import` は ES Module でのみ使える構文だからです。

最小再現コードは次の通りです。`package.json` に `type` 指定が無い状態で以下を実行します。

```js
// index.js
import { readFile } from 'node:fs/promises';

console.log(typeof readFile);
```

```bash
node index.js
# SyntaxError: Cannot use import statement outside a module
```

この `index.js` は CommonJS 扱いになり、先頭の `import` 行でエラーになります。`.cjs` 拡張子の場合も同様に CommonJS 固定なので発生します。

## 修正方法

最も素直なのは、`package.json` に `"type": "module"` を追加してプロジェクト全体を ES Module として扱う方法です。

```json
{
  "name": "my-app",
  "type": "module",
  "version": "1.0.0"
}
```

これで `import` 文を含む `.js` が ES Module として解釈され、`Cannot use import statement outside a module` は解消します。

プロジェクト全体を切り替えたくない場合は、その1ファイルだけ拡張子を `.mjs` にします。`.mjs` は `type` 指定に関係なく常に ES Module 扱いです。

```bash
# index.js を index.mjs にリネーム
mv index.js index.mjs
node index.mjs
```

要点は以下の通りです。

- 全体を ESM にするなら `package.json` に `"type": "module"`。
- 一部だけなら拡張子を `.mjs`。
- 逆に CommonJS のまま残したいファイルは `.cjs` にし、そこでは `import` ではなく `require` を使う。

`"type": "module"` を入れると既存の `require` を使ったファイルが動かなくなることがあるため、その場合は該当ファイルを `.cjs` にリネームして切り分けてください。判定基準は「拡張子」と「`type` フィールド」の2つだけ、と覚えておくと原因の切り分けが速くなります。
