# 自動化botを動かすレンタルサーバー選び――個人開発者のための実務ガイド

## 結論から言う

「常時起動が必要なbotには、月額1,000円前後のVPSが最も費用対効果が高い」。クラウド(AWS/GCP)はスポットインスタンスやCold Startの管理コストがかかりすぎ、共有レンタルサーバーはSSHでプロセスを常駐させられないケースが多い。[個人開発](https://www.amazon.co.jp/s?k=%E5%80%8B%E4%BA%BA%E9%96%8B%E7%99%BA%20%E6%9C%AC&tag=1280itsuya22-22)のDiscord bot、X(旧Twitter) bot、スクレイパー、定期バッチなど「24時間起動し続けるプロセス」には、VPSが素直に合う。

---

## VPS選びの判断軸

### 1. プロセスを常駐させられるか

共有サーバー(さくらのレンタルサーバーのスタンダードプランなど)では、SSHでログインして `nohup python bot.py &` を走らせても、セッション終了時に強制終了されることがある。VPSであれば `systemd` にサービス登録できるため、再起動後も自動復旧する。

```ini
# /etc/systemd/system/mybot.service の例
[Unit]
After=network.target

[Service]
ExecStart=/usr/bin/python3 /home/user/bot/main.py
Restart=always

[Install]
WantedBy=multi-user.target
```

### 2. スペックと料金のバランス

料金は各社・プランによって頻繁に変動するため、必ず公式ページで最新価格を確認すること。参考として2026年7月時点での感触を書くと:

- **ConoHa VPS** … 512MB〜1GBプランで月額数百円台から。キャンペーン割引が多く、長期割適用時にコストが下がりやすい
- **Xserver VPS** … 2GBプランが定番帯として紹介されることが多い。管理パネルが国内ユーザーに馴染みやすい
- **さくらのVPS** … 歴史が長く安定運用の実績が多い。512MBプランは軽量botに十分
- **お名前.com VPS** … [ドメイン](https://px.a8.net/svt/ejp?a8mat=4B3XB4+4VMW8I+50+2HU3GX)管理と同じアカウントで一元管理できる点が副業運用に地味に便利

軽量botなら512MB/1coreで動くことが多い。[Python](https://www.amazon.co.jp/s?k=Python%20%E5%85%A5%E9%96%80%20%E6%9C%AC&tag=1280itsuya22-22)プロセス1本で常時稼働させるだけなら、メモリ使用量は体感50〜200MB程度に収まることが多いが、ライブラリ構成によって異なるので `htop` で実測してから上位プランへの移行を検討すること。

### 3. SSH接続と鍵認証

どのVPSでもSSH接続は前提だが、初期設定で必ずやること:

```bash
# パスワード認証を無効化し、鍵認証のみに
sudo sed -i 's/#PasswordAuthentication yes/PasswordAuthentication no/' /etc/ssh/sshd_config
sudo systemctl restart sshd
```

rootでのSSH直接ログインも禁止しておく(`PermitRootLogin no`)。botを動かすための専用ユーザーを作り、sudo権限は最小限にとどめる。

### 4. ドメインとDNS設定

Webhookを受け取るbotや、外部サービスからのコールバックが必要なbotはドメインが必要になる。お名前.comでドメインを取得してVPSのIPをAレコードに向けるのが最もシンプルな構成。DNSの反映には最大数十分〜数時間かかるため、テスト前に余裕を持って設定すること。Let's Encryptでの証明書発行(certbot)もVPS上で完結できる。

## 運用上のチェックリスト

- [ ] `systemd` でbot登録済み、`Restart=always` を設定した
- [ ] ファイアウォール(`ufw`)でSSHポート以外を最小限に絞った
- [ ] ログが `/var/log/mybot.log` などに出力され、`journalctl -u mybot` で確認できる
- [ ] APIキーや秘密情報は `.env` ファイルに分離し、GitHubに上げていない
- [ ] 月次でサーバー費用を確認し、不要になったら即スナップショット→削除できる体制にしている

---

## まとめ

常時起動botには「VPS一択」。料金は最安帯から始めて実測してから上げる判断でよい。ただし各社の料金は頻繁に変更されるため、申し込み前に必ず公式の最新料金を確認すること。SSH鍵認証・systemdへの登録・ログ設計の3点を最初に固めれば、その後の運用コストは限りなく低くなる。
