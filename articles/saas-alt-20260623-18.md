## 結論（おすすめ1つ）

**乗り換え先は Bruno 一択**。

コレクションをローカルの `.bru` ファイル群として保存するため、Git で差分管理できる。完全オープンソースでアカウント登録不要・クラウド強制なしで運用できる。Postman 形式の JSON を直接インポートできるため移行コストが他ツールより低い。

---

## 比較表（料金/無料枠/移行コスト/対応プロトコル）

| ツール | ライセンス/料金 | 無料枠 | 移行コスト | 対応プロトコル |
|---|---|---|---|---|
| **Bruno** | MIT・完全無料 | 制限なし | 低（Postman インポート対応） | REST, GraphQL |
| Insomnia | フリーミアム（公式ページで要確認） | ローカル利用は無料（v8 以降ログイン必須） | 中（独自フォーマット） | REST, GraphQL, gRPC, WebSocket |
| Hoppscotch | OSS・セルフホスト無料 / クラウド有料（要確認） | セルフホスト版は無制限 | 中 | REST, GraphQL, WebSocket, SSE |
| Thunder Client | VS Code 拡張・フリーミアム（要確認） | 個人利用は無料 | 低〜中 | REST, GraphQL |
| HTTPie Desktop | フリーミアム（要確認） | 基本機能は無料 | 高（フォーマット差異が大きい） | REST |

## 移行手順

Bruno への移行を段階的に示す。

**Step 1: Bruno をインストールする**

```bash
# macOS (Homebrew)
brew install bruno

# Windows (winget)
winget install Bruno.Bruno

# Linux — 公式サイト https://www.usebruno.com/ から .deb / .rpm を取得してインストール
sudo dpkg -i bruno_*.deb
```

**Step 2: Postman からコレクションをエクスポートする**

Postman 画面で対象コレクションを右クリック → 「Export」→ フォーマットは **Collection v2.1** を選択して JSON ファイルを保存する。

**Step 3: Bruno にインポートする**

```
Bruno 起動
  → 「Import Collection」
  → 「Postman」を選択
  → Step 2 で保存した JSON ファイルを指定
  → インポート先ディレクトリを選択して完了
```

インポート後、コレクションは指定ディレクトリに `.bru` ファイルとして展開される。

**Step 4: 環境変数を移行する**

Postman の Environment は別途エクスポートが必要。Bruno 側は平文テキストで記述する。

```bash
# Postman: Environments → 対象 Env の「...」→ Export → JSON 保存

# Bruno 側: collection ディレクトリ配下の environments/ に .bru 形式で作成
# 例: environments/local.bru
```

```
vars {
  BASE_URL: http://localhost:3000
  API_KEY: your-api-key-here
}
```

**Step 5: Git リポジトリに追加してチームと共有する**

```bash
cd /path/to/your-collection-dir
git init
git add .
git commit -m "init: migrate Postman collections to Bruno"
git remote add origin git@github.com:your-org/api-collections.git
git push -u origin main
```

以降、リクエストの追加・変更はファイルの差分として PR レビューできる。

## 向き不向き

**向いているチーム・ケース**

- Git 中心のワークフローを採用しており、API コレクションの変更を PR でレビューしたいチーム
- コレクションをアプリケーションコードと同一リポジトリで管理したい場合
- Postman の料金プラン改定や強制クラウド同期に不満を抱えているチーム
- オンプレミス・エアギャップ環境で外部サービスへの接続が禁止されているプロジェクト
- 個人開発者やスタートアップでライセンスコストを抑えたい場合

**避けるべきケース・ツール選定の注意点**

- **gRPC / WebSocket を多用するチーム**: Bruno のサポートは執筆時点で限定的。Insomnia の方が成熟している
- **Insomnia**: v8 以降にログイン必須ポリシーが導入された経緯があり、今後の方針変更リスクを許容できないチームには推奨しにくい
- **Hoppscotch セルフホスト**: Docker 運用が前提。インフラ管理リソースのない小規模チームはクラウド版のコストを公式ページで確認してから判断すること
- **Thunder Client**: VS Code 専用設計のため、JetBrains 系 IDE メインのチームや IDE に依存しない独立ツールが必要な場面には合わない
- **Postman の `pm.expect` ベースのテストスクリプトを大量に書いている場合**: Bruno への移行時にスクリプトを書き直すコストが発生する。移行前にスクリプト資産の棚卸しをしてから判断すること
