## 結論から先に言う

`whereHas` が遅いのは N+1 ではなく、Eloquent が生成する `where exists (...)` のサブクエリが「親テーブル全件 × 子テーブル」で評価されるからです。対処はシンプルで、**サブクエリ内に親の絞り込み条件が効くようインデックスを張り、必要なら `whereHas` を `whereIn` か `join` に置き換える**こと。まずは生成 SQL を見るところから始めます。

## つまずく原因: whereHas は EXISTS サブクエリになる

`whereHas` は内部で相関サブクエリ（correlated subquery）を作ります。

```php
$users = User::whereHas('orders', function ($q) {
    $q->where('status', 'paid');
})->get();
```

生成される SQL はこうです。

```sql
select * from users
where exists (
    select * from orders
    where orders.user_id = users.id
      and status = 'paid'
)
```

問題は `orders.user_id = users.id` という相関条件。`users` が10万行あれば、サブクエリが10万回評価されることもあります。N+1（クエリが行数だけ飛ぶ）とは違い、クエリは1本ですが、その1本が全件スキャンになって遅い、という別物の現象です。

まず `toSql()` か `DB::enableQueryLog()` で実際の SQL を確認しましょう。

```php
DB::enableQueryLog();
$users = User::whereHas('orders', fn ($q) => $q->where('status', 'paid'))->get();
dump(DB::getQueryLog());
```

## 解決1: 相関カラムに複合インデックスを張る

`whereHas` を使い続けるなら、EXISTS が高速に判定できるインデックスが必須です。`orders(user_id, status)` の複合インデックスを張ると、サブクエリが行を見つけた瞬間に打ち切れます。

```php
Schema::table('orders', function (Blueprint $table) {
    $table->index(['user_id', 'status']);
});
```

`EXPLAIN` で `Using index` や `ref` が出れば成功です。多くのケースはこれだけで解決します。

## 解決2: whereHas を whereIn に置き換える

子テーブルの絞り込み結果が小さいなら、相関サブクエリをやめて非相関の `whereIn` にすると、子側を一度だけ評価できます。

```php
$users = User::whereIn('id', function ($q) {
    $q->select('user_id')
      ->from('orders')
      ->where('status', 'paid');
})->get();
```

`whereHas` のように毎行評価されず、`orders` 側のインデックス（`status`）が一度効くだけになります。

## 解決3: 件数だけ欲しいなら join + distinct

「条件を満たす親を取る」だけが目的なら `join` が最速のことが多いです。

```php
$users = User::join('orders', 'orders.user_id', '=', 'users.id')
    ->where('orders.status', 'paid')
    ->select('users.*')
    ->distinct()
    ->get();
```

ただし `join` は重複行や列名衝突に注意が必要で、`whereHas` のような可読性・再利用性は失われます。

## 使い分けのまとめ

- まず生成 SQL と `EXPLAIN` を見る
- インデックスで足りるなら `whereHas` のまま `(user_id, status)` を張る
- 子の絞り込みが軽いなら `whereIn` サブクエリ
- 重複を許容できて件数重視なら `join` + `distinct`

`whereHas` 自体が悪いわけではなく、EXISTS の相関条件にインデックスが効いていないだけ、というのが本質です。
