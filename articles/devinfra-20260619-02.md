# 自動化botを動かすレンタルサーバー選び――個人開発者のための実務ガイド

## 結論：bot常駐に必要な条件は「SSH常時接続・cron・固定IP」の三点セット

自動化botを自宅PCで動かすと、PCのスリープや回線断で止まる。クラウド(AWS/GCP)は従量課金の請求が怖い。そこでVPSが現実解になる。選定で見るべきポイントは以下に絞られる。

1. **SSH接続が安定して使えること**（ポート変更・鍵認証が当然可能か）
2. **cronまたはsystemdが自由に使えること**（共有ホスティングでは不可）
3. **固定グローバルIPが付与されること**（外部WebhookのIP許可リストに登録できる）
4. **月額が固定で予算が読めること**

これを満たすのがVPS契約だ。以下、実務の視点で各社を比較する。

## 主要VPS4社の実務的な見どころ

### ConoHa VPS

GMOグループ運営。コントロールパネルが直感的で、OSインストール後すぐにSSH接続できる。料金は変動するため[公式サイト](https://www.conoha.jp/vps/)で必ず確認すること。メモリ512MBプランから契約でき、[Python](https://www.amazon.co.jp/s?k=Python%20%E5%85%A5%E9%96%80%20%E6%9C%AC&tag=1280itsuya22-22)製botなら1GB以上が安定の目安。Webコンソールがブラウザから使えるので、SSH接続を失っても復旧しやすい点が初心者に優しい。

### Xserver VPS

Xserverブランドの信頼感と、国内データセンター完結による低遅延が特徴。アプリイメージ(Dockerプリセット等)が充実しており、環境構築を短縮したい場合に有効。料金は変動するため[公式サイト](https://vps.xserver.ne.jp/)で確認。ディスクがSSD固定なのでIOが速く、DBを内包するbotにも向く。

### さくらのVPS

老舗の安定感。2週間無料試用があり、本番移行前に動作確認できるのが実務的に助かる。料金プランの変動があるため[公式サイト](https://vps.sakura.ad.jp/)で最新情報を確認すること。スペックは同価格帯他社より控えめな場合があるが、サポート・ドキュメントが日本語で充実しており、トラブル対処に安心感がある。

### お名前.com VPS

[ドメイン](https://px.a8.net/svt/ejp?a8mat=4B3XB4+4VMW8I+50+2HU3GX)管理と同じアカウントで契約でき、DNS設定とサーバー管理をワンストップで行いたい場合に便利。Webhookでドメイン名を使う場合に手間が減る。料金は変動するため[公式サイト](https://www.onamae.com/vps/)で確認。

## ドメイン・DNS設定の実務手順

botがWebhookを受け取る場合、固定IPだけでなくドメイン名が必要になることが多い（SSL証明書取得のため）。

```
1. VPS契約 → 固定グローバルIPを取得
2. ドメイン取得（お名前.com / Cloudflare等）
3. Aレコードにサーバーの固定IPを登録
4. Let's Encrypt で SSL証明書を取得（certbot推奨）
5. bot側のWebhookエンドポイントをhttpsで公開
```

DNS反映は数分〜数時間かかる。bot本番稼働前に`dig`や`nslookup`で名前解決を必ず確認すること。

---

## SSH設定の最低限ハードニング

VPSをインターネットに公開した瞬間からブルートフォース攻撃が来る。最低限以下を実施する。

```bash
# /etc/ssh/sshd_config の変更箇所
Port 2222              # デフォルト22から変更
PermitRootLogin no     # root直接ログイン禁止
PasswordAuthentication no  # 鍵認証のみ
```

変更後は`systemctl reload sshd`。ファイアウォール(ufw/iptables)でSSHポートと必要なポートのみ開放し、それ以外は閉じる。

## プラン選定の目安

| botの種類 | 推奨メモリ |
|---|---|
| cronで定期実行するスクリプト | 512MB〜1GB |
| 常駐プロセス(Discord botなど) | 1GB〜2GB |
| LLM APIを呼ぶ処理+DB内包 | 2GB以上 |

最初は最小プランで試し、`free -h`でメモリ消費を確認しながらスケールアップする方針が無駄がない。

---

## bot運用で詰まりやすい実務ポイント

**プロセス常駐にはsystemdを使う**

`nohup`や`screen`は再起動で消える。`/etc/systemd/system/mybot.service`に定義し`systemctl enable mybot`で自動起動を設定する。

**cronの環境変数に注意する**

cronはログインシェルと環境変数が異なる。`.env`ファイルを`source`するか、cronの冒頭で明示的に`PATH`や`PYTHONPATH`を設定しないとbotが起動しない罠がある。

**ログローテーションを設定する**

放置するとディスクがログで埋まる。`logrotate`を使うか、`systemd`の`journal`上限を設定しておく。

## まとめ

自動化botのVPS選びは「SSH・cron・固定IP」が揃えばどこでもよい。初めてなら**ConoHa VPS**か**さくらのVPS**が日本語ドキュメントの豊富さで入りやすい。料金は各社とも頻繁に改定されるため、必ず契約時点の公式ページで確認すること。最小プランから始め、実測のメモリ消費を見てスケールアップするのが過剰投資を避ける現実的な運用だ。
