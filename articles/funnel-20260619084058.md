# GitHub Actions × Claude APIでZenn/はてなブログに毎日自動投稿するCI/CDパイプライン構築

## やること

GitHub Actionsのcronスケジューラ + Claude API（`claude-haiku-4-5-20251001`）で記事を生成し、ZennのGit連携またははてなブログのXMLRPCへ自動投稿する。手元に何もなくても毎朝記事が公開される構成を作る。

---

## 全体構成

```
.github/workflows/daily_post.yml
  └─ cron 21:00 UTC (日本時間06:00)
       ├─ generate.py  → Claude APIで記事Markdown生成
       ├─ post_zenn.py → git commit & push（Zennはpushが公開トリガー）
       └─ post_hatena.py → XMLRPC API呼び出し
```

---

## ステップ1: シークレット設定

GitHubリポジトリの **Settings → Secrets and variables → Actions** に以下を追加する。

| キー | 値 |
|---|---|
| `ANTHROPIC_API_KEY` | Anthropic Consoleで発行したAPIキー |
| `ZENN_GITHUB_PAT` | `contents: write`スコープのPAT |
| `HATENA_API_KEY` | はてなの「APIキー」（アカウント設定から取得） |
| `HATENA_BLOG_ID` | `yourname.hatenablog.com` 形式 |

---

## ステップ2: 記事生成スクリプト

```python
# generate.py
import anthropic, datetime, pathlib, sys

client = anthropic.Anthropic()  # ANTHROPIC_API_KEY を自動参照

TOPICS = [
    "PythonでCSVを高速処理する5つのテクニック",
    "Dockerマルチステージビルドで本番イメージを半分に削る",
    "GitHub Actionsのcache活用でCI時間を短縮する",
]

today = datetime.date.today()
topic = TOPICS[today.toordinal() % len(TOPICS)]

message = client.messages.create(
    model="claude-haiku-4-5-20251001",
    max_tokens=2048,
    messages=[{
        "role": "user",
        "content": (
            f"現役エンジニア向けに「{topic}」について実用的な技術記事を書いてください。"
            "要件: Markdown形式、1行目は# タイトル、コード例必須、1200〜1800字。"
            "誇大表現・架空の数値は使わない。"
        )
    }]
)

out = pathlib.Path("articles") / f"{today}.md"
out.parent.mkdir(exist_ok=True)
out.write_text(message.content[0].text, encoding="utf-8")
print(f"Generated: {out}")
```

## ステップ3: Zennへ投稿（Git push）

Zennはリポジトリ連携設定済みであれば、`articles/`配下にMarkdownをpushするだけで公開される。

```python
# post_zenn.py
import subprocess, datetime, pathlib, sys

today = datetime.date.today()
src = pathlib.Path("articles") / f"{today}.md"

# Zennが要求するfrontmatter付きファイルに変換
text = src.read_text(encoding="utf-8")
title_line = text.splitlines()[0].lstrip("# ").strip()
zenn_path = pathlib.Path("zenn_articles") / f"{today}.md"
zenn_path.parent.mkdir(exist_ok=True)
zenn_path.write_text(
    f"""---
title: "{title_line}"
emoji: "🤖"
type: "tech"
topics: ["python", "githubactions", "automation"]
published: true
---

""" + "\n".join(text.splitlines()[1:]),
    encoding="utf-8"
)

subprocess.run(["git", "config", "user.email", "bot@example.com"], check=True)
subprocess.run(["git", "config", "user.name", "AutoBot"], check=True)
subprocess.run(["git", "add", str(zenn_path)], check=True)
subprocess.run(["git", "commit", "-m", f"auto: {today}"], check=True)
subprocess.run(["git", "push"], check=True)
```

## ステップ4: ワークフロー定義

```yaml
# .github/workflows/daily_post.yml
name: daily-blog-post

on:
  schedule:
    - cron: "0 21 * * *"   # UTC 21:00 = JST 06:00
  workflow_dispatch:        # 手動実行用

jobs:
  post:
    runs-on: ubuntu-latest
    permissions:
      contents: write       # git push に必要

    steps:
      - uses: actions/checkout@v4
        with:
          token: ${{ secrets.ZENN_GITHUB_PAT }}

      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"

      - run: pip install anthropic

      - name: Generate article
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: python generate.py

      - name: Push to Zenn
        env:
          GIT_TERMINAL_PROMPT: "0"
        run: python post_zenn.py
```

`GIT_TERMINAL_PROMPT: "0"` を入れないと、PAT認証ダイアログ待ちで**無音ハング**する。これが最もハマりやすい点。

---

## つまずきポイント3選

**1. pushが403で失敗する**  
`actions/checkout`の`token:`に`ZENN_GITHUB_PAT`を渡さないとデフォルトの`GITHUB_TOKEN`が使われ、保護ブランチへのpushが弾かれる。

**2. Zennのfrontmatterが不正で非公開扱いになる**  
`published: true`が欠落しているか、`topics`に存在しないタグを指定すると下書きのまま。Zenn CLIの`npx zenn validate`をローカルで事前確認する。

**3. スケジュールが動かない**  
GitHubのcron実行はリポジトリが60日間非アクティブだと停止する。`workflow_dispatch`トリガーを残しておき、定期的に手動実行して活性化しておく。

## コスト感

`claude-haiku-4-5-20251001`で1記事あたり約1,500トークン出力した場合、1回あたり数円以下。月30記事でも数十円〜百円台に収まる（公式価格ページで現在の料金を必ず確認すること）。

## まとめ

- cronスケジューラ + Claude API + Git pushの3点セットで毎日投稿が無人化できる
- `GIT_TERMINAL_PROMPT=0`とPATのcheckout渡しは必須
- Zennのfrontmatter仕様はCLIで事前検証する
