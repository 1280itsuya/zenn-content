## 発生条件

- [Python](https://www.amazon.co.jp/s?k=Python%20%E5%85%A5%E9%96%80%20%E6%9C%AC&tag=1280itsuya22-22)ファイル内でタブ文字とスペースを混在させてインデントしている
- ブロック外で突然インデントが存在する（`if`/`for`/`def`等のブロック開始なしに字下げされている）
- エディタの「タブをスペースに変換」設定がオフのまま、別のエディタで書いたコードを貼り付けた

## 原因

Pythonはインデントでブロック構造を表現するため、予期しないインデントを検出すると `IndentationError: unexpected indent` を送出する。以下は最小再現コードの例：

```python
x = 10
    print(x)  # この行に不要なインデントがある
```

このコードを実行すると：

```
  File "sample.py", line 2
    print(x)
    ^
IndentationError: unexpected indent
```

上記のように、`print(x)` の前にスペースが入っているにもかかわらず、直前に `if` や `def` などのブロック開始文がないため、Pythonパーサーが「ここにインデントがあるのはおかしい」と判断してエラーを出す。

タブ/スペース混在のケースも同様に発生しやすい：

```python
def greet():
	print("hello")  # タブでインデント
        print("world")  # スペース8つでインデント
```

見た目上は揃っているように見えても、内部的にはタブとスペースが混在しており、Pythonはこれを区別するため `IndentationError` が発生する。

## 修正方法

インデントをすべてスペース4つに統一することで解消できる。

**ブロック外インデントの修正：**

```python
# 修正前
x = 10
    print(x)

# 修正後
x = 10
print(x)
```

**タブ/スペース混在の修正：**

```python
# 修正前（タブとスペースが混在）
def greet():
	print("hello")   # タブ
        print("world")   # スペース8つ

# 修正後（スペース4つに統一）
def greet():
    print("hello")
    print("world")
```

VSCodeを使っている場合は、ファイルをひらいた状態でコマンドパレット（`Ctrl+Shift+P`）から「インデントをスペースに変換」を実行すると一括変換できる。

また、`pycodestyle` や `flake8` などのリンターを導入しておくと、保存時にインデントの不整合を検出してくれるため、同種のエラーを事前に防ぎやすい。

プロジェクトで複数人が開発する場合は `.editorconfig` を以下のように設定しておくことで、エディタを問わずインデントを統一できる：

```ini
[*.py]
indent_style = space
indent_size = 4
```

`IndentationError: unexpected indent` はコードの実行前にパーサーが検出するため、スタックトレースよりも早い段階でエラー行番号が表示される。エラーメッセージのファイル名と行番号を確認し、その行と直前の行のインデントを見比べることが最初の確認手順として有効だ。
