## Tabnine とは

Tabnine はコード補完に特化した AI コーディングアシスタントで、プライバシーを最優先に設計されているのが最大の特徴だ。コードをクラウドに送信せずにローカルモデルで完結できるため、業務委託先の NDA や社内セキュリティポリシーが厳しい現場でも導入しやすい。副業で受託開発をしながら本業のコードと環境を分けたい開発者や、クローズドなプロジェクトを個人で抱えているエンジニアに特に向く。

## 起動速度

Tabnine のローカルモデルは VS Code や JetBrains 系 IDE の起動と同時に読み込まれ、エディタが使える状態になった時点でほぼ補完が機能する。クラウドへのラウンドトリップがないぶん、キーストロークから候補が現れるまでのラグは体感で短い。低スペックのノート PC でも極端な遅延は起きにくく、カフェなどオフライン気味の環境でも安定して動作する点は実用上の大きなメリットだ。

## 補完精度

Tabnine が強いのは定型パターンの繰り返しが多いコードだ。たとえば Python でデータクラスのフィールドを何十行も書いていると、次のフィールド名を高精度で先読みしてくれる。Express.js のルート定義やテストコードの `describe/it` ブロックのような構造的な繰り返しでも同様に機能する。一方、複雑なビジネスロジックや抽象度の高いアルゴリズムの実装補助は苦手で、コード全体の文脈を深く読んだ提案は期待しすぎないほうがよい。また自然言語コメントからコードを生成する能力は限定的で、その用途には別のツールを使うのが現実的だ。

## 月額コスト

無料プランが用意されており、基本的なコード補完機能はコストなしで使い始められる。有料の Pro プランは月額 $9 程度で、より高精度なクラウドモデルや長いコンテキスト補完が利用可能になる。Enterprise プランは価格が別途見積もりとなり、オンプレミスデプロイや SSO などの機能が追加される。個人の副業用途であれば無料プランから試して、補完の物足りなさを感じてから Pro に移行する判断で十分だろう。

## 使ってみた所感

副業でスクレイピングスクリプトを量産している場面で Tabnine の恩恵を感じた。リクエスト送信・レスポンスパース・エラーハンドリングという同じ構造が繰り返されるコードでは、2〜3行書き始めただけで残りのブロックをほぼそのまま使える形で補完してくれる。GitHub Copilot のように説明文からコードを生成する体験ではなく、あくまで「自分の書き方を学習して先読みする」感覚に近い。コードをクラウドに出したくない案件でも気兼ねなく使える安心感は、Tabnine を選ぶ実質的な理由として十分に成り立つ。
