## Continue.dev とは

Continue.dev は VS Code および JetBrains 系 IDE 向けのオープンソース AI コーディングアシスタント拡張機能だ。商用 SaaS に依存せず、手元の LLM（Ollama 等）や任意の API エンドポイントを自由に繋げられるのが最大の特徴で、「自分のモデルを自分で選びたい」というエンジニアや、コードをクラウドに送りたくない業務用途に向く。ライセンスは Apache 2.0 で、個人・商用いずれも無料で利用できる。

## 起動速度

Continue.dev は IDE 拡張として動作するため、IDE を開けばそのまま使える。専用アプリやデーモンの起動待ちは発生しない。インラインの補完レスポンス速度はバックエンドに依存し、ローカルで Ollama を動かす場合は GPU の有無で体感が変わる。Anthropic や OpenAI の API を繋いだ場合はネット遅延分だけ加算されるが、チャットパネルへの応答開始は概ね快適だ。補完候補の表示タイミングもモデル次第のため、まずローカルで試して許容できるか確かめるとよい。

## 補完精度

Continue.dev の補完精度は接続するモデルに直結する。GPT-4 クラスを繋いだ場合、関数単位の補完・コメントからの実装生成は十分な品質が出る。一方、Ollama 上の小型モデル（7B 前後）だと型の取り違いや既存コードの文脈理解が甘くなる場面が出る。チャットでのコードブロック書き換えには強く、「この関数をエラーハンドリング付きに書き直して」といった指示で丸ごと差し替えるフローが使いやすい。新しいフレームワークや社内固有の DSL には事前に学習コンテキストとして定義を渡す必要があり、何もなしではハルシネーションが起きやすい。

## 月額コスト

Continue.dev 本体は完全無料のオープンソースソフトウェアだ。拡張機能を入れるだけでサブスクリプションは一切発生しない。費用が生じるのはバックエンド部分のみで、OpenAI や Anthropic の API を使えばそのトークン料金が別途かかる。Ollama でローカルモデルを動かす構成にすれば、ランニングコストはゼロのままにできる。フリーランスや副業での個人開発においては、API 費用を使った分だけ払う従量モデルで運用できる点が予算管理しやすい。

## 使ってみた所感

個人開発で Python スクリプトを日々書く場面で Continue.dev を使ったところ、チャットパネルから「このクラスに型アノテーションを追加して」と頼み、差分プレビューを確認してから一括適用するフローが特に効率的だった。エディタを離れずにコードと会話できるため、ブラウザでチャット画面を行き来するよりコンテキスト切り替えが少ない。副業で複数の小規模プロジェクトを並行管理している場合、Continue.dev の設定ファイル（`.continue/config.json`）をリポジトリごとに持てるため、プロジェクト別にモデルやプロンプトを使い分けられるのも地味に便利だ。
