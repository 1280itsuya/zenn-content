## この本のざっくり要約

『Web API The Good Parts』は、RESTful APIを設計・実装する際の「なぜそうするのか」を丁寧に説いた一冊だ。単なるREST入門書ではなく、エンドポイント設計からバージョニング、セキュリティ、レスポンス形式まで、実務で必ず直面する判断ポイントを網羅している。「動くAPIを作る」だけでなく「長く使えるAPIを設計する」ための思考軸を与えてくれる本だと言える。

---

## 章ごとのポイント

### エンドポイント設計の原則

URIにはリソースの名詞を使い、動詞はHTTPメソッドで表現するという基本を丁寧に解説する。`/getUser`ではなく`/users/{id}`というシンプルな話のようで、実務では慣れ前に崩れやすいルールだ。複数形の使い方や階層の深さについても、具体的な判断基準が示されている。

### HTTPメソッドとステータスコードの正しい使い方

GETとPOSTだけで済ませがちな設計に対して、PUT/PATCH/DELETEの使い分けを明確にしている。ステータスコードについても、200を何にでも返す悪習を避けるための実例が豊富だ。201と200の違い、404と410の使い分けなど、細かい選択に根拠が与えられる。

### レスポンスデータの構造設計

エラーレスポンスの形式統一、ネストの深さ、配列かオブジェクトかの選択基準など、クライアント側の実装しやすさを意識した設計論が展開される。envelope（ラッパーオブジェクト）パターンの是非についても議論があり、近年の傾向と照らして考えさせられる内容だ。

### バージョニング戦略

URLにバージョンを埋め込む方法（`/v1/`）とヘッダで管理する方法を比較し、それぞれのトレードオフを率直に論じている。「どちらが正解」と断言しないスタンスが誠実で、チームの状況に合わせた判断ができるよう情報が整理されている。

### セキュリティとアクセス制御

OAuth 2.0の基本フローを軸に、APIキー管理やレートリミットの設計まで扱う。特に「認証と認可は別物」という整理は、実務で混同しがちな概念を落ち着いて捉え直すきっかけになる。

## 実務でどう効いたか

私はBtoBのSaaS開発でAPIを複数設計してきたが、この本を読む前はステータスコードの選択を感覚で決めていた。特に更新系でPUTとPATCHのどちらを使うかで、チーム内に「どっちでもよくない？」という空気があった。本書を読んだ後、差分更新ならPATCH・全体置換ならPUTという軸をチームに共有することができ、レビューでの議論がかみ合うようになった。

また、エラーレスポンスの構造を統一するうえでも助かった。以前はエンドポイントごとにエラーの返し方がばらばらで、クライアント側の開発者から「どのフィールドを見ればいいか分からない」と言われていた。本書のエラーオブジェクト設計を参考に、`code`・`message`・`details`を必ず含む形に統一したことで、フロントエンドチームとの認識齟齬が減った。

## どこが古い・効きにくいか

出版が2014年であることは正直に踏まえておく必要がある。GraphQLやgRPCが普及した現在、「REST一択」という前提で読むと視野が狭くなる。本書はRESTの文脈に特化しているため、トレードオフの比較という意味での情報は別途補完が必要だ。

OAuthの解説も2.0の基本フローまでで、PKCE・DPoP・RAR（Rich Authorization Requests）といった現代的な拡張には触れていない。セキュリティ要件が厳しいAPIを作る場合、本書だけでは不十分で最新のRFC類を参照する必要がある。

ページネーションについても、カーソルベースのページネーションへの言及が薄く、大規模データセットへの適用には他のリソースで補う場面が出てくる。

## こんな人に向く / 向かない

**向く人**

- APIを初めて設計するバックエンドエンジニア
- 「なんとなく動くAPI」から「設計に根拠があるAPI」へ移行したい人
- チームのAPIレビュー基準を言語化したいテックリード
- フロントエンドエンジニアでAPIの消費者側から設計を理解したい人

**向かない人**

- GraphQL・gRPCを主戦場にしているエンジニア
- 最新のOAuth/OIDCセキュリティ要件を詳しく学びたい人
- 大規模分散システムのAPI Gateway設計を扱いたい人
- すでにREST設計の原則を実務で使いこなしている中上級者

---

Web API The Good Partsは、派手な新技術の本ではないが、「なぜそう設計するか」を丁寧に説明する姿勢は今も色あせていない。古さを理解したうえで、REST設計の思考基盤を作る一冊として手元に置く価値はある。
