# Claude CodeにMCPサーバーを追加してファイル操作・DB参照を自動化する最小手順

Claude Codeは標準でファイル読み書きやシェル実行ができるが、MCPサーバーを追加すると「特定ツールの呼び出し」や「外部DBへの直接クエリ」をプロンプトから自然に扱えるようになる。この記事では、最小構成で動かすまでの手順とはまり点を具体的に示す。

---

## 前提

- Node.js 18以上がインストール済み
- `claude` CLIが `~/.npm-global/bin` などにある（`claude --version` で確認）
- MacまたはLinux（Windowsの場合はパスの区切り文字に注意）

---

## 1. MCP設定ファイルの場所を確認する

Claude Codeは `~/.claude.json` にMCP設定を持つ。まずここを確認する。

```bash
cat ~/.claude.json | python3 -m json.tool | head -30
```

`mcpServers` キーがなければ、後述の手順で追加する。

---

## 2. 公式サンプル：filesystemサーバーを追加する

もっとも動かしやすいのは `@modelcontextprotocol/server-filesystem` だ。特定ディレクトリへのファイル操作をMCP経由で行えるようにする。

```bash
claude mcp add filesystem -- npx -y @modelcontextprotocol/server-filesystem /tmp/sandbox
```

このコマンドが `~/.claude.json` の `mcpServers` に以下を書き込む。

```json
"filesystem": {
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-filesystem", "/tmp/sandbox"]
}
```

確認：

```bash
claude mcp list
```

`filesystem` が表示されれば追加成功。

---

## 3. 動作確認

`claude` を対話モードで起動し、以下を入力する。

```
/tmp/sandbox に hello.txt を作って「Hello, MCP」と書いて
```

Claude Codeがファイルシステムツールを呼び出し、ファイルを生成する。ターミナル上でツール呼び出しのログが表示されるので、実際に何が起きているか追える。

```bash
ls /tmp/sandbox
# hello.txt
cat /tmp/sandbox/hello.txt
# Hello, MCP
```

---

## 4. SQLiteサーバーを追加してDBを参照する

開発でよく使うのがDB参照だ。`@modelcontextprotocol/server-sqlite` を使うとSQLiteファイルへ自然言語でクエリを投げられる。

```bash
# テスト用DBを作っておく
sqlite3 /tmp/test.db "CREATE TABLE users (id INTEGER, name TEXT); INSERT INTO users VALUES (1, 'Alice'), (2, 'Bob');"

# MCPサーバーを登録
claude mcp add sqlite -- npx -y @modelcontextprotocol/server-sqlite /tmp/test.db
```

その後、Claudeに「usersテーブルの中身を全部見せて」と頼むだけで、SQLを書かずにレコードが返ってくる。

---

## つまずきポイント3選

**① npxが毎回ダウンロードする問題**

`-y` フラグが付いているため初回は自動インストールされるが、オフライン環境や速度を求める場合はグローバルにインストールしておく。

```bash
npm install -g @modelcontextprotocol/server-filesystem
```

そのうえで `command` を `node`、`args` をグローバルモジュールのパスに変える。

**② パスが絶対パスでないと動かない**

`~/projects` のような `~` 展開はMCP設定内では機能しないことがある。必ず `/home/yourname/projects` の形で書く。

**③ サーバーが起動しているかの確認方法**

`claude mcp list` はあくまで登録の確認であり、サーバープロセスが生きているかは別問題だ。うまく動かないときは `claude` 起動時のログ（`--debug` フラグ）を見る。

```bash
claude --debug
```

`MCP server started` のログが出ていれば正常。出ていなければコマンドパスやNode.jsのバージョンを疑う。

---

## まとめ

MCPサーバーの追加は `claude mcp add` 一行で完結する。まずfilesystemで動作を体感し、次にsqliteや自作サーバーへ拡張するのが最短ルートだ。自作サーバーはMCP SDKの `@modelcontextprotocol/sdk` でTypeScriptから実装でき、社内APIや独自ツールをClaude Codeから呼び出せるようになる。
