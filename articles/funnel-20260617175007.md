# ベクタDBなしで作る最小RAG：社内PDFをClaudeにQ&Aさせる実装手順

社内の仕様書や議事録に対して「あの機能の仕様ってどこに書いてあったっけ？」と探し回るのをやめたい。でもPineconeやWeaviateを立てるほどの規模でもない。そんな現場向けに、**ベクタDB不要のシンプルなRAGをPython 100行以内で実装する**手順を説明する。

---

## 前提とアーキテクチャ

ベクタDBが必要になるのは「数千ドキュメント以上」「低レイテンシ」「本番トラフィック」の組み合わせが揃ったときだ。社内の資料検索では大抵どれも当てはまらない。

今回の構成はシンプルに3ステップで済む。

```
[ドキュメント群] → [テキスト抽出] → [BM25スコアリング] → [上位チャンクをContextに] → [Claude API]
```

BM25はキーワードベースの古典的な全文検索アルゴリズムで、ライブラリ1行で使える。埋め込みモデルもGPUも不要だ。

---

## 実装手順

### 1. 依存ライブラリのインストール

```bash
pip install anthropic rank_bm25 pypdf python-dotenv
```

### 2. ドキュメントのチャンク分割

PDFをページ単位で切り出し、重複なしの固定チャンクに分割する。

```python
from pathlib import Path
from pypdf import PdfReader

def load_chunks(doc_dir: str, chunk_size: int = 500) -> list[dict]:
    chunks = []
    for path in Path(doc_dir).rglob("*.pdf"):
        reader = PdfReader(str(path))
        for page_num, page in enumerate(reader.pages):
            text = page.extract_text() or ""
            # 500文字ごとに分割（段落の切れ目は無視）
            for i in range(0, len(text), chunk_size):
                chunk = text[i:i + chunk_size].strip()
                if len(chunk) > 50:  # 短すぎるチャンクを除外
                    chunks.append({
                        "text": chunk,
                        "source": f"{path.name} p.{page_num + 1}"
                    })
    return chunks
```

### 3. BM25で関連チャンクを検索

```python
from rank_bm25 import BM25Okapi

def build_retriever(chunks: list[dict]):
    tokenized = [c["text"].split() for c in chunks]
    bm25 = BM25Okapi(tokenized)
    
    def retrieve(query: str, top_k: int = 5) -> list[dict]:
        scores = bm25.get_scores(query.split())
        top_indices = sorted(range(len(scores)), key=lambda i: scores[i], reverse=True)[:top_k]
        return [chunks[i] for i in top_indices if scores[i] > 0]
    
    return retrieve
```

### 4. ClaudeにContextを渡してQ&Aさせる

```python
import anthropic
import os
from dotenv import load_dotenv

load_dotenv()

def ask(query: str, retrieve, client: anthropic.Anthropic) -> str:
    relevant = retrieve(query, top_k=5)
    
    if not relevant:
        return "該当する資料が見つかりませんでした。"
    
    context = "\n\n---\n\n".join(
        f"【{c['source']}】\n{c['text']}" for c in relevant
    )
    
    message = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=1024,
        messages=[{
            "role": "user",
            "content": (
                f"以下の社内資料の抜粋を参照して、質問に答えてください。"
                f"資料に記載のない内容は「資料に記載がありません」と答えてください。\n\n"
                f"{context}\n\n質問: {query}"
            )
        }]
    )
    return message.content[0].text
```

### 5. 全体を組み合わせる

```python
if __name__ == "__main__":
    client = anthropic.Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])
    chunks = load_chunks("./docs")
    retrieve = build_retriever(chunks)
    
    print(f"{len(chunks)}チャンクを読み込みました")
    
    while True:
        query = input("\n質問: ").strip()
        if not query:
            break
        print(ask(query, retrieve, client))
```

## つまずきポイント

**日本語のトークナイズが効かない問題**
`text.split()`は空白区切りなので、日本語では1文が1トークンになり検索精度が落ちる。`janome`や`fugashi`で形態素解析してから渡すと改善する。ただし資料が英語混じりなら先にMeCabなしで試すこと。

**チャンクサイズと検索精度のトレードオフ**
500文字は目安に過ぎない。仕様書のように表が多い資料はページ単位（=大きめ）のほうが文脈が保たれる。議事録のように短い発言が続く資料は200文字程度が合う。

**スコア0のチャンクを混入させない**
`if scores[i] > 0`のガードは必須。BM25はクエリと無関係なチャンクにもスコアを返すため、上位5件を無条件で渡すとノイズだけのContextになりClaudeが誤答する。
