## 結論を先に

個人開発でVPSを選ぶなら、**まず月額料金と転送量の上限を確認し、次にSSH接続の安定性を検証する**のが最短ルートだ。ConoHa VPS・Xserver VPS・さくらのVPS・お名前.comを実際に触った感触では、用途別に向き不向きが明確に分かれる。

## 各サービスの実務的な特徴

### ConoHa VPS

時間課金と月額課金を併用できる点が個人開発に刺さる。検証用インスタンスを数時間だけ立ててすぐ削除する使い方ができるため、実験コストを抑えやすい。コントロールパネルのUIが比較的モダンで、OSイメージ選択からSSH鍵登録まで数クリックで完結する。

料金は公式サイトで頻繁に変わるため、本稿執筆時点の数値は載せない。**必ず公式料金ページで現在の価格を確認すること**。

### Xserver VPS

サーバー応答に安定感があり、長期運用向きという印象を持った。ゲームサーバーやBotを常時起動させるような用途で選ばれやすい。キャンペーン割引が多く、タイミングによっては実質コストを大きく下げられる。一方でUIはConoHaより古風な部分が残る。

### さくらのVPS

長年の実績があり、ドキュメントや日本語コミュニティが豊富。困ったときに検索すると古い記事でも参考になるものが多い。エントリープランの月額は比較的リーズナブルな印象だが、**料金は変動するため公式を参照**のこと。石橋を叩いて渡りたい[エンジニア](https://www.amazon.co.jp/s?k=%E3%82%A8%E3%83%B3%E3%82%B8%E3%83%8B%E3%82%A2%20%E6%9C%AC&tag=1280itsuya22-22)には安心感がある。

### お名前.com VPS

[ドメイン](https://px.a8.net/svt/ejp?a8mat=4B3XB4+4VMW8I+50+2HU3GX)管理と同一アカウントで扱える点が地味に便利。ドメインをお名前.comで取得している場合、DNS設定からサーバー管理まで一元化できる。ただしコントロールパネルの動線がやや複雑で、慣れるまで時間がかかった。

## 比較するべき3つのポイント

### 1. 料金体系

月額固定か時間課金かで総コストが変わる。個人開発では「本番は月額・検証は時間課金」と使い分けられるサービスが実務上有利だ。また、初期費用の有無・解約時の日割り計算の有無も確認する。**料金は各社のキャンペーンや為替で頻繁に変動するため、比較記事の数値をそのまま信じず、必ず契約直前に公式サイトで確認する**。

### 2. SSH接続の安定性と設定手順

VPSを選んだあと最初にやるべきことはSSHの鍵認証への切り替えだ。以下は共通の手順。

```bash
# ローカルで鍵ペアを生成（既にある場合はスキップ）
ssh-keygen -t ed25519 -C "your_comment"

# 公開鍵をVPSへ転送
ssh-copy-id -i ~/.ssh/id_ed25519.pub user@YOUR_SERVER_IP

# パスワード認証を無効化（/etc/ssh/sshd_config）
PasswordAuthentication no
PubkeyAuthentication yes

sudo systemctl reload sshd
```

接続テストは別ターミナルを開いたまま行う。設定ミスで締め出されてもコントロールパネルのコンソールから復旧できるが、手間がかかる。

また、SSHのポート番号をデフォルトの22番から変更しておくと、ブルートフォース攻撃のログノイズが大幅に減る。

### 3. 転送量(トラフィック)の上限と課金モデル

個人開発では転送量上限を超えることは少ないが、動画配信や大量のAPIレスポンスを返すサービスを作る場合は注意が必要だ。各社の仕様は以下の観点で確認する。

- 月あたりの転送量上限はあるか
- 上限超過時は速度制限か追加課金か
- インバウンド(受信)は無料か

実際に `vnstat` コマンドでサーバーのトラフィックを計測しておくと、請求前に異常を検知できる。

```bash
sudo apt install vnstat
vnstat -i eth0 --days   # 日別集計
vnstat -i eth0 --months # 月別集計
```

## DNS設定の実務メモ

VPSにドメインを向ける際、Aレコードの反映には最大数時間かかる。`dig` コマンドで引きに行くDNSサーバーを指定して反映確認するのが確実だ。

```bash
dig @8.8.8.8 yourdomain.example A
```

お名前.comでドメインを管理している場合、同社のVPSとの組み合わせはネームサーバー変更の手間が省けるが、DNSの管理UIが独特なため初回は公式ドキュメントを読みながら進めるのを勧める。
