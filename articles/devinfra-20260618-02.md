## なぜVPSか: bot運用に必要な要件を整理する

共用レンタルサーバー（いわゆるレンタルサーバー）ではcronの最短実行間隔が1分に制限されていたり、常駐プロセスを強制終了されたりするケースがある。Scraping・API呼び出し・通知送信など副作用の多いbotは、**自分だけのOSプロセスを持てるVPS**でないと安定しない。

チェックリスト:

- `systemd` または `cron` で常駐・定期実行できるか
- SSHキー認証でログインできるか（パスワード認証のみは避ける）
- 固定グローバルIPが付与されるか（Webhook受信・IP制限APIに必須）
- 月次トラフィック上限が十分か（API botなら転送量は小さいが余裕を見る）
- コントロールパネルからOSを再インストール・スナップショットが取れるか

## 主要サービスの特徴

### ConoHa VPS

SSD標準・APIでサーバー操作可能・東京リージョンのレイテンシが安定している。コントロールパネルからスナップショットを取れるため、設定を誤ったときのロールバックが速い。長期割引（1年・3年契約）と時間課金を選択できる柔軟さが個人開発者に向く。

### Xserver VPS

「Xserver」ブランドの知名度から初心者が選びやすい。Ubuntuなど主要ディストリビューションを選択でき、コントロールパネルのUIがわかりやすい。長期契約割引あり。料金は公式で確認を。

### さくらのVPS

老舗の安心感と安価な石狩・東京リージョンが魅力。最小プランが安く、軽量なAPIポーリングbotなら十分動く。ゾーンによってはIPv6対応状況が異なるので事前確認が必要。

### お名前.com VPS

[ドメイン](https://px.a8.net/svt/ejp?a8mat=4B3XB4+4VMW8I+50+2HU3GX)管理と同一アカウントで完結できるため、独自ドメイン＋SSL＋Webhookエンドポイントをセットで運用したい場合に便利。ただし、コントロールパネルのUI改善は他社に比べ遅い印象を持っているユーザーもいる。

## 具体的な初期セットアップ手順

以下はUbuntu 22.04 LTSを例にした最低限の手順だ。

```bash
# 1. rootでログイン後、まず一般ユーザーを作成
adduser botuser
usermod -aG sudo botuser

# 2. SSHキー認証を設定
mkdir -p /home/botuser/.ssh
cat your_public_key >> /home/botuser/.ssh/authorized_keys
chmod 700 /home/botuser/.ssh && chmod 600 /home/botuser/.ssh/authorized_keys

# 3. パスワード認証を無効化
sed -i 's/^PasswordAuthentication yes/PasswordAuthentication no/' /etc/ssh/sshd_config
systemctl restart sshd

# 4. UFWで必要ポートだけ開放
ufw allow 22/tcp
ufw enable
```

cronで定期実行する場合は `crontab -e` より **systemd timer** を推奨する。失敗時のログが `journalctl -u mybot.service` で追跡できるからだ。

## DNS・固定IPの使い方

Webhookを受け取るbotはIPを直打ちせず、**Aレコードで独自ドメインをVPSの固定IPに向ける**ことでIP変更に強い構成にできる。お名前.com・Cloudflare・Route 53など好みのDNSプロバイダで設定すればよい。
