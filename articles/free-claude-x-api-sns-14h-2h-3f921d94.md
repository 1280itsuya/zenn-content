# 結論：SNS情報収集の「巡回」は人間がやる仕事ではない

私は週に14時間、X（旧Twitter）でAI・自動化系の情報を追っていた。タイムラインを上から下まで眺め、有益そうなポストをブックマークし、後で読む——この巡回作業を Claude と X API で自動化したところ、**週14時間が約2時間まで圧縮**された。削れた12時間は、そのまま Book 執筆と受託に回せている。

仕組みはシンプルだ。

1. **X API v2 の `recent search`** で、対象キーワード（例：`Claude API`, `自動化`, `RAG`）を15分おきにポーリング
2. 取得した直近ポストを **Claude Haiku 4.5** に渡し、「自分にとって読む価値があるか」を0〜100でスコアリング
3. スコア70未満は自動ミュート（=通知もタイムラインも素通り）、70以上だけを Discord に要約付きで通知

ポイントは、フォロー/アンフォローではなく**「読む前にミュートで間引く」**こと。アカウント単位ではなくポスト単位で判定するので、普段は有益な人の宣伝ポストだけを静かに落とせる。

## 定量データ

実測値はこうなった（直近30日、対象キーワード8個）。

| 指標 | 自動化前 | 自動化後 |
|---|---|---|
| 巡回時間 | 14.2h/週 | 1.8h/週 |
| 1日の処理ポスト数 | 約400件（手動） | 約2,600件（自動） |
| 「読む価値あり」通知 | — | 1日11件 |
| API コスト | — | 月 約¥240 |

Haiku 4.5 は入力 $1 / 100万トークン。1ポスト平均120トークン、判定プロンプト込みでも1件あたり**約0.04円**。月2,600件×30日でも¥240に収まる。GPT を判定に併用するなら `gpt-4o-mini` でほぼ同コスト帯、精度比較も後述する。

## ハマりどころ

最初の壁は X API の Free プランだ。**`recent search` は Basic（月$100）以上**でないと叩けず、Free では取得数が実用に足りない。ここを回避するために、私は判定対象を「フォロー中リストのタイムライン」に絞り、`user timeline` 系で代替する構成に切り替えた。この設計判断が、月額を$100から$0に落とした分岐点だった。
