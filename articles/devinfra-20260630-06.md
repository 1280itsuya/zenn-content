# [Claude](https://www.anthropic.com/) Code常駐用に選ぶ格安VPSの条件

## 結論：2GB RAM・2コア・SSD・東京リージョンが最低ライン

Claude Code を SSH 越しにリモート常駐させる場合、ローカルマシンのスペックに依存せず 24 時間稼働できる点が最大のメリットだ。ただし「格安なら何でもいい」ではない。常駐プロセスとして動かすには **メモリ 2GB 以上、CPU 2 コア以上、SSD ストレージ、東京リージョン** を最低条件として揃える必要がある。これを下回るプランを選ぶと、ビルドや lint 実行のタイミングで OOM Killer に刈られ、セッションが無言で落ちる。

料金は各社とも随時変更されるため、契約前に必ず公式サイトの最新価格を確認してほしい。

---

## 条件の根拠

### メモリ：2GB が実質的な下限

Claude Code 本体（Node.js プロセス）は起動直後で 200〜400 MB 程度消費する。これに加えて、呼び出されるサブプロセス（tsc、eslint、pytest 等）がスパイク的にメモリを食う。1GB プランは OS とデーモン類だけで大半が埋まるため、複数ツールを並列実行したタイミングでスワップに落ちる。**2GB を最小値、余裕を持たせるなら 4GB** と覚えておくとよい。

### CPU：シングルコアは避ける

LLM の推論はサーバー側で行われるため VPS の CPU 性能は推論速度に影響しないが、ファイル探索（glob）・差分計算・テスト実行はローカル CPU に依存する。シングルコアの格安プランは並列タスクで詰まりやすく、レスポンスが体感で数倍遅くなる。2 コア以上を選ぶこと。

### SSD：HDD プランは論外

大規模リポジトリでのファイル読み書き、`node_modules` のインストールなど、I/O バウンドな処理が多い。HDD プランは同一 CPU・メモリでも体感が大きく下がる。現在の格安 VPS はほぼ SSD 標準だが、プラン詳細で「HDD」と書いてある場合は除外する。

### リージョン：東京一択

Claude Code は Anthropic の API を都度呼び出す。東京リージョンの VPS ならレイテンシが低く抑えられ、ストリーミング応答のチラつきが少ない。海外リージョンを選ぶと RTT が数十〜百数十 ms 増え、長い応答のストリーミングで体感差が出る。

料金は変動するため、以下は傾向の整理であり確定値ではない。

**ConoHa VPS**：時間課金と月額の切り替えが柔軟。コントロールパネルが使いやすく、スナップショット取得が簡単なので検証用途に向く。SSH 鍵はコンソールから一発で設定できる。

**Xserver VPS**：国内のホスティング大手が提供しており、サポートへのアクセスのしやすさが特徴。ネットワーク品質が安定しているという評判が多い。

**さくらのVPS**：老舗かつ長年の実績がある。カスタム OS イメージを使いたい場合や、細かいネットワーク設定が必要な場面で柔軟性が高い。

**お名前.com VPS**：[ドメイン](https://px.a8.net/svt/ejp?a8mat=4B3XB4+4VMW8I+50+2HU3GX)管理と同一アカウントで運用できる点が便利。ただしコントロールパネルの UI が古めという声も見かける。

どのサービスでも、**2GB RAM・2 コア・SSD・東京リージョン** のプランを選べばほぼ同じ用途に使える。価格差は数百円/月以内に収まることが多いため、コントロールパネルの使い勝手やサポート体制で選んでよい。

## 具体的なセットアップ手順

### 1. SSH 鍵認証を必ず有効にする

パスワード認証のまま運用するのは論外。VPS 契約直後に以下を実施する。

```bash
# ローカルで鍵ペア生成（既存があればスキップ）
ssh-keygen -t ed25519 -C "claude-code-vps"

# 公開鍵をサーバーへ転送
ssh-copy-id -i ~/.ssh/id_ed25519.pub user@YOUR_VPS_IP
```

その後 `/etc/ssh/sshd_config` で `PasswordAuthentication no` に変更し、`systemctl restart sshd`。

### 2. Claude Code のインストール

```bash
# Node.js（LTS）を入れる
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt install -y nodejs

# Claude Code CLI
npm install -g @anthropic-ai/claude-code
```

### 3. tmux で常駐させる

SSH 切断後もセッションを維持するために tmux を使う。

```bash
sudo apt install -y tmux
tmux new-session -d -s claude 'claude --dangerously-skip-permissions'
```

`tmux attach -t claude` で再接続できる。systemd サービス化すると再起動時も自動復帰する。

### 4. DNS と逆引き（任意）

固定 IP を持つ VPS なら、取得済みドメインの A レコードを向けておくと `ssh dev.example.com` でアクセスできて便利だ。各社コントロールパネルから IP は確認できる。
