## 発生条件

- [TypeScript](https://www.amazon.co.jp/s?k=TypeScript%20%E6%9C%AC&tag=1280itsuya22-22) で `import [React](https://www.amazon.co.jp/s?k=React%20%E5%85%A5%E9%96%80%20%E6%9C%AC&tag=1280itsuya22-22) from 'react'`（または `import { useState } from 'react'`）を書いた瞬間に `TS2307: Cannot find module 'react' or its corresponding type declarations.` が出る
- `react` 本体は `npm install` 済みなのに、型定義パッケージ `@types/react` を入れ忘れている
- `tsconfig.json` の `moduleResolution` が実際のパッケージ配置（`node_modules`）と噛み合っておらず、TypeScript が型宣言を解決できていない

## 原因

`react` の実体は入っていても、TypeScript はコンパイル時に「型宣言（`.d.ts`）」を別途探します。React 本体には型が同梱されていないため、`@types/react` が無いと `Cannot find module 'react' or its corresponding type declarations.` になります。最小再現は以下です。

```bash
# react は入れたが @types/react を入れていない
npm i react
```

```tsx
// App.tsx
import React from 'react'; // TS2307: Cannot find module 'react' ...

export const App = () => <div>hello</div>;
```

加えて、`tsconfig.json` の `moduleResolution` が古い設定だと、`node_modules` 配下の型を正しく辿れず同じエラーになることがあります。

```jsonc
// tsconfig.json（解決に失敗しやすい例）
{
  "compilerOptions": {
    "moduleResolution": "classic" // node_modules を見に行かない
  }
}
```

## 修正方法

まず型定義パッケージを開発依存として入れます。

```bash
npm i -D @types/react @types/react-dom
```

次に `moduleResolution` を `node_modules` 解決方式に揃えます。バンドラ（Vite 等）利用なら `bundler`、それ以外は `node` を指定します。

```jsonc
// tsconfig.json
{
  "compilerOptions": {
    "module": "esnext",
    "moduleResolution": "bundler", // 非バンドラ環境なら "node"
    "esModuleInterop": true,
    "jsx": "react-jsx"
  }
}
```

要点は3つです。

- `@types/react` を入れれば `Cannot find module 'react' or its corresponding type declarations.` の大半は解消する
- 型を入れても直らない場合は `moduleResolution` が `classic` のままでないか確認する
- 設定変更後はエディタの TypeScript サーバーを再起動（VS Code なら「TypeScript: Restart TS Server」）し、キャッシュされた解決結果を破棄する

`react-dom` を使うコードなら `@types/react-dom` も同時に入れておくと、同種の `TS2307` を後追いで踏まずに済みます。
