## 発生条件

- Node.js 12以降、または `"type": "module"` を指定したプロジェクトで `exports` フィールドを定義しているパッケージを利用している
- バンドラー（webpack 5、Vite、Rollup）が `exports` フィールドを参照する設定になっており、サブパス（例: `import 'my-pkg/utils'`）でインポートしている
- 利用側の `package.json` に `"type": "module"` が無い、あるいは tsconfig の `moduleResolution` が `node` のままになっている

## 原因

次のような `exports` 定義を持つパッケージが原因になりやすい。

```json
{
  "name": "my-pkg",
  "version": "1.0.0",
  "exports": {
    ".": "./dist/index.js",
    "./utils": "./dist/utils.js"
  }
}
```

この定義自体は正しいが、利用側の設定が古いと解決に失敗する。典型的な壊れ方は tsconfig の設定ミスだ。

```jsonc
// tsconfig.json（壊れている例）
{
  "compilerOptions": {
    "moduleResolution": "node"
  }
}
```

`moduleResolution: "node"` は Node.js 12以前の解決アルゴリズムを使う。このアルゴリズムは `exports` フィールドを一切参照しない。そのため `import 'my-pkg/utils'` とサブパスでインポートしようとすると、TypeScript はファイルシステム上の `node_modules/my-pkg/utils.js` や `node_modules/my-pkg/utils/index.js` を探しに行き、見つからずにエラーになる。

バンドラー側でも同様の問題が起きる。webpack 4 はデフォルトで `exports` フィールドを無視するため、webpack 4 から 5 へ移行していないプロジェクトでも同じエラーに遭遇する。

エラーメッセージは環境によって異なるが、よく見かける形式は以下の通りだ。

```
Cannot find module 'my-pkg/utils' or its corresponding type declarations.
```

または

```
Module not found: Error: Package path ./utils is not exported from package ...
```

後者は webpack 5 が `exports` フィールドを認識しているが、定義されていないパスへのアクセスを拒否している場合に出る。

## 修正方法

まず TypeScript を使っているなら `moduleResolution` を更新する。

```jsonc
// tsconfig.json（修正後）
{
  "compilerOptions": {
    "module": "ESNext",
    "moduleResolution": "bundler"
  }
}
```

Node.js 環境で TypeScript を直接実行する場合は `"moduleResolution": "node16"` または `"nodenext"` を使う。`"bundler"` は TypeScript 5.0 以降でのみ使用可能で、Vite や webpack などのバンドラー経由でビルドするフロントエンドプロジェクトに適している。

パッケージ側の `exports` 定義を修正する権限がある場合は、条件付きエクスポートで CommonJS と ESM の両方を提供すると互換性が広がる。

```json
{
  "exports": {
    ".": {
      "import": "./dist/index.mjs",
      "require": "./dist/index.cjs"
    },
    "./utils": {
      "import": "./dist/utils.mjs",
      "require": "./dist/utils.cjs"
    }
  }
}
```

webpack を使っているなら、`resolve.conditionNames` の設定も確認する。

```js
// webpack.config.js
module.exports = {
  resolve: {
    conditionNames: ['import', 'require', 'default'],
  },
};
```

デフォルトでは webpack 5 は `conditionNames` に `browser` や `import` を含むが、カスタム設定で上書きしている場合に条件が抜けてエラーになることがある。設定変更後は必ずキャッシュを削除してから再ビルドすること。
