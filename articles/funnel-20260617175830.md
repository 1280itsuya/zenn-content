# Claude Code に MCP サーバーを導入して開発を自動化する最小手順

Claude Code はデフォルトのツール群に加え、MCP（Model Context Protocol）サーバーを追加することで外部サービスや独自ツールを呼び出せるようになる。この記事では「動かすまでの最小手順」に絞って解説する。

---

## MCP サーバーとは

MCP は Anthropic が策定したオープンプロトコルで、Claude が外部ツールを統一インターフェース越しに呼び出すための仕組みだ。たとえば「GitHub のイシューを読む」「社内 DB を検索する」「ブラウザを操作する」といった処理を、Claude Code のチャット上から自然言語で依頼できるようになる。

---

## 前提

- Claude Code（CLI または VSCode 拡張）がインストール済み
- Node.js 18 以上がある（npm ベースのサーバーを使う場合）

---

## 手順 1：設定ファイルの場所を確認する

MCP サーバーの登録は `.claude/settings.json`（プロジェクトスコープ）または `~/.claude/settings.json`（グローバル）に書く。プロジェクト固有の用途なら前者を推奨する。

```jsonc
// .claude/settings.json
{
  "mcpServers": {}
}
```

ファイルがなければ作成する。

---

## 手順 2：サーバーを追加する

試しに **Filesystem MCP サーバー**（ローカルファイルへのアクセスを提供する公式サーバー）を追加する。

```bash
npm install -g @modelcontextprotocol/server-filesystem
```

設定ファイルにエントリを追加する：

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/path/to/allowed/dir"
      ]
    }
  }
}
```

`args` の末尾はアクセスを許可するディレクトリのパスだ。複数指定も可能。

---

## 手順 3：Claude Code を再起動して確認する

設定変更は再起動後に反映される。起動したら `/mcp` コマンドで接続状態を確認できる。

```
> /mcp
```

`filesystem: connected` と表示されれば成功だ。`failed` の場合は後述のつまずき所を参照する。

---

## 手順 4：実際に使う

接続後はチャットで普通に依頼するだけでよい。

```
src/以下のTypeScriptファイルをすべてリストアップして、
importの循環依存がないか確認してください
```

Claude Code が `filesystem` サーバーを通じてファイルを読み、分析結果を返す。

---

## GitHub MCP サーバーを追加する例

より実用的な例として GitHub サーバーも追加しておく。

```json
{
  "mcpServers": {
    "filesystem": { "...": "省略" },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "${GITHUB_PAT}"
      }
    }
  }
}
```

環境変数は `env` ブロックに書くか、シェル側でエクスポートしておく。PAT をそのまま設定ファイルに書くとリポジトリに混入するリスクがあるため、`${変数名}` 形式で参照するか `.env` を使う。

---

## よくあるつまずき所

**`failed to connect` が出る**
`command` に指定したバイナリがパスに存在しないケースが多い。`npx` の代わりに絶対パスを書くか、グローバルインストール後に `which npx` でパスを確認する。

**権限エラーで読めないファイルがある**
Filesystem サーバーは `args` で指定したディレクトリ以下しかアクセスできない。意図的な制限なので、必要なディレクトリを追加する。

**環境変数が渡らない**
Claude Code の起動元シェルで変数がエクスポートされているか確認する。VSCode 拡張経由だと GUI 起動のため `.bashrc` / `.zshrc` が読まれないことがある。`.env` ファイルを `env` ブロックで直接参照するか、ターミナルから起動する方が確実だ。

---

## まとめ

MCP 導入の最小手順は「npm でサーバーをインストール → `settings.json` に登録 → 再起動して `/mcp` で確認」の 3 ステップだ。公式リポジトリ（`modelcontextprotocol/servers`）には Filesystem・GitHub・Slack・PostgreSQL など主要なサーバーが揃っているので、用途に合わせて追加していくとよい。
