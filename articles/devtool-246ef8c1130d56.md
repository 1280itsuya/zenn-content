## Claude Code とは

Claude Code は Anthropic が提供するターミナル完結型の AI コーディングアシスタントだ。IDE プラグインではなく CLI として動作するため、SSH 先のサーバーや CI 環境でもそのまま使える。ファイル読み書き・シェル実行・Git 操作をモデル自身が行う設計なので、「コードを書いてもらう」より「タスクを一緒に片付ける」感覚に近い。ローカル環境を直接触らせることに慣れている開発者、あるいは副業で複数プロジェクトを並走させている人に特に向く。

## 起動速度

ターミナルで `claude` と打つと対話セッションが 2 秒台で立ち上がる。VS Code のような GUI エディタを開き直す手間がなく、作業中のシェルからそのまま呼び出せる点が日常ワークフローにはまりやすい。初回応答も体感的に速く、コマンドを入力してから最初のトークンが返るまでのラグはほぼ気にならない。ただし長い推論が必要な場面では最終出力まで数十秒かかることがある。

## 補完精度

Claude Code が得意なのは「複数ファイルにまたがる文脈を保持した修正」だ。たとえば `src/` 配下の 10 ファイルを参照しながら型定義を一括変更させる、あるいはバグの原因ファイルを自分で grep して該当箇所を特定し修正するといった作業を自律的にこなす。大規模コンテキストが強みなので、関数単位の小さな補完より、機能単位のリファクタリングや未知コードベースの把握で威力が出やすい。逆に、既存 LSP の補完のようなカーソル位置インライン予測は用途が異なるため比較対象にはならない。

## 月額コスト

個人利用の場合は **Claude Max** プランへの加入が現実的な選択肢で、月額 $20 前後から使える(上位ティアもある)。このプランは Claude.ai の会話利用と Claude Code の利用を定額内に含む。API キーをそのまま使う従量制とは異なり、コードを大量に読み書きさせてもコストが青天井にならない点が副業・個人開発には合っている。無料プランでも Claude Code 自体は試せるが、利用制限に早めに引っかかるため継続利用には有料プランが前提になる。

## 使ってみた所感

副業でアフィリエイトブログの自動化パイプラインを組んでいるとき、Claude Code に既存スクリプト群を渡して「Qiita への投稿が静かにスキップされる原因を調べて直して」と頼んだら、`load_dotenv()` の呼び出し漏れを自力で発見しその場で修正まで完了した。自分でログを追って原因特定するより明らかに早かった。Claude Code は「どこが壊れているか分からないが直したい」という状況でとくに頼りになる。コードを読む・仮説を立てる・修正する・確認するというループを人間の代わりに回してくれるので、本業の隙間に副業コードを進めるスタイルにフィットしやすい。
