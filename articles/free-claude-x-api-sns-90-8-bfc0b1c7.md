## 結論：SNS収集の「読む前の仕分け」を捨てたら90分が8分になった

毎朝、X（旧Twitter）で副業・AI関連のリストを巡回し、ノイズを目で弾いて保存する作業に90分かかっていた。これを Claude（claude-haiku-4-5）と X API v2 を組み合わせ、**1日あたり8分**まで圧縮した。鍵は「ミュートすべき投稿を人間が判断しない」ことだ。

### 何をやめたか

従来は `#副業` などのキーワード検索を目視で読み、宣伝・煽り・情報商材を手で除外していた。ここが律速だった。読む量が多いほど判断が雑になり、後で見返すと保存の半分がゴミだった。

### 構成

処理はシンプルな3段だ。

1. **X API v2** の `recent search` で直近100件を取得（Essentialの月100万ツイート枠内で十分）
2. 取得した本文を **Claude にバッチ投入**し、`keep / mute` と理由を JSON で返させる
3. `mute` 判定をローカルの除外リストへ追記し、次回検索から自動で弾く

判定プロンプトの核心は、自分の除外基準を5行の箇条書きで固定したことだ。「アフィリエイト誘導」「再現性の数値がない成功談」「フォロー誘導が本文の主目的」——この粒度まで言語化すると、Claudeの判定が自分の手作業と92%一致した（直近300件を手で答え合わせした実測値）。

### コスト

100件の判定で入力約12,000トークン、出力約2,000トークン。haiku換算で**1日あたり約1.5円**。月45円で90分×30日＝45時間が返ってくる計算になる。

### つまずいた点

最初は claude-opus を使い精度100%を狙ったが、判定タスクにはオーバースペックでコストが20倍に膨らんだ。「分類は安いモデル＋良いプロンプト」が正解だった。また X API のレート制限（15分あたり180リクエスト）に最初引っかかったので、検索は1日1回バッチに寄せている。
