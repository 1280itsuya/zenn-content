## 結論から言う

個人開発でVPSを選ぶとき、「どこが速いか」より「どこが**運用で詰まらないか**」の方が重要だった。SSH鍵認証の初期設定、転送量上限の透明度、料金の予測しやすさ——この3点で各社に明確な差がある。

筆者が実際に使ってきた感触を軸に、ConoHa VPS・Xserver VPS・さくらのVPSの3社を比較する。お名前.com VPSは過去に試したが現在は使っていないため補足程度に触れる。

> **注意**: 料金プランは頻繁に改定される。本記事の数値はあくまで参考値であり、契約前に必ず公式ページで最新料金を確認すること。

---

## 料金感の実態

### ConoHa VPS

最小構成(例: メモリ512MB)で**例: 月額650円前後**から使える時間帯がある。「時間課金」と「月額課金」を切り替えられる点が他社にない特徴で、検証環境を一時的に立ち上げてすぐ削除するワークフローに向いている。ただし長期割引を使わない場合、月額換算は割高になりやすい。

### Xserver VPS

料金体系がシンプルで、**例: 月額830円程度(最小プラン)**の固定料金がメイン。長期契約の割引率が明示されているため、1年・2年で運用するプロジェクトなら総額計算しやすい。ConoHaの時間課金ほど柔軟ではないが、「毎月いくら払うか」が読みやすい。

### さくらのVPS

老舗だけあって最小プランは**例: 月額600円台〜**と安価な帯域がある。ただしデフォルトの転送量上限が他社より小さいプランが存在するため、帯域仕様を必ず確認する必要がある。2週間の無料お試し期間が公式にある点は評価できる。

### お名前.com VPS

過去に使った印象では料金はやや高め。コントロールパネルの動作が重く、初期設定に手間取った記憶がある。今は積極的には選ばない。

## SSH接続の初期設定でどこが詰まるか

VPSを借りて最初にやることはSSH鍵認証の設定だが、**コントロールパネル上での操作フローが各社で異なる**。

### 共通手順

```bash
# ローカルで鍵ペア生成(既存があればスキップ)
ssh-keygen -t ed25519 -C "your_email"

# 公開鍵の中身を確認
cat ~/.ssh/id_ed25519.pub
```

- **ConoHa**: サーバー作成時に公開鍵を貼り付けるUIがある。作成後の追加も可能だが、コントロールパネルの「コンソール」からrootで一度ログインして`~/.ssh/authorized_keys`に追記する手順が必要になる場合がある。
- **Xserver VPS**: サーバー作成画面で公開鍵を登録できる。UIが比較的素直で初回設定は迷いにくかった。
- **さくらのVPS**: デフォルトはパスワード認証のため、初回ログイン後に自分で`sshd_config`を変更してパスワード認証を無効にする手順が必要。この作業を忘れると辞書攻撃を受けやすい状態で放置される。

```bash
# さくらのVPS: 初回ログイン後に実施する設定例
sudo nano /etc/ssh/sshd_config
# PasswordAuthentication no に変更
sudo systemctl restart sshd
```

どの会社でも**ポート22のデフォルト開放**は変えた方がいい。`/etc/ssh/sshd_config`でポートを変更し、ファイアウォールで22を閉じるのが基本だ。

## 転送量(帯域)の扱い

転送量上限は各社で「月○GB」や「共有帯域○Mbps」など表記形式が違い、単純比較しにくい。

- **ConoHa・Xserver**: 上位プランでは転送量無制限または大容量を謳っているケースが多い。ただし「ベストエフォート」であり、混雑時の帯域保証ではない。
- **さくらのVPS**: 低価格プランは転送量の上限が明記されているプランがあるため、画像配信や動画転送を行うサービスには注意が必要。

個人開発のWebアプリやAPIサーバー程度なら、月間転送量が問題になるケースはほぼない。問題になるのはファイル配信や動画ストリーミングを兼ねる場合だ。

## 選択の指針

| ユースケース | 向いてるサービス |
|---|---|
| 検証環境を都度作って壊す | ConoHa(時間課金) |
| 長期運用・料金固定 | Xserver VPS |
| 無料お試しで比較したい | さくらのVPS |
| 初心者がSSH設定で詰まりたくない | Xserver VPS |

スペックのベンチマークより、**自分のワークフローに合ったコントロールパネルと料金体系を選ぶこと**が長期的な運用負荷を下げる。どの会社も最小プランで始めて、必要になったらプランアップグレードするのが無駄のない進め方だ。
