# Claude Code のサブエージェント／フックで定型作業を自動消去する実践設定

## なぜフックとサブエージェントか

Claude Code には「モデルに指示するだけ」以外の自動化レイヤーが2つある。**フック**（settings.json の `hooks` セクション）と**サブエージェント**（`Agent` ツール経由の並列タスク）だ。この2つを組み合わせると、「コミット前に毎回手でリント走らせる」「テスト失敗したら原因を毎回説明させる」といった定型ループをセッション外に押し出せる。

---

## 1. フック：ツール呼び出しの前後に任意コマンドを差し込む

### 設定ファイルの場所

プロジェクトローカルは `.claude/settings.json`、グローバルは `~/.claude/settings.json`。

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "echo '[hook] Bash実行前チェック: $CLAUDE_TOOL_INPUT' >> /tmp/claude_audit.log"
          }
        ]
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Edit",
        "hooks": [
          {
            "type": "command",
            "command": "cd $CLAUDE_PROJECT_DIR && npx eslint --fix $CLAUDE_TOOL_INPUT_FILE_PATH 2>&1 || true"
          }
        ]
      }
    ]
  }
}
```

`PreToolUse` はツール実行前、`PostToolUse` は実行後に走る。`matcher` にはツール名（`Bash` `Edit` `Write` `Read` など）を指定する。

### 実用例：Edit のたびに自動フォーマット

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit",
        "hooks": [
          {
            "type": "command",
            "command": "prettier --write \"$CLAUDE_TOOL_INPUT_FILE_PATH\" 2>/dev/null || true"
          }
        ]
      },
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "command",
            "command": "prettier --write \"$CLAUDE_TOOL_INPUT_FILE_PATH\" 2>/dev/null || true"
          }
        ]
      }
    ]
  }
}
```

Claude が Edit/Write を呼ぶたびに Prettier が走る。モデルへの「フォーマットして」という指示が不要になる。

### 環境変数一覧（フック内で使える主なもの）

| 変数名 | 内容 |
|---|---|
| `CLAUDE_TOOL_INPUT_FILE_PATH` | Edit/Write 対象のファイルパス |
| `CLAUDE_TOOL_INPUT` | ツール入力全体（JSON文字列） |
| `CLAUDE_PROJECT_DIR` | プロジェクトルート |
| `CLAUDE_SESSION_ID` | 現在のセッション ID |

## 2. つまずきポイント：フックが「静かに失敗」する

フックのコマンドが終了コード非0を返すと、Claude はエラーをモデルに伝えるがセッションは続行する（`|| true` を付けていない場合、ツール呼び出しをブロックすることもある）。

**よくある落とし穴**：
- `$CLAUDE_TOOL_INPUT_FILE_PATH` が空になるケース（Bash ツールには渡ってこない）
- Windows PowerShell 環境で `2>/dev/null` が構文エラーになる → `2>$null` に変える
- フック内でさらに Claude CLI を呼ぶとセッション競合が起きる（`~/.claude.json` の書き込み競合）

---

## 3. サブエージェント：並列タスクをモデルに割り当てる

Claude Code は `Agent` ツールを内部で使い、独立したタスクを別エージェントへ委譲できる。プロンプト内で「これとこれを並行してやれ」と指示するだけでは直列になりがちだが、明示的に Agent ツールを呼ぶよう促すと並列実行される。

### 使い方：CLAUDE.md に並列化の指示を書く

```markdown
## 並列タスクの方針
独立したファイル調査（Grep/Read）が3件以上ある場合は Agent ツールで並列化すること。
例：「A.ts と B.ts と C.ts を別々に調べる」→ 3つの Agent を同時起動。
```

### 実践：コードレビューを並列サブエージェントに割り当てる

プロジェクトの `.claude/settings.json` に以下を追加しておくと、レビュー系タスクでの挙動が変わる：

```json
{
  "permissions": {
    "allow": [
      "Agent"
    ]
  }
}
```

そしてプロンプトで：

```
変更されたファイルを3つのサブエージェントに分けてレビューさせ、
それぞれの結果をまとめて報告してください。
```

## 4. 実務で効いた設定セット

### コミット前の自動テスト（PreToolUse × Bash）

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "if echo '$CLAUDE_TOOL_INPUT' | grep -q 'git commit'; then npm test --passWithNoTests 2>&1; fi"
          }
        ]
      }
    ]
  }
}
```

`git commit` を含む Bash コマンドが来たときだけテストを先に実行する。テスト失敗なら Claude がコミットを止めて原因調査に入る。

### 新規ファイル作成時に lint チェック

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "command",
            "command": "FILE=\"$CLAUDE_TOOL_INPUT_FILE_PATH\"; case $FILE in *.ts|*.tsx) npx tsc --noEmit --skipLibCheck 2>&1 | head -20 ;; esac"
          }
        ]
      }
    ]
  }
}
```

TypeScript ファイルが書かれるたびに型チェックを走らせ、結果をモデルが確認できる状態にする。

## まとめ

- **フック**は「モデルが特定ツールを呼んだタイミング」に外部コマンドを差し込む仕組みで、フォーマット・リント・ログなど副作用系の定型を完全に自動化できる
- **サブエージェント**は独立タスクの並列実行に使い、直列待ちを減らす
- フックの失敗は静かなことがあるので、`2>&1` でログを拾ってデバッグすること
- Windows 環境では bash 構文が使えないフックコマンドに注意（PowerShell 構文で書く）

設定ファイルに書いておけばセッションをまたいで効くため、「毎回 Claude に同じことを頼む」ループを構造的に断ち切れる。
