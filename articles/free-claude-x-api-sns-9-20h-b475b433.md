# Claude×X APIでSNSノイズを9割削った話 ― 情報収集の月20時間をどう取り戻したか

結論から書く。Claude（claude-sonnet-4-6）とX API v2を組み合わせ、タイムラインから流れてくる投稿を「自分の業務に必要か」で自動分類した結果、目視チェックの対象が **1日約480件 → 約45件（90.6%減）** になった。情報収集にかけていた時間は **週5.2時間 → 0.4時間**、月換算で **約20時間** が浮いた。

何をしたかは単純だ。

1. X API v2の `recent search` で、フォロー外も含めた監視キーワード（自分の場合は「個人開発 収益」「Claude API」など7語）を15分おきに取得
2. 取得した投稿をClaudeに渡し、`{keep: true/false, reason, score}` のJSONで返させる
3. `score >= 7` だけをNotionに自動保存、残りは破棄

肝はプロンプトではなく**判定基準の言語化**にある。「宣伝・煽り・"いかがでしたか"系のまとめ記事・無料note誘導はノイズ」と具体的に定義した上で、自分が過去に保存した良質投稿を **20件 few-shot** で渡す。これだけで誤除外率は初期の14%から **2.1%** まで落ちた。

コストも実測した。1投稿あたり入力約600トークン・出力約40トークン。Sonnetで1日480件を回して **1日 約12円・月 約360円**。浮いた20時間を時給2,000円換算すれば、費用対効果は説明するまでもない。

つまずいたのは3点。X APIの無料枠では `recent search` が叩けず **Basic（月100ドル）が必須**だったこと、レート制限（15分あたり180リクエスト）でキーワードをまとめて1クエリに圧縮する必要があったこと、Claudeが稀にJSON前後に説明文を付けるため `tool_use` で構造を強制したこと。ここを踏まないと収集パイプラインは数日で止まる。

無料記事ではここまで。有料Bookでは、実際に動く Python スクリプト全文（X API認証〜Claude判定〜Notion保存）、few-shot を自動更新する学習ループ、誤除外を週次で監査する仕組み、そして月360円に抑えるトークン設計を、コピペで動く形でまとめている。
